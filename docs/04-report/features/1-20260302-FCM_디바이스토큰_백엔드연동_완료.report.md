# FCM 디바이스 토큰 백엔드 연동 완료 보고서 (Report)

## 1. 개요
*   **작업 내용:** 앱 배포(APK 자체 배포 등 모든 배포 채널) 시 **푸시 알림**을 수신하기 위해 기기의 FCM 고유 토큰을 서버 데이터베이스에 안전하게 보관 및 동기화하는 기반 작업을 완료했습니다.
*   **작업 기간:** 2026-03-02

## 2. 주요 구현 내용

### 2.1. 데이터베이스 스키마 확장 (`shared/schema.ts`)
*   `user_devices` 테이블을 새로 설계 및 Drizzle ORM 마이그레이션을 적용했습니다 (`npm run db:push`).
*   한 명의 사용자가 여러 개의 디바이스(앱, 웹 등)를 사용하더라도 모두 개별적으로 토큰을 추적할 수 있도록 1:N 관계(`user` -> `user_devices`)로 맵핑했습니다.
*   `isActive` 및 `lastUsedAt` 컬럼을 도입하여 죽은 토큰에 불필요한 푸시 알림을 발송하지 않도록 최적화 설계를 마쳤습니다.

### 2.2. 토큰 수집 API 엔드포인트 구축 (`server/routes/profile.ts`)
*   `POST /api/users/device-token` API를 새롭게 구성했습니다.
*   **스마트 스위칭(Upsert) 로직 도입:** 
    *   완전 처음 연결하는 기기면 신규 등록 (Insert)
    *   만약 친구의 폰으로 내 아이디를 로그인했다면, 해당 기기의 토큰 소유자를 내 아이디로 자동 갱신 (Update)
    *   앱을 켤 때마다 `lastUsedAt`을 현재 시간으로 갱신하여 "활성 사용자"임을 증명합니다.

### 2.3. 프론트엔드 - 백엔드 연동 (`client/src/hooks/usePushNotifications.ts`)
*   사용자 앱이 켜지고 알림 권한을 승인받아 `token`을 성공적으로 수신하는 즉시 백엔드 API로 토큰을 쏘아주는 `fetch` 로직을 주입했습니다.
*   `Capacitor.getPlatform()` 코드를 통해 현재 웹인지, 안드로이드인지 추적하는 `deviceType` 파라미터도 함께 서버에 전송되도록 처리 완료했습니다.

## 3. 결론 및 향후 계획 (Next Steps)
모바일 앱의 가장 중요한 뼈대인 푸시 알림 서버 수집 연동이 **Web/OS 무관하게** 호환되도록 완벽히 종료되었습니다. 

이제 앱을 감싸고 다운로드 형태로 추출하는 **진짜 모바일 앱 빌드(Capacitor + Android Studio)** 작업을 진행할 준비가 끝났습니다.
