# 회원 탈퇴 기능 (소프트 삭제) 완료 보고서

## 1. 개요
- **작업일**: 2026-03-05
- **기능**: 일반 사용자 회원 탈퇴 기능 (소프트 삭제 방식 지원)
- **목적**: 기존 데이터를 보존하면서 사용자 계정을 '탈퇴' 상태로 전환하여 로그인 및 접근을 차단하고, 관리자 단에서 이를 인지 및 관리할 수 있도록 함.

## 2. 작업 내용
### 2.1. 데이터베이스 스키마 수정
- `shared/schema.ts`의 `users` 테이블에 두 개의 컬럼을 추가했습니다.
  - `isDeleted`: 탈퇴 여부를 나타내는 Boolean 필드 (기본값 false).
  - `deletedAt`: 탈퇴 시점을 기록하는 Timestamp 필드.

> **주의**: 로컬 환경에서 `npx drizzle-kit push`를 실행하면 다른 기존 테이블들의 대규모 데이터 유실(컬럼 삭제 및 테이블 크론)이 경고되므로 자동 푸시를 중단했습니다. 실제 데이터베이스 반영은 `ALTER TABLE users ADD COLUMN is_deleted boolean DEFAULT false; ALTER TABLE users ADD COLUMN deleted_at timestamp;` SQL 스크립트를 수동으로 실행하여 반영해야 합니다.

### 2.2. 백엔드 API 및 인증 로직 업데이트
- **탈퇴 API 추가**: `server/routes/profile.ts`에 `DELETE /api/users/me` 엔드포인트를 추가하여 사용자가 스스로 탈퇴할 수 있도록 구현했습니다 (isDeleted 처리 및 세션 파기).
- **인증(Auth) 차단**:
  - `server/services/auth.ts`: Local Strategy 및 JWT 미들웨어에 `isDeleted` 확인 로직 추가. 탈퇴한 회원이 로그인 시도 시 403 에러 또는 "탈퇴한 계정입니다" 메시지 반환.
  - `server/routes/auth.ts`: Firebase 연동 시 발급되는 JWT 라우트 로직에 탈퇴 회원 차단 추가.
  - `server/routes/google-oauth.ts`: 구글 OAuth 로그인 콜백에 탈퇴 회원 차단 분기 추가.
- **관리자 API 수정**: `server/routes/admin-routes.ts`에서 사용자 목록 조회 시 `isDeleted`와 `deletedAt` 필드를 포함하도록 변경했습니다.

### 2.3. 프론트엔드 UI/UX 구현
- **계정 설정 페이지**: `client/src/pages/account-settings.tsx`의 '설정' 탭 하단에 "위험 구역(Danger Zone)" 블록과 "회원 탈퇴" 버튼을 추가했습니다.
- **탈퇴 확인 다이얼로그**: 휴먼 에러를 방지하고 탈퇴 후의 제약 사항을 안내하기 위해 `UserSelfDeleteConfirmDialog` 컴포넌트를 분리하여 구현했습니다.
- **관리자 회원 목록**: `client/src/components/admin/MemberManagement.tsx`에서 탈퇴한 사용자의 행(Row)은 시각적으로(붉은색 배경, 반투명 효과, 취소선, [탈퇴] 뱃지 등) 구분되도록 UI 처리를 추가했습니다.

## 3. 향후 과제 (Next Steps)
1. **데이터베이스 반영**: 운영 환경 또는 스테이징 환경에서 `users` 테이블에 `isDeleted`, `deletedAt` 컬럼 추가 (필수).
2. **관리자 완전 삭제 기능**: 계획대로 추후 관리자가 탈퇴 회원을 선택하여 '완전 삭제(Hard Delete)' 할 수 있는 기능 추가.
3. **개인정보 파기 정책 반영**: 탈퇴한 사용자의 정보 중 법적 의무 보관 기간이 만료된 데이터를 자동으로 삭제하는 스케줄링(Cron) 작업 도입 검토.
