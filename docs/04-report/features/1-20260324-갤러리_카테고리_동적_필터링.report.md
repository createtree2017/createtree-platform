# [Report] 갤러리 카테고리 필터 동적 표시 수정 완료

- 작성일: 20260324
- 최종수정일: 0324
- 결과: 성공

## 1. 개요
- 사용자의 갤러리 상단 필터에서 이미지가 있는 카테고리만 버튼으로 노출되도록 개선하였습니다.
- 이를 통해 미사용 카테고리나 개발 중인 메뉴가 비어 있는 상태로 노출되는 문제를 해결하였습니다.

## 2. 변경 내용
### 2.1. 프론트엔드 (`client/src/components/GalleryEmbedSimple.tsx`)
- **전체 이미지 조회 쿼리 추가**: `useQuery`를 사용하여 사용자 전체 이미지 목록(`all`)을 백그라운드에서 조회합니다.
- **활용 카테고리 계산**: 조회된 데이터의 `type` 필드를 분석하여 이미지가 1개 이상 존재하는 카테고리 ID 세트를 동적으로 계산합니다.
- **필터 버튼 필터링**: `GALLERY_FILTERS` 배열을 렌더링하기 전, 위에서 계산된 활성 카테고리 세트에 포함된 경우에만 버튼을 생성하도록 수정하였습니다.
- **실시간 갱신 보장**: 이미지 생성(`imageCreated`) 또는 삭제 시, 전체 이미지 목록 쿼리를 무효화(`invalidateQueries`)하여 필터 목록이 즉시 업데이트되도록 하였습니다.

## 3. 확인 결과
- 이미지가 없는 카테고리는 상단 버튼 목록에서 사라집니다.
- '전체' 버튼은 항상 유지되어 모든 이미지를 볼 수 있습니다.
- 새로운 카테고리의 이미지가 생성되면 즉시 해당 카테고리 버튼이 나타납니다.

## 4. 향후 계획
- 현재는 프론트엔드에서 전체 데이터를 기반으로 필터링하고 있으나, 유저당 이미지 수가 수천 개 이상으로 많아질 경우 백엔드에서 집계된 카테고리 목록만 반환하는 전용 API 도입을 검토할 수 있습니다.
- (현재 수준의 유스케이스에서는 현재 구현 방식으로도 충분히 빠르고 안정적입니다.)
