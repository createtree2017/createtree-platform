# 1-20260428-GPT_Image_2_모델_추가 완료 보고서

## 작업 요약
OpenAI의 새로운 모델인 `gpt-image-2`와 `gpt-image-2-mini`를 시스템에 연동하기 위한 마이그레이션 작업(DB 스키마 추가, 서비스 함수 구현, 관리자 페이지 UI 연동, 이미지 라우트 연동)을 성공적으로 완료했습니다.

## 핵심 변경 내용

1. **데이터베이스 스키마 확장** (`shared/schema.ts`)
   - `AI_MODELS` 상수에 `OPENAI_GPT2` ('openai_gpt2') 및 `OPENAI_GPT2_MINI` ('openai_gpt2_mini') 추가.
   - `systemSettings`의 `supportedAiModels` 및 모델 타입에 신규 모델 허용 추가.
   - `drizzle-kit push`를 통해 데이터베이스 스키마 실시간 반영 완료.

2. **OpenAI 서비스 레이어 분리 및 신규 모델 지원** (`server/services/openai.ts`, `server/routes/image.ts`)
   - 기존의 `openai-dalle3.ts`는 그대로 유지하여 하위 호환성을 확보했습니다.
   - 라우터(`image.ts`)의 이미지 생성 및 다중 변환 로직(`transformWithOpenAI`, `transformWithOpenAIMulti`)에서 `finalModel`이 `openai_gpt2` 계열일 경우 분기 처리하여, 모델명 파라미터로 `gpt-image-2` 또는 `gpt-image-2-mini`를 주입하도록 구현 완료했습니다.

3. **시스템 설정 및 프론트엔드 연동** (`server/utils/settings.ts`, `client/src/hooks/useModelCapabilities.ts`)
   - `useModelCapabilities` 훅의 기본값 객체에 `openai_gpt2`, `openai_gpt2_mini`의 지원 해상도(Aspect Ratio)를 등록했습니다.
   - `settings.ts`의 `SystemSettings` 타입 검증에서 발생한 TypeScript 컴파일 에러를 완벽하게 수정하여 컴파일 오류가 없도록 조치했습니다.
   - 폐기 모델 필터링 로직에서 새 모델들이 제외되지 않도록 안전 장치를 마련했습니다.

4. **UI 구성 요소 추가** (`client/src/components/ImageGenerationTemplate.tsx`, `client/src/components/modal/admin/ConceptFormModal.tsx`)
   - 프론트엔드 관리자 페이지의 **컨셉 추가/수정 모달**에 `GPT-Image-2` 및 `GPT-Image-2 Mini` 모델 선택 체크박스 UI를 추가했습니다.
   - 사용자용 **이미지 생성 템플릿**에 `GPT-Image-2`와 `GPT-Image-2 Mini` 모델 버튼을 동적으로 렌더링하도록 반영했습니다.

## ⚠️ API 호출 내부 테스트 결과 (필독)

개발 완료 후 백그라운드에서 백엔드 단독으로 OpenAI API 호출 테스트 스크립트(`scripts/test-openai-gpt2-json.ts`)를 실행하여 검증한 결과, 아래와 같은 응답을 확인했습니다.

```json
{
  "error": {
    "message": "The model 'gpt-image-2-mini' does not exist.",
    "type": "image_generation_user_error",
    "param": "model",
    "code": "invalid_value"
  }
}
```

OpenAI 공식 서버에서 해당 모델명(`gpt-image-2-mini`, `gpt-image-2`)을 지원하지 않거나, 아직 계정/키에 롤아웃되지 않은 것으로 보입니다.
