# 일반 사용자 회원탈퇴 기능 개발 계획

사용자 요구사항에 따라, 일반 사용자가 탈퇴 시 **모든 데이터를 그대로 남겨두고 상태만 '탈퇴'로 변경(Pure Soft Delete)** 합니다.
이후 **관리자(Admin) 페이지에 삭제 기능을 추가**하여, 관리자가 직접 탈퇴한 회원의 모든 데이터(DB, 스토리지 파일 등)를 완전 삭제할 수 있도록 구현합니다.

## Proposed Changes

### Database Schema

---
#### [MODIFY] `shared/schema.ts`
- `users` 테이블에 Soft Delete 상태를 관리하기 위한 필드 추가:
  - `isDeleted`: boolean (default: false)
  - `deletedAt`: timestamp (nullable)

### Backend (Server)

---
#### [NEW] / [MODIFY] `server/routes/profile.ts` (또는 `user-settings.ts`)
- **[일반 사용자용]** `DELETE /api/users/me` API 엔드포인트 추가.
- **처리 로직 (Pure Soft Delete)**:
  1. 사용자 인증 확인.
  2. `users` 테이블 업데이트: `isDeleted = true`, `deletedAt = now()`. (데이터는 아무것도 지우지 않음)
  3. 로그인 불가 처리를 위해, 선택적으로 소셜 토큰 등을 무효화할지 결정 (일반적으로 세션만 끊고, 이후 로그인 시도 시 `isDeleted`를 체크해 막음).
  4. 세션 파기(`req.logout()`, `req.session.destroy()`) 및 200 OK 반환.

#### [MODIFY] `server/routes/auth.ts` (로그인 라우트)
- 로그인 시도 시, `isDeleted === true`인 유저는 "탈퇴한 계정입니다"라는 메시지와 함께 로그인을 차단하는 방어 로직 추가.

#### [MODIFY] `server/routes/admin-routes.ts`
- **[관리자용]** 기존 유저 관리 API 로직 보강:
  - 유저 목록 조회(`GET /api/super/users`) 시, `isDeleted`, `deletedAt` 필드 포함 반환.
  - 완전 삭제 API (`DELETE /api/super/users/:id`):
    - **(기존 로직 분석 완료)** 관리자가 이 API를 호출하면, **해당 유저의 모든 연결 레코드(이미지, 음악 등 DB 데이터)와 실제 스토리지 파일(GCS 등)을 영구 삭제**하도록 처리합니다.

### Frontend (Client)

---
#### [MODIFY] `client/src/pages/account-settings.tsx`
- 계정 설정 하단에 **"회원 탈퇴" 버튼** 추가.

#### [NEW] `client/src/components/dialogs/UserSelfDeleteConfirmDialog.tsx`
- 탈퇴 재확인 모달 다이얼로그 추가.
- "정말 탈퇴하시겠습니까? 탈퇴 시 로그인이 제한 되고 보유하신 모든 자료가 삭제될 수 있습니다." 구조의 경고 문구 표시.

#### [MODIFY] `client/src/components/admin/MemberManagement.tsx` (관리자 페이지)
- 유저 목록 테이블에 '상태(정상/탈퇴)' 및 탈퇴일시 컬럼 노출 처리.
- 탈퇴한 회원만 필터링해서 볼 수 있는 기능 제공.
- 기존에 존재하는 **[삭제]** 버튼(`UserDeleteConfirmDialog` 호출)을 통해, 관리자가 최종적으로 "완전 삭제(Hard Delete)"를 수행할 수 있도록 UI 연동.

## Verification Plan

### Manual Verification
1. **[사용자] 회원 탈퇴 처리 검증**
   - 개발 환경에서 테스트 계정 설정 -> [회원 탈퇴] 진행.
   - 로딩 완료 후 즉시 로그아웃되는지 확인.
   - 해당 계정으로 **다시 로그인 시도 시 차단**되는지(Auth 로직) 확인.
   - DB에 들어가서 해당 유저의 `isDeleted`가 `true`이고, 만든 이미지/음악 등 데이터는 그대로 살아있는지 쿼리로 확인.
2. **[관리자] 탈퇴 회원 관리 및 완전 삭제 검증**
   - 최고 관리자(`super`) 계정으로 어드민 접속.
   - User Management 탭에서 방금 탈퇴한 회원이 '탈퇴' 상태로 노출되는지 확인.
   - 해당 회원 우측의 **[삭제]** 버튼을 눌러 완전 삭제 진행.
   - DB상 해당 유저와 유저가 만든 콘텐츠(이미지/음악)가 말끔하게 지워지는지, 그리고 스토리지(GCS)에서도 실제 파일이 파기되는지 확인.
