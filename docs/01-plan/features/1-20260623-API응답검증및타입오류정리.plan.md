# API 응답 검증 및 타입 오류 정리 Plan

## 작업 개요

관리자 승인취소 기능 검증 중 확인된 API HTML fallback false success 위험과 `npm run verify` 환경변수 검증 오류를 보강한다. 동시에 전체 TypeScript 검증을 막던 기존 타입 오류 중 사이트 구동에 직접 위험이 낮은 범위에서 정리한다.

## 목표

- 미등록 `/api/*` 요청이 HTML `200 OK`로 성공 처리되지 않게 한다.
- 프론트 공통 `apiRequest`가 API 경로의 HTML 응답을 성공으로 오판하지 않게 한다.
- `npm run verify`가 `.env`를 읽고 실제 개발 환경 기준으로 검증되게 한다.
- 기존 `Persona*`, `milestones.ts` 타입 오류를 정리해 전체 타입 검증 통과를 목표로 한다.

## 작업 관리표

## 제외

- DB 스키마 변경
- 운영 데이터 직접 수정
- 브라우저 자동 검증 (`!!테스트!!` 없음)
- git add/commit/push (`!!푸시!!` 없음)

## 검증 결과

- `npm run check -- --pretty false`: 통과
- `npm run verify`: 통과
- `git diff --check`: 통과
- `npm run build`: 통과
- 임시 dev 서버 `PORT=5010` 런타임 확인:
  - `POST /api/__missing_route_false_success_probe` → `404 application/json`
  - `POST /api/admin/review/submissions/845/cancel-approval` 인증 없음 → `401 application/json`
  - 일반 클라이언트 라우트 → `200 text/html`
