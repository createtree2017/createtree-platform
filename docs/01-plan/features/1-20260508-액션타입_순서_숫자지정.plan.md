# 액션 타입 순서 숫자 지정 기능 추가 Plan

작성일: 2026-05-08

## 작업 배경

관리자 페이지의 문화센터 액션 타입 관리 목록에는 `순서` 값이 표시되지만, 액션 타입 추가/수정 모달에서 이 값을 직접 지정할 수 없어 관리자가 노출 순서를 조정하기 어려웠다.

## 작업 관리표

| 상태 | 우선순위 | 항목 | 대상 | 검증 기준 | 비고 |
| --- | --- | --- | --- | --- | --- |
| 완료 | P0 | 액션 타입 모달에 순서 입력 추가 | 관리자 UI | 추가/수정 모달에서 `순서` 숫자 입력 가능 | `0` 이상 정수 |
| 완료 | P0 | 저장 요청에 order 포함 | 프론트 저장 흐름 | 수정 후 목록 순서 값 반영 | 신규 기본값은 마지막 순서 다음 값 |
| 완료 | P0 | 서버 입력 검증 및 정렬 안정화 | API/Service | 허용 필드만 저장, `order ASC, id ASC` 정렬 | DB 변경 없음 |
| 완료 | P1 | 빌드 검증 | 전체 프로젝트 | `npm run build` 성공 | 브라우저 검증은 `!!테스트!!` 요청 시 수행 |

## 구현 방침

- 기존 `action_types.order` 컬럼을 사용하고 DB 마이그레이션은 하지 않는다.
- `ActionTypeModal`에 `order` 필드를 추가해 현재 값 또는 기본값을 표시한다.
- 신규 액션 타입은 현재 목록의 최대 `order + 1`을 기본값으로 사용한다.
- 서버는 `name`, `iconUrl`, `isActive`, `order`만 저장 대상으로 검증한다.
- 같은 `order` 값이 여러 개 있어도 저장은 허용하고, 표시 순서는 `order ASC, id ASC`로 고정한다.

## 검증 계획

- `npm run build` 실행
- 관리자 액션 타입 관리에서 수정 모달의 `순서` 입력 확인
- 수정 가능한 액션 타입의 순서 저장 후 목록 표시 위치 확인
- 신규 액션 타입 추가 시 순서 기본값 확인
