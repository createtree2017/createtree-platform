# [Plan] 갤러리 카테고리 필터 동적 표시 수정

- 작성일: 20260324
- 최종수정일: 0324
- 상태: In Progress

## 1. 목적 및 필요성
- 사용자 갤러리에서 실제 이미지가 있는 카테고리만 필터 버튼으로 노출함으로써, 비어 있는 카테고리가 버튼으로 나타나 완성도가 떨어져 보이는 문제를 해결.
- 현재 개발 중인 메뉴와 같은 내부 카테고리들도 이미지가 있을 때만 노출되도록 개선.

## 2. 현황 분석
- `client/src/components/GalleryEmbedSimple.tsx`에서 `@shared/constants`의 `GALLERY_FILTERS` 배열을 그대로 매핑하여 버튼으로 노출.
- 이미지 개수와 무관하게 모든 필터 버튼이 항상 보임.

## 3. 구현 목표
- 전체 이미지를 기반으로 실제 존재하는 카테고리 목록을 동적으로 계산.
- 해당 목록에 있는 카테고리만 갤러리 상단 필터 버튼으로 렌더링.

## 4. 추진 계획 (Todo 차트)
1. **GalleryEmbedSimple.tsx 분석**
   - 현재 쿼리와 카테고리 버튼 처리 방식 분석 [완료]
2. **동적 필터링 로직 구현**
   - 전체 이미지 데이터(`all`)를 조회하는 쿼리 및 유니크 카테고리 세트 계산 로직 추가.
   - 필터 버튼 영역을 계산된 세트 기반으로 필터링 렌더링하도록 수정.
3. **검증 및 테스트**
   - 이미지 삭제/추가 시 버튼 목록이 즉시 업데이트되는지 확인.

## 5. 기대 성과
- 유저 갤러리가 더 깔끔해지고 필요한 정보만 집중해서 보여줄 수 있음.
- 미완성 신규 기능을 숨긴 상태로 점진적으로 이미지 생성 시에만 오픈하는 효과를 가짐.
