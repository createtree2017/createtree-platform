# 1. 개요
* **목적**: 사용자가 '나의 미션(큰미션)'의 진행률을 100% 달성했을 때 자발적으로 리워드 선물을 신청하고, 관리자가 이 신청 내역을 조회/승인(지급)할 수 있는 전체 워크플로우를 구축합니다.
* **채택된 접근 방식**: 사용자가 명시적으로버튼을 클릭하여 '신청(pending)' 상태로 전환 + 신청 즉시 백엔드에서 관리자 알림 발송 + 관리자 대시보드에서 최종 통제의 하이브리드(권장) 방식입니다.

# 2. 데이터베이스 스키마 반영 내역
**작업 파일: `shared/schema.ts`**
기존 `userBigMissionProgress` 테이블을 확장하여 보상 진행과 관련된 상태값을 관리합니다.

* `rewardStatus`: `varchar (length: 20)`
  * 상태: `not_eligible` (기본값) -> `can_apply` (달성 후) -> `pending` (신청완료) -> `approved` (지급완료)
* `rewardAppliedAt`: `timestamp` (사용자 신청 일시)
* `rewardProcessedAt`: `timestamp` (관리자 처리 일시)
* `rewardContactInfo`: `text` (선택적: 신청 시 연락처나 주소 등을 받을 경우)

# 3. 개발 범위
## 3.1 Backend API (`server/routes/`)
1. **나의 미션 라우터 (`/api/mymissions`)**
   * 사용자 보상 신청 API (`POST /:id/apply-reward`)
     - 진행률 100% 검증.
     - `rewardStatus` 업데이트 및 시간 저장.
     - **알림 트리거 발송** (시스템 푸시, 텔레그램 등 기존 인프라 활용).
2. **관리자 라우터 (`/api/admin/big-missions`)**
   * 보상 목록 조회 API (GET) - 상태가 `pending` 또는 `approved` 인 리스트.
   * 보상 처리 API (`POST /rewards/:progressId/approve`)

## 3.2 Frontend User App (`client/src/pages/my-missions/`)
* **상태별 렌더링 추가**:
  - `rewardStatus === 'can_apply'`: `[선물 신청하기]` 활성 버튼 (애니메이션 효과).
  - `rewardStatus === 'pending'`: `[신청 완료 (관리자 승인 대기중)]` 비활성 버튼.
  - `rewardStatus === 'approved'`: `[선물 수령 완료]` (리본 뱃지 표출 등).
* 클릭 시 확인 다이얼로그 모달 오픈 및 상태 갱신.

## 3.3 Frontend Admin UI (`client/src/pages/admin/`)
* 어드민 패널(`BigMissionsPanel` 등)에 **리워드 선물 신청 관리 탭** 섹션 추가.
* DataTable 형태로 사용자/신청일시/연락처/신청미션내역 렌더링.
* `[승인 및 지급완료]` 버튼을 통한 간편 처리.

# 4. 검증 계획
* Admin 계정에서 로컬 DB 값을 인위적으로 100% 로 만들어 사용자 UI에서 버튼이 잘 나오는지 확인합니다.
* 사용자가 버튼 클릭 시 텔레그램/앱푸시로 관리자 알림이 수신되는지 확인합니다.
* 관리자 페이지에서 신청건을 조회하고, `지급완료`를 눌러 사용자의 앱의 뱃지 UI가 갱신되는지 확인합니다.
