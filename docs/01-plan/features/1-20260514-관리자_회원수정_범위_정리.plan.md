# 관리자 회원수정 범위 정리 Plan

## 작업 개요

관리자 회원관리에서 회원 정보 수정 실패 시 원인이 불명확하게 보이는 문제를 해결한다. 관리자 화면의 수정 가능 범위를 명확히 제한하고, 서버도 같은 정책으로 검증한다.

## 목표

- 관리자 회원 수정 가능 항목을 `회원 등급`, `소속 병원`, `전화번호`로 제한한다.
- `이름`, `회원명`, `이메일`, `생년월일`은 읽기 전용으로 표시한다.
- 서버는 수정 불가 필드가 전달되면 명확한 400 메시지를 반환한다.
- 전화번호는 하이픈 입력을 허용하되 서버 저장값은 숫자만 유지한다.
- 실패 토스트는 서버가 내려준 실제 오류 메시지를 표시한다.

## 설계 메모

- 관리자 화면은 고객 본인 식별 정보까지 편집하는 흐름으로 확장하지 않는다.
- 서버 API는 프론트 실수나 이전 번들 요청까지 방어하기 위해 수정 불가 필드를 명시적으로 거절한다.
- 전화번호는 기존 전화번호 하이픈 표시 개선 정책과 동일하게 UI에서는 하이픈을 보여주고 DB/API 저장값은 숫자만 유지한다.
