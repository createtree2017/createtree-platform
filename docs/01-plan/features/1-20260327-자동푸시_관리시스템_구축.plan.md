# [Plan] 트리거 기반 자동 푸시 관리 시스템 구축

## 1. 개요
* **목적**: 기존 하드코딩된 자동 푸시(미션 승인 등)를 DB 기반의 관리형 자동 푸시(Rule Engine) 시스템으로 개편.
* **배경**: 병원(문화센터)별 커스텀 푸시 지원, 템플릿 내 동적 변수 지원, 특정 이벤트 알림 On/Off 제어 등 운영 확장성 확보.
* **핵심 고려 사항**: 모듈화된 라우터 파일 구조, 상속(Inheritance) 기반 우선순위 매핑 로직, 딥링크와 통계 데이터 관리.

## 2. 아키텍처 및 라우터 모듈화 전략

단일 라우터 비대화를 방지하기 위해 자동 푸시 전용 라우터, 컨트롤러, 서비스를 분리 운영합니다.

### 2.1 Backend 구조
* **Router**: `server/routes/push-automation-routes.ts` (`/api/admin/push-automation`)
* **Controller**: `server/controllers/push/push.automation.controller.ts`

  * Rule CRUD 담당 메서드

## 3. DB 스키마 설계 (`auto_push_rules` 테이블 신설)

`shared/schema.ts`에 추가될 테이블 명세입니다.

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | Serial | PK |
| `eventType` | Varchar | 트리거 종류 (예: `mission_approved`, `mission_rejected`) |
| `hospitalId` | Integer | NULL이면 "공통(All)", 값(ID)이 있으면 "해당 병원 전용" |
| `titleTemplate` | Text | 변수 (예: `{{userName}}`) 포함 푸시 제목 |
| `bodyTemplate` | Text | 변수 포함 푸시 본문 |
| `actionUrlTemplate` | Text | 딥링크 템플릿 (예: `/mymissions/{{submissionId}}`) |
| `category` | Varchar | 알림 카테고리 (수신동의 여부 필터링 용. 예: `mission`, `marketing`) |
| `isActive` | Boolean | 규칙 활성화 여부 기본값 true |
| `priority` | Integer | 우선순위 (기본값 0, 병원 매핑 시 보통 가장 높은 순위) |
| `sentCount` | Integer | 캐싱용 총 발송 건수 (기본 0) |
| `lastSentAt` | Timestamp | 마지막 발송 시간 |
| `createdAt` / `updatedAt` | Timestamp | 메타데이터 |

## 4. 핵심 비즈니스 로직 (Rule Engine)

### 4.1 상속 및 충돌 방지 로직 (Inheritance)
이벤트(`eventType`) 발생 시 백엔드 엔진 수행 순서:
1. 대상 유저의 `hospitalId` 조회.
2. Rule 조회 우선순위:
   - 1순위: `eventType` = 일치 AND `hospitalId` = 유저병원ID AND `isActive` = true
   - 2순위: `eventType` = 일치 AND `hospitalId` IS NULL AND `isActive` = true
3. 매칭되는 규칙 중 가장 `priority`가 높은 규칙 1개 선택.
4. 매칭되는 활성 규칙이 없으면 푸시 발송 스킵 (No-op).

### 4.2 변수 치환 로직 (Interpolation)
* 동적 변수는 정규식 `\{\{\s*(\w+)\s*\}\}` 기반으로 치환합니다.
* 발송 전 입력된 `variables` 맵과 템플릿 텍스트를 파싱합니다.
* 변수 부재 시 Fallback: `{{userName}}` → 데이터 없을 시 `"회원"` 또는 시스템에서 빈칸이 없도록 처리.

## 5. UI/UX 구현 명세 (`PushAutomation.tsx`)

* **대시보드 목록 (Datatable)**: `eventType`, `hospitalId`, `isActive` Toggle, `sentCount` 표시 및 On/Off 토글 버튼 지원.
* **편집 다이얼로그 (생성/수정)**:
  - **이벤트 종류 선택**: 정해진 종류(미션 승인, 미션 반려, 등급 업 등)만 선택 가능 (Select).
  - **대상 제한**: '전체 회원 공통' vs '특정 병원 선택'.
  - **제목/본문 작성 폼**: 자주 쓰는 치환 변수 `Chips` (클릭 시 커서 위치에 추가).
  - **테스트 발송 (미리보기)**: 저장 전 관리자 폰으로 더미 데이터 치환 푸시 테스트 발송 기능 탑재.
