# 앱 푸시 관리자 기능 확장 설계서

**작성일**: 2026-03-27
**상태**: 설계 검토 대기

---

## 1. 목적

현재 앱 푸시 관리자 UI에 누락된 4가지 핵심 기능을 구현합니다:
1. 병원별(거래처) 전체 회원 푸시 발송
2. 회원 검색/목록/다중선택 기능
3. 이미지 URL 리치 푸시 실제 동작
4. 동적 알림 템플릿 관리 시스템

---

## 2. 현재 상태 분석

| 항목 | 파일 | 줄 수 | 비고 |
|------|------|:-----:|------|

| 백엔드 토큰 관리 | `push-routes.ts` | 50줄 | ✅ 독립 파일 |

### 라우트 등록 방식
- `admin-routes.ts`는 `registerAdminRoutes(app)` 함수로 Express app에 직접 등록 (Router 사용 안 함)
- `push-routes.ts`는 `Router()` 인스턴스로 export → `routes.ts`에서 `app.use('/', pushRoutes)` 등록

## 3. Phase별 설계

### Phase 1: 인프라 (라우트 분리 + 리치 푸시)

#### [NEW] `server/routes/push-admin-routes.ts`

`admin-routes.ts`에서 **푸시 관련 코드(L3617~3713)를 복사(잘라내기 금지!)** → 새 파일에 옮긴 후 → 원본에서 삭제.

```
// 새 파일 구조 (Router 방식 사용)
import { Router } from 'express';
const router = Router();

router.get("/push-logs", ...)          // 기존 이동
router.post("/push-send", ...)         // 기존 이동 + hospital 분기 추가
router.get("/push-users", ...)         // Phase 2 새 API
router.get("/push-templates", ...)     // Phase 3 새 API
router.post("/push-templates", ...)    // Phase 3 새 API
router.patch("/push-templates/:id", ...)
router.delete("/push-templates/:id", ...)

export default router;
```

**등록**: `routes.ts`에 `app.use('/api/admin', pushAdminRouter)` 추가

> [!CAUTION]
> API 경로가 변경되지 않도록 주의! 기존: `app.post("/api/admin/push-send")` → Router에서는 `router.post("/push-send")` + 마운트 prefix `/api/admin`

#### [MODIFY] `server/services/push/push.service.ts`

`sendBatch()` 함수에서 FCM `notification.image` 필드 추가:

```typescript
// 기존: notification에 title, body만
const message = {
  tokens: chunk,
  notification: payload.notification,  // { title, body } 만 있음
  data: payload.data,
};

// 변경: image 필드 포함
const message = {
  tokens: chunk,
  notification: {
    ...payload.notification,
    ...(payload.data?.image_url ? { image: payload.data.image_url } : {}),
  },
  data: payload.data,
};
```

## 6. 검증 계획

### 자동 검증 (터미널)

1. **서버 시작 확인**
   ```
   npm run dev
   ```
   → 에러 없이 `PORT=5000`으로 기동 확인

2. **API 응답 확인** (curl)
   ```
   # Phase 1 검증
   curl -s localhost:5000/api/admin/push-logs | head -c 200
   
   # Phase 2 검증
   curl -s "localhost:5000/api/admin/push-users?search=test"
   
   # Phase 3 검증
   curl -s localhost:5000/api/admin/push-templates
   ```
   → JSON 응답 확인 (HTML 반환 시 라우트 누락)

3. **DB 스키마 반영**
   ```
   npx drizzle-kit push --force
   ```

### 수동 테스트 (사용자 요청)

```text
👤 수동 테스트 요청:
1. localhost:5000/api/dev/auto-login 접속 (자동 로그인)
2. /admin?tab=push-management 이동
3. "발송 내역" 탭 → 기존 로그 표시 확인
4. "수동 알림 발송" 탭 → 발송 대상 드롭다운에 3개 옵션 확인
   - 전체 앱 사용자 / 특정 사용자(ID) / 지정 병원(거래처)
5. "특정 사용자(ID)" 선택 → "회원 선택" 버튼 → 모달 열림 확인
6. "알림 템플릿" 탭 → 템플릿 추가/수정/삭제 확인
```
