# 관리자페이지 이미지 갤러리 1K 원본 보기 모달 설계서 (Design)

**작성일**: 2026-03-20
**작업 요약**: `ImageGalleryAdmin` 컴포넌트 내 Dialog UI 구조 설계

---

## 1. 개요
현재 `handleViewImage` 함수와 이에 해당하는 React 상태(`viewImageDialog`, `selectedImage`)가 준비되어 있으나 JSX 부분에서 `Dialog` 컴포넌트의 렌더링 코드가 누락되어 있습니다. 이를 보강하여 모달을 구현합니다.

## 2. 모달(Dialog) UI 아키텍쳐
```tsx
<Dialog open={viewImageDialog} onOpenChange={setViewImageDialog}>
  <DialogContent className="max-w-4xl max-h-[90vh] overflow-y-auto">
    <DialogHeader>
      <DialogTitle>{selectedImage?.title}</DialogTitle>
      <DialogDescription>
        생성일: {new Date(selectedImage?.createdAt || '').toLocaleString()}
      </DialogDescription>
    </DialogHeader>
    <div className="...">
      {/* 고해상도(1024x1024) 이미지 출력 */}
      <img src={selectedImage?.transformedUrl || selectedImage?.originalUrl || selectedImage?.url} />
    </div>
    <div className="flex gap-2 justify-end">
      {/* 기존 구현된 함수들 연결 */}
      <Button onClick={() => handleDownloadClick(selectedImage!)}>다운로드</Button>
      <Button onClick={() => handleShare(selectedImage!)}>공유</Button>
    </div>
  </DialogContent>
</Dialog>
```

## 3. 이미지 리소스 우선순위 정책
리스트에서는 `thumbnailUrl`(85% 품질)을 우선 렌더링하여 트래픽을 최적화하고, 상세 모달에서는 `transformedUrl`이나 `originalUrl`(90% 이상 품질)을 노출하여 1K 해상도를 확실히 유지합니다.
