# 콜라주 UI 모바일 개선 설계서

## 개요
- **기능명**: 콜라주_UI_모바일_개선
- **작성일**: 2026-03-16
- **분류**: Major Feature (3개 파일 전면 개편 + 신규 탭/업로드 UI)
- **배경**: 현재 UI가 PC 중심 좌우 분할 구조로, 모바일에서 과도한 스크롤 없이는 사용이 어려움

## 컴포넌트 설계

### 1. LayoutPicker.tsx — 수정

**변경 전**: 각 분할이 `w-full` 큰 카드 (아이콘 + 설명 + 카운터)로 세로 4개 나열  
**변경 후**: `flex` 가로 탭 한 줄

```tsx
// 신규 UI 구조
<div className="flex gap-2">
  {layouts.map(layout => (
    <button className="flex-1 flex flex-col items-center gap-1 py-3 rounded-xl border">
      {/* 분할 그리드 미니 프리뷰 */}
      <div className="w-8 h-8">
        {layout.preview}
      </div>
      {/* 텍스트 */}
      <span className="text-xs font-semibold">{layout.label}</span>
    </button>
  ))}
</div>
```

- 선택된 탭: 보라색 배경 + 흰색 텍스트
- 미선택: 어두운 배경 + 회색 텍스트
- 프리뷰 아이콘: 미니 격자 도형 (SVG or div grid)

---

### 2. ImageSelector.tsx — 수정

**변경 전**: 갤러리 이미지 그리드만 존재  
**변경 후**: 상단 탭(디바이스/갤러리) + 컨텐츠 영역

#### 내부 상태 추가
```tsx
const [uploadTab, setUploadTab] = useState<'device' | 'gallery'>('gallery');
const [deviceImages, setDeviceImages] = useState<DeviceImage[]>([]);

interface DeviceImage {
  id: string;       // UUID (로컬 전용)
  file: File;
  previewUrl: string; // URL.createObjectURL(file)
}
```

#### 탭 구조
```
[📷 디바이스 첨부] [🖼️ 갤러리 선택]
```

#### 디바이스 첨부 탭
- `<input type="file" accept="image/*" multiple />` 숨김 처리
- "+" 버튼 클릭 시 파일 선택 다이얼로그 오픈
- 선택한 이미지: `URL.createObjectURL()`로 미리보기
- **서버 업로드 없음** — 브라우저 임시 처리 (콜라주 생성 시 FormData로 전송)
- 구현 방식: `File` 객체를 상태로 보관 → 생성 API 호출 시 multipart/form-data

#### 갤러리 선택 탭
- 현재 구현 유지 (`/api/gallery` 데이터)

#### 이미지 선택 공통 상태
- 현재: `selectedImages: number[]` (갤러리 ID 배열)
- 변경: `selectedImages: SelectedImage[]` — 갤러리/디바이스 통합 타입

```tsx
type SelectedImage =
  | { type: 'gallery'; id: number }
  | { type: 'device'; localId: string; file: File; previewUrl: string };
```

#### 카운터 + 선택 그리드
- 상단: `선택된 이미지: 1/2개` 표시
- 3열 그리드 (`grid-cols-3`)
- 각 셀: 이미지 + 삭제 버튼 (X)

---

### 3. gallery-collage.tsx — 수정

**변경 전**: `grid grid-cols-1 lg:grid-cols-3` (PC 좌우 분할)  
**변경 후**: 단일 컬럼 세로 플로우

```tsx
// 신규 레이아웃 구조
<div className="min-h-[var(--dvh)] flex flex-col">
  {/* 헤더 - 컴팩트 */}
  <div className="text-center py-6 px-4">
    <h1>✦ 콜라주 만들기</h1>
    <p>...설명...</p>
  </div>

  {/* 레이아웃 선택 */}
  <section className="px-4 mb-4">
    <CollageLayoutPicker ... />
  </section>

  {/* 이미지 선택 — 분할 선택 시 표시 */}
  {selectedLayout && (
    <section className="px-4 flex-1">
      <CollageImageSelector ... />
    </section>
  )}

  {/* 하단 생성 버튼 (sticky) */}
  <div className="sticky bottom-0 p-4 bg-gradient-to-t from-background">
    <Button className="w-full" onClick={handleCreateCollage}>
      ↓ 콜라주 생성하기
    </Button>
  </div>
</div>
```

#### 상태 타입 변경
- `selectedImages: number[]` → `selectedImages: SelectedImage[]`
- 콜라주 생성 시 갤러리 ID + 디바이스 파일 모두 처리

#### API 호출 변경 (handleCreateCollage)
```ts
// 갤러리 이미지만 있는 경우: 기존 JSON 방식 유지
// 디바이스 이미지 포함 시: FormData 방식으로 변경
const formData = new FormData();
formData.append('layout', selectedLayout);
formData.append('resolution', resolution);
selectedImages.forEach((img, idx) => {
  if (img.type === 'gallery') formData.append(`imageIds[${idx}]`, String(img.id));
  if (img.type === 'device') formData.append(`files`, img.file);
});
```

> **주의**: 백엔드(`/api/collage/create`) 수정이 필요할 수 있음 — 현재 JSON만 받음  
> → 1차 구현은 **갤러리 탭 유지 + 디바이스 탭 UI만 구현**, 백엔드 연동은 2차에서 처리

## 디자인 방향

- **색상**: 기존 다크 테마 유지 (보라색 #7c3aed 계열 포인트)
- **폰트**: 기존 폰트 유지
- **탭 스타일**: 둥근 모서리(`rounded-xl`), 선택 시 보라색 채움
- **이미지 그리드**: `grid-cols-3`, 정사각형(`aspect-square`), 라운드 모서리
- **생성 버튼**: 그라디언트(`from-purple-600 to-indigo-600`), 하단 sticky

## API 설계

### 기존 엔드포인트 유지
| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/gallery` | 갤러리 이미지 목록 (변경 없음) |
| POST | `/api/collage/create` | 콜라주 생성 (1차: 갤러리 ID만) |

### 1차 구현 제한 (디바이스 업로드 백엔드 연동 제외)
- 디바이스 첨부 탭 UI는 구현
- 생성 버튼 클릭 시 디바이스 이미지 포함된 경우 안내 토스트 표시
- 실제 업로드 API 연동은 2차 작업으로 분리

## 구현 순서

1. `LayoutPicker.tsx` — 가로 탭 UI로 전환 (약 1시간)
2. `gallery-collage.tsx` — 모바일 단일 컬럼 레이아웃 전환 + 상태 타입 변경 (약 1시간)
3. `ImageSelector.tsx` — 디바이스/갤러리 탭 추가 + 통합 선택 타입 적용 (약 2시간)
