# 신청 UI 콘텐츠 영역 이동 & 탭슬롯 확보

**작업일**: 2026-03-25  
**작업자**: Antigravity  
**작업 유형**: Feature (UI 리팩토링)

---

## 변경 배경 / 의사결정 이유

세부미션 탭바(1열)에 신청 탭이 포함되어 있어 실질적으로 사용 가능한 세부미션 슬롯이 최대 3개로 제한되는 문제가 있었다.

**핵심 의사결정**:
- 신청은 세부미션이 아님 (참여 자격 관리 시스템). 탭바에 있을 이유 없음.
- 콘텐츠 영역(행사정보 박스 아래)으로 이동 → 탭슬롯 1개 확보 (최대 4→5개 가능)
- 백엔드 변경 없음 (UI-only 1단계). 관리자 승인/엑셀 기능 영향 없음.

## 주의사항 / 알려진 이슈

- **2단계 미진행**: 주제미션 레벨로 신청 시스템 완전 이관은 DB 마이그레이션 필요 → 별도 PDCA
- **ApplicationButton 컴포넌트 잔존**: 현재는 사용되지 않음. 추후 정리 가능.
- **참여취소 → SubmissionForm 모달 오픈**: 기존 신청 취소 로직 그대로 재사용 (handleTabClick)

## [추가 작업] 모달 미닫힘 버그 수정

**증상**: 제출/신청 완료 후 토스트는 나오는데 모달이 닫히지 않음

**원인**: Stale Closure 문제
- `FormWrapper`가 `useEffect` 내부에서 정의됨
- `useEffect` 실행 시점(modal stack = 0)의 `modal.closeTopModal`이 캡처됨
- 이 stale한 함수는 `modalStack.length === 0` 조건으로 항상 조기 return

**수정**: `SubmissionForm`, `AttendancePasswordForm`, `cancelApplicationMutation` 각각의 컴포넌트에서 `useModalContext()`를 직접 사용하여 항상 fresh한 `closeTopModal` 호출

```diff
// submitMutation.onSuccess
- await invalidateQueries...
- onSuccess();
+ onSuccess();           // 부모 상태 정리 먼저
+ modal.closeTopModal(); // fresh context로 직접 닫음
+ await invalidateQueries...
```

- ✅ 제출 후 모달 자동 닫힘 확인
