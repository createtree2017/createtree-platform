# 20260326-4-FCM앱푸시검증및버그수정.인수인계.md

## 1. 개요
* **목적:** 신규 구축된 FCM 앱 푸시 시스템에 대한 10년차 시니어 레벨의 코드 정밀 검수 중 발견된 미션 리뷰 트리거 에러 수정
* **작업 파일:**
  * `server/services/mission/admin.mission.review.service.ts`

## 2. 변경 내용
* **미션 승인/반려 알림 객체 파라미터화**:
  기존 `createNotification()` 호출 부가 레거시 시그니처 (개별 파라미터 5개) 형태로 작성되어 있어 신규 객체 폼 기반 FCM 훅과 호환되지 않았습니다. 이를 `{ userId, type, title, message, ... }` 객체 스펙으로 재작성했습니다.
* **일괄 상태 변경(Bulk Update) 부분 푸시 연동**:
  여러 건의 미션을 동시 다발적으로 상태 변경(`bulkUpdateStatus`)할 때에도 정상적으로 푸시 알림이 발송되도록 인자 타입 및 트리거를 교정했습니다.

## 3. 왜 변경했는지
* 신규 알림 시스템은 `notifications.ts`의 `createNotification({ ... })` 단일 파라미터 객체 형태로 설계되었으나, 리뷰 서비스 쪽에 구 스펙으로 연결되어 있어 서버 런타임 에러(500)가 확실시되는 상황이었기 때문에, 앱 서비스 운영의 안정성을 보장하기 위해 반드시 수정되어야 했습니다.

## 4. 알려진 이슈 및 주의사항
* 현재 모든 FCM 전송 기반 코드가 TypeScript 상 컴파일 에러 없이 완벽히 매칭됨을 확인했습니다. (`npx tsc --noEmit` 통과 - GUI 관련 구형 폼 에러 제외). 실제 발송 라이브 테스트는 `!!테스트!!` 명령어 진행 시 실기기로 이루어지면 됩니다.

## 5. 실행 결과
* ✅ `server/services/mission/admin.mission.review.service.ts` 백엔드 오류 해결 완료
* ✅ 토큰 사이클(2-Phase 로직), `App.tsx` 모듈 임포트 연동 100% 매치 검증
