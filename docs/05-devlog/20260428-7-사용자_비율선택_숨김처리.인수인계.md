- **작업 목적**: 모델 정리 개발 과정에서 타 개발자가 비율 선택 UI를 사용자 화면에 노출하는 실수를 범함. 기획 및 정책상 해상도/비율 설정은 관리자만 제어하고 사용자는 신경 쓰지 않도록 해당 영역을 `hidden` (기본값 비활성) 처리함.

## 3. 핵심 변경 내용
- **`showAspectRatioSelector` 속성 기본값 변경**: 
  - 기존: `showAspectRatioSelector = true`
  - 변경: `showAspectRatioSelector = false`
- **로직 설명**: 
  - UI 렌더링이 되지 않더라도 컴포넌트 내부의 상태 업데이트 로직(`useEffect` 내 `effectiveRatios[0]`)을 통해, 관리자가 해당 스타일(Concept)에 지정해둔 사용 가능한 비율 중 첫 번째 기본 비율이 폼 데이터(`aspectRatio` 변수)로 자동 세팅되어 서버에 무사히 전달됩니다. 
  - 즉, 이미지 생성 백엔드 기능은 기존과 완벽하게 동일하게 작동하며, 눈에 보이는 선택 창만 사라집니다.

## 4. 알려진 이슈 및 주의사항
- 관리자 패널이나 향후 비율 선택을 명시적으로 보여주어야 할 페이지가 생긴다면, 부모 컴포넌트에서 `<ImageGenerationTemplate showAspectRatioSelector={true} />` 처럼 명시적으로 Props를 전달하면 언제든 다시 표시할 수 있도록 유연성을 남겨 두었습니다.
