# 20260331-1-갤러리_콜라주_전역버그수정

## 📋 작업 요약
갤러리에서 콜라주 이미지가 전체 사용자에게 보이지 않는 **전역 버그** 수정

## 🔴 변경 배경 (왜 변경했는지)
- SQL에서 `NULL != 'value'`는 `NULL`(=false)로 처리됨
- 콜라주 이미지 373개의 `category_id`가 모두 **NULL**이었음
- 갤러리 API의 '전체' 탭: `ne(images.categoryId, 'firebase_upload')` 조건이 category_id=NULL인 행을 **전부 제외**함
- 결과: **모든 사용자**의 갤러리에서 콜라주 이미지가 보이지 않고, 필터 버튼도 표시되지 않았음

## 📁 변경 파일

### 1. `server/routes/gallery.ts` (핵심 수정)
- **변경**: `ne(images.categoryId, EXCLUDED_CATEGORY)` → `or(isNull(images.categoryId), ne(images.categoryId, EXCLUDED_CATEGORY))`
- **import**: `or`, `isNull` 추가
- **효과**: category_id가 NULL인 이미지(콜라주)가 '전체' 탭에서 정상 표시

### 2. `server/services/collageServiceV2.ts` (예방 수정)
- **변경**: DB insert 시 `categoryId: 'collage'` 추가 (449줄)
- **효과**: 향후 생성되는 콜라주가 category_id=NULL이 되지 않음

### 3. DB 데이터 수정 (스크립트 실행 완료 후 삭제)
- 기존 373개 콜라주: `category_id = NULL` → `category_id = 'collage'` 업데이트 완료

## ✅ 동작 확인
- ✅ 갤러리 전체 탭에서 콜라주 이미지 표시됨
- ✅ 갤러리 '콜라주' 필터 버튼 표시됨
- ✅ 콜라주 필터 클릭 시 콜라주만 표시됨
- ✅ 미션 제출 정상 동작 (사용자 직접 확인)

## ⚠️ 주의사항
- **프로덕션 DB에서도 동일 스크립트 실행 필요**: `UPDATE images SET category_id = 'collage' WHERE style = 'collage' AND category_id IS NULL`
- Railway 배포 후 위 SQL을 Railway DB에서도 실행해야 프로덕션에서 콜라주가 보임
