# 20260323-3-상세페이지_업로드_용량_상향.인수인계.md

## 1. 작업 개요
- **목적**: 문화센터 주제 미션의 상세페이지 이미지 업로드 용량을 10MB로 상향하고, 가로 1024px 제한 + 세로 무제한 리사이징을 적용하여 화질 개선.
- **작업 타입**: Minor Change + 버그 수정

## 2. 변경 파일 목록 및 내용

### 백엔드 (Backend)
- [mission-routes.ts](file:///c:/Users/TOP/Desktop/createAI/createAI_v1_home/server/routes/mission-routes.ts)
  - `missionHeaderUpload` 미들웨어의 `maxFileSize`를 5MB에서 10MB로 상향 조정.
- [gcs-image-storage.ts](file:///c:/Users/TOP/Desktop/createAI/createAI_v1_home/server/utils/gcs-image-storage.ts)
  - `saveImageToGCS` 함수에 커스텀 리사이징 옵션(`GCSImageOptions`: `maxWidth`, `maxHeight`, `quality`, `fit`)을 지원하도록 확장.
  - **[버그 수정]** 세로 무제한 리사이징 로직 수정: `maxHeight: null` 전달 시 `sharp.resize(width, undefined)`로 호출하여 진짜 세로 무제한을 구현.
    - 기존 문제: `sharp.resize(1024, null, { fit: 'inside' })`는 세로도 1024 기준 박스 안에 맞추려 해 세로 10400 이미지가 약 2048px로 손상됨.
    - 수정 후: `sharp.resize(1024, undefined, { fit: 'inside' })`로 가로만 기준 잡아 세로는 원본 비율 그대로 유지됨.
- [mission.theme.service.ts](file:///c:/Users/TOP/Desktop/createAI/createAI_v1_home/server/services/mission/mission.theme.service.ts)
  - 상세페이지 이미지 업로드 시 `{ maxWidth: 1024, maxHeight: null }` 옵션 적용.
  - **결과**: 가로 1024px 초과 시에만 가로를 1024px로 리사이징, 세로는 제한 없이 원본 유지.

### 프론트엔드 (Frontend)
- [MissionManagement.tsx](file:///c:/Users/TOP/Desktop/createAI/createAI_v1_home/client/src/components/admin/MissionManagement.tsx)
  - 상세페이지 이미지 업로드 영역에 "최대 10M까지 업로드 가능, 가로 1024 이미지 권장" 안내 문구 추가.
  - `FormDescription` 컴포넌트 사용.

## 4. 다음 작업 참고사항
- 세로 길이가 극단적으로 긴 이미지(예: 1024x50000)는 GCS에 원본 그대로 저장되므로, 사용자 페이지에서 로딩 시간이 길어질 수 있음. 필요 시 세로 상한을 별도로 지정 가능.
- `saveImageToGCS`의 `GCSImageOptions`는 다른 도메인 업로드에서도 재사용 가능 (범용 설계).
