# 개발 인수인계: [회원] 가입 시 푸시 자동화 이벤트 추가 

**작업일자:** 2026-04-08 (순번 2)
**제목:** 회원가입 시 푸시 발송 트리거 자동화 추가

## 1. 📂 변경된 파일 목록
- \`client/src/components/admin/PushAutomation.tsx\` (수정)
- \`server/routes/auth.ts\` (수정)

## 2. 📝 핵심 변경 내용
- **관리자 페이지 앱 푸시 자동화 설정 (PushAutomation.tsx):** 
  - 새 자동화 규칙 추가 시 트리거 이벤트 리스트(\`EVENT_TYPES\`)에 \`user_register\` ("[회원] 가입 시") 옵션을 배열 가장 끝에 추가했습니다.
- **회원가입 API (auth.ts):**
  - 회원가입(\`/auth/register\`) 요청 처리 완료 직후, \`pushAutomationService.evaluateAndSend()\` 함수를 호출하여 \`user_register\` 이벤트가 발동하도록 로직을 구현했습니다.
  - 가용 변수로 가입한 사용자의 이름(\`userName\`)을 함께 전송하여, 추후 알림 제목/본문에 "OOO님 환영합니다"라는 메시지 치환이 되도록 구현했습니다.

## 3. 🤔 핵심 의사결정 배경 (원인 및 이유)
- **앱 푸시 타겟팅 확장 목적:** 현재 [미션 승인/반려/상태 변경] 세 단계에서만 발송되던 자동화 룰을 다각화하여, 누군가 앱에 신규 가입했을 때 안내사항 등을 자동으로 푸시 전송하려는 운영상의 니즈를 반영했습니다.
- **알림함 저장 방식(A안 채택 이유):** 사용자가 회원 가입을 성공하는 정확히 그 찰나의 시점엔 디바이스의 고유 FCM 토큰이 백엔드 DB에 완전히 동기화되어 있지 않을 가능성이 높습니다. (보통 클라이언트에서 로그인 이후에 FCM 토큰을 얻어 \`/api/push/register-token\`을 쏘기 때문) 
  - 그러므로 실제 기기엔 알람이 울리지 않을 수 있으나, 앱 내부의 알림함(\`/api/notifications\`) 서버 기록에는 온전히 남아 있으므로 신규 유저가 진입 후 곧바로 환영 알림을 확인할 수 있는 형태의 기획을 수용했습니다.

## 5. 🚨 주의사항 및 알려진 이슈
- **FCM 토큰 동기화 타이밍 딜레이 이슈:** 위 의사결정 배경에 기술한 바와 같이, 당장 회원가입하는 스텝에서는 디바이스 토큰이 100% 매핑되기 전이라 실제 스마트폰 상단 팝업 푸시는 수신되지 않을 확률이 큽니다. 팝업 푸시까지 보장하려면 백엔드 엔진 차원에서 메세지 큐(Delay)를 적용하거나 프론트에서 가입 튜토리얼 이후 강제로 지연 발송하는 별도의 이벤트 처리가 필요합나다. 현재는 앱 내부 '알림함 보관' 목적으로 사용하시는 것을 권장합니다.
- **연관 파일:** 이 기능을 기반으로 차후 '\`user_login\`', '\`level_up\`' 등의 다분화된 이벤트를 자유롭게 `auth.ts` 혹은 비즈니스 로직에 심을 수 있습니다.
