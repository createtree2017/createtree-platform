# 20260428-2-GPT_Image_2_모델_추가.인수인계

## 1. 주요 변경 내용 요약
사용자의 요청에 따라 OpenAI의 새로운 모델인 `gpt-image-2` 및 `gpt-image-2-mini`를 시스템에 연동하기 위한 백엔드/프론트엔드/데이터베이스 마이그레이션 작업을 완료했습니다. 이 작업은 기존 서비스 구조(DALL-E 3)를 유지하면서 신규 모델을 병렬로 지원하기 위한 구조적 확장을 포함합니다.

## 2. 변경된 파일 목록
- `shared/schema.ts` : `AI_MODELS` 상수에 신규 모델 추가 및 Zod 검증 로직 반영
- `server/services/openai.ts` : `gpt-image-2` 계열 전용 서비스 생성 함수(`generateImageWithGPT2`, `generateImageWithGPT2Mini`) 신규 작성
- `server/routes/image.ts` : 생성 및 변환 API에서 `openai_gpt2` 계열일 경우 올바른 모델명 파라미터(`gpt-image-2`, `gpt-image-2-mini`)를 분기 처리
- `server/utils/settings.ts` : 컴파일 에러 발생 부분 수정 (DB에서 `SystemSettings` 타입 추출 및 폐기 모델 제거 로직 타입 캐스팅 고도화)
- `client/src/hooks/useModelCapabilities.ts` : 프론트엔드에서 신규 모델 지원 가능 해상도(Aspect Ratios) 기본값 정의
- `client/src/hooks/useSystemSettings.ts` : fallback 모델 배열에 명시적 타입 지정하여 컴파일 에러 수정
- `client/src/components/ImageGenerationTemplate.tsx` : UI에 `GPT-Image-2` 및 `GPT-Image-2 Mini` 버튼 노출
- `client/src/components/modal/admin/ConceptFormModal.tsx` : 관리자 페이지 컨셉 생성/수정 모달에 신규 모델 체크박스 추가
- `server/utils/topmedia-retry.ts` : fetch timeout 옵션을 AbortSignal.timeout(5000)으로 대체하여 TypeScript 에러 수정
- `scripts/test-openai-gpt2.ts` (신규) : OpenAI API를 직접 호출하기 위한 내부 테스트 스크립트 작성
- `scripts/test-openai-gpt2-json.ts` (신규) : OpenAI API JSON Payload 테스트 스크립트 작성

## 3. 왜 변경했는지 (의사결정 배경)
- 사용자가 "ChatGPT Images 2.0 (API 모델명: gpt-image-2) 모델을 도입하라"고 요청하였으며, "기존 모델을 유지하면서 신규 모델을 추가한 다음, 검증 후 제거하는 병행 전략"을 지시했습니다.
- 따라서 기존의 `openai-dalle3.ts`는 그대로 유지하고, 서비스 확장과 UI 노출까지만 진행하여 사용자가 직접 테스트해볼 수 있도록 조치했습니다.
- 시스템 전반에 걸친 TypeScript의 타입 안정성을 확보하기 위해, 발생한 컴파일 에러(`settings.ts`, `useSystemSettings.ts`)를 함께 수정했습니다.

## 4. 동작 확인 상태
- `npx tsc --noEmit` : ✅ 수정 관련 파일 타입 에러 모두 제거 완료
- 시스템(DB/UI) 연동 여부 : ✅ 컴파일 및 UI 추가 완료
- **API 연동 상태** : ❌ (OpenAI 응답 오류 확인)
  - `gpt-image-2-mini` 모델로 OpenAI 서버에 JSON Payload 요청을 보냈으나, `The model 'gpt-image-2-mini' does not exist.` 라는 400 Bad Request 응답이 반환되었습니다.
  - OpenAI 공식 API에는 해당 모델명이 등록되지 않았거나 계정에 접근 권한이 없는 상태입니다. (사용자에게 보고 및 직접 테스트 요청)

## 5. 알려진 이슈 / 주의사항
- 관리자 페이지에서 `GPT-Image-2`를 선택하고 앱에서 이미지를 생성하면 **현재는 OpenAI API에서 에러(모델 없음)**가 발생합니다.
- `gpt-image-2` 모델이 실제로 유효한지 계정 상태나 OpenAI 문서 확인이 필요합니다.

## 6. 다음 개발자를 위한 참고사항
- 만약 해당 모델명이 틀린 것이라면, 사용자 피드백을 받은 후 `shared/schema.ts`의 `AI_MODELS`에서 해당 값을 롤백해야 할 수 있습니다.
- 이미지 생성 라우트(`server/routes/image.ts`) 내 분기문은 `gpt-image-2-mini`와 `gpt-image-2`를 문자열로 주입하도록 개발되었습니다. 모델명이 확정되면 이 부분의 `openaiModelName`을 수정하면 됩니다.
