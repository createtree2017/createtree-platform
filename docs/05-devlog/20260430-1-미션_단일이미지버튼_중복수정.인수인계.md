# 미션 단일 이미지 버튼 중복 수정 인수인계

작성일: 2026-04-30

## 1. 배경

세부미션 제출 타입이 `image` 1개일 때 이미지 선택 버튼이 보이지 않는 문제를 이전 작업에서 복구했으나, `image`가 2개 이상일 때는 기존 제출 항목 목록의 `이미지1`, `이미지2`, `이미지3` 버튼 아래에 별도 이미지 버튼이 한 번 더 표시되는 문제가 발생했다.

## 2. 원인

`client/src/pages/mission-detail.tsx`에서 제출 항목 목록은 기존에 `availableTypes.length > 1`일 때만 렌더링되었다.

이 때문에 단일 이미지 제출에서는 목록 자체가 숨겨졌고, 이를 보완하기 위해 이미지 제출 영역 내부에 별도 이미지 선택 버튼이 추가되어 있었다.

하지만 현재 UI 정책은 `이미지1`, `이미지2`, `이미지3` 제출 항목 버튼 자체가 이미지 첨부 진입 버튼 역할을 하는 구조다. 따라서 다중 이미지에서는 기존 목록 버튼과 별도 이미지 버튼이 동시에 렌더링되어 중복이 발생했다.

## 3. 변경 내용

### `client/src/pages/mission-detail.tsx`

- `shouldShowSubmissionTypeSelector` 조건을 추가했다.
  - 제출 타입이 2개 이상이면 기존처럼 제출 항목 목록을 표시한다.
  - 제출 타입이 1개여도 선택 타입이 `image`이면 제출 항목 목록을 표시한다.
- 이미지 제출 영역 내부에 있던 별도 이미지 선택 버튼을 제거했다.
- 결과적으로 이미지 제출 UI는 개수와 관계없이 동일한 제출 항목 목록 버튼을 사용한다.
  - 이미지 1개: `이미지1` 버튼 1개
  - 이미지 2개 이상: `이미지1`, `이미지2`, `이미지3` 버튼
  - `제출하기` 위쪽의 중복 이미지 버튼 없음
- 이미지 첨부 버튼과 제출 버튼의 색상이 혼동되지 않도록 `제출하기` 버튼 색상을 보라색에서 초록색(`emerald`) 계열로 변경했다.
- 이미지 제출 항목 버튼은 더 이상 선택 상태를 보라색으로 표시하지 않도록 변경했다.
  - 기존 `selectedTypeIndex` 기반 보라색 강조는 이미지 항목에 적용하지 않는다.
  - 미제출 이미지 버튼은 모두 흰색으로 표시된다.
  - 제출 완료된 이미지 항목은 완료 상태 표시만 유지한다.
- 링크 URL 입력창과 텍스트/리뷰 작성 영역을 흰색 박스와 검정 글자 스타일로 통일했다.
  - placeholder는 회색으로 표시한다.
  - 포커스 ring은 기존 보라색 계열을 유지하되 입력 영역 자체는 흰색으로 고정한다.

- `git diff --check -- client/src/pages/mission-detail.tsx`
  - 이미지 버튼 선택 강조 제거 후 재확인 통과
- `git diff --check -- client/src/pages/mission-detail.tsx`

  - 실패

  - 이번 변경 파일인 `client/src/pages/mission-detail.tsx` 관련 신규 타입 오류는 출력되지 않았다.
- `npm.cmd run build`
  - 최초 실행은 Vite/esbuild spawn이 샌드박스 `EPERM`으로 실패했다.
  - 승인된 샌드박스 외부 실행에서 빌드 통과.
  - 기존 Vite dynamic import/chunk size 경고만 표시됨.

## 5. 주의사항

- 이번 변경은 프론트엔드 제출 모달 렌더링 조건만 수정했다.
- 서버 API, DB 스키마, 제출 데이터 저장 구조는 변경하지 않았다.
- 요청에 `!!테스트!!`가 포함되지 않아 브라우저 직접 화면 검증은 수행하지 않았다.
- `!!푸시!!` 요청이 없어 `git add`, `git commit`, `git push`는 수행하지 않았다.
