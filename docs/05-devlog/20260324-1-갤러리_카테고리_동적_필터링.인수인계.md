## 📅 작성일: 2026-03-24

### 📝 변경 내용
- **대상 파일**: `client/src/components/GalleryEmbedSimple.tsx`
- **핵심 변경 내역**:
    - 갤러리 진입 시 `all` 필터로 모든 이미지를 가져오는 보조 쿼리(`allImagesQuery`) 추가.
    - 가져온 전체 이미지의 `type` 필드를 분석하여 이미지가 1개 이상 존재하는 카테고리 세트를 동적으로 계산 (`activeCategoryKeys`).
    - 필터 버튼 렌더링 시 이 목록에 없는 카테고리는 버튼으로 노출하지 않도록 수정.
    - 이미지 생성(`imageCreated`) 및 삭제 시 해당 쿼리를 무효화하여 필터 목록 자동 업데이트 보장.

### ❓ 왜 변경했는가 (배경 및 의사결정)
- 사용자가 아직 이미지를 생성하지 않은 카테고리가 버튼으로 계속 노출되는 것이 비효율적이고 완성도가 떨어져 보인다는 사용자 요청 반영.
- '현재 개발 중인 메뉴'가 카테고리에 상수로 포함되어 있어 나타나는 상황에서, 실제 이미지가 발생할 때만 버튼을 노출해 자연스러운 유저 경험을 제공하려 함.

### ✅ 동작 확인 상태
- [x] 이미지 갤러리 진입 시 이미지가 있는 카테고리만 필터 버튼 노출 확인.
- [x] '전체' 버튼은 여전히 항상 노출됨 확인.
- [x] 이미지 삭제 시 해당 카테고리가 비면 필터 버튼이 즉시 사라짐 확인.
- [x] 이미지 생성 완료 시 새 카테고리 버튼이 즉시 나타남 확인.

### ⚠️ 주의사항 / 알려진 이슈
- **성능**: 이미지 개수가 매우 많아질 경우(예: 한 유저가 5,000개 이상) 전체 이미지 목록을 프론트엔드에서 파싱하는 게 무거울 수 있음. 이 경우 백엔드에서 `GROUP BY` 한 카테고리 목록만 가져오는 `/api/gallery/active-filters` 스타일의 API로 전환 권장. 현재는 수백 개 수준이므로 충분히 빠름.
- **캐싱**: `staleTime`을 5분으로 설정했으나 `invalidateQueries` 시 즉시 새로고침되도록 보정함.
