# 일일 개발 인수인계

## 변경 파일 목록
- `client/src/components/admin/RewardApplicationManagement.tsx`
- `client/src/components/admin/MenuManagement.tsx`
- `client/src/pages/mymission-detail.tsx`
- `server/controllers/admin/admin.big.mission.controller.ts`

## 핵심 변경 내용
1. **리워드 신청 내역 UI 복구 및 백엔드 500 에러 해결**
   - 백엔드 코드의 ESM 환경에서 호환되지 않는 `require('fs')` 호출로 인해 관리자 페이지에 500 에러가 발생하던 크리티컬 버그 수정.
2. **리워드 관리 대기 건수 배지 추가**
   - 관리자 대시보드 "큰미션" -> "리워드 신청 관리" 탭 옆에 지급 대기 중(`pending`)인 신청 건수를 시각적으로 알려주는 빨간색 알림 배지 (🔴) 추가 기능 구현
3. **'나의 미션 상세' ↔ '리워드 신청 관리' 양방향 이동 연동**
   - 관리자(superadmin, admin)가 일반 유저의 미션 UI를 조회할 때 상단에 `🎁 관리자 리워드 신청 관리` 바로가기 버튼 추가.
   - 버튼 클릭 시 관리자 페이지로 이동하면서 URL 파리미터(`?missionTitle=...`)에 미션명이 전달되도록 구현.
   - 탭 진입 시 URL 파라미터가 있을 경우 자동으로 해당 `missionTitleFilter`가 선택되도록 로직 개선.

## 동작 확인 상태
- ✅ **API 500 에러 복구:** `fs` 모듈 디버깅 제거 완료로 목록 재활성화
- ✅ **배지 컴포넌트:** MenuManagement에 React Query Fetch 구현 확인
- ✅ **라우팅:** Wouter 및 queryParam으로 매끄러운 단독 뷰 연동
