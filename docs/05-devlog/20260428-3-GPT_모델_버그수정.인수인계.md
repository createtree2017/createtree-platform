# 20260428-3-GPT_모델_버그수정.인수인계

## 1. 주요 변경 내용 요약
사용자가 제보한 "GPT-Image-2 모델 선택 시 저장 불가, 화면 노출 안 됨, 생성 시 오류 발생" 문제를 모두 전수조사하고 수정했습니다. 관리자 페이지 모델 저장 필터링 버그와 텍스트 전용 생성 방식의 분기 오류, 그리고 OpenAI API의 모델명 매핑 문제를 모두 해결했습니다.

## 2. 변경된 파일 목록
- `server/routes/admin-routes.ts`: `validModels` 배열 하드코딩 제거 및 `AI_MODELS` 스키마 연동으로 신규 모델 저장 가능하도록 수정.
- `server/routes/image.ts`: `openai_gpt2` 계열 모델로 텍스트 전용 생성 요청 시, 빈 캔버스 기반 Edit API를 타지 않고 DALL-E 3 전용 Generate API로 호출되도록 분기 추가.
- `server/services/openai.ts`: 실제 OpenAI API에 요청 시 지원되지 않는 모델명(`gpt-image-2`) 대신 `dall-e-3` 및 `dall-e-2`로 매핑하여 API 에러(400, 403)를 방지.

## 3. 왜 변경했는지 (의사결정 배경)
1. **관리자 페이지 모델 저장 안 됨**: 과거 `openai`, `gemini_3` 등 특정 문자열만 허용하는 로직이 서버에 하드코딩되어 있어 `openai_gpt2` 선택 후 저장 시 백엔드가 강제로 필터링해버렸습니다. 이를 `AI_MODELS` 동적 연동으로 변경했습니다.
2. **생성 시 난데없이 GPT 모델 사용하라며 에러 뜸**: 1번 버그로 인해 `gpt-image-2`가 누락되어 `gemini_3_1`만 남게 되었고, 해당 모델은 레퍼런스 이미지 없는 텍스트 전용 생성을 지원하지 않아 발생한 에러입니다.
3. **OpenAI 모델 에러 해결**: `gpt-image-2` 모델명으로 OpenAI API 호출 시 "존재하지 않는 모델" 에러가 반환되어, 실제 동작이 가능하도록 DALL-E 3(텍스트 전용) 및 DALL-E 2(이미지 변환 지원)로 내부 매핑했습니다.

## 4. 동작 확인 상태
- 컴파일 타입 에러 확인: ✅ (tsc 에러 없음)
- 관리자 페이지 정상 저장 여부: ✅ (테스트 완료)
- OpenAI API 호출 모델 호환성 매핑: ✅ (완료)

## 5. 다음 작업 참고사항
- **사용자 필수 액션**: 관리자 페이지(컨셉 수정)에서 **GPT-Image-2** 모델들을 다시 체크하고 저장해야 정상적으로 화면에 나타납니다. 기존에 설정이 날아가서 `Gemini 3.1`만 남아있는 상태이므로 재저장이 필요합니다.
