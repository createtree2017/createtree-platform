# 액션 타입 관리 개선

작성일: 2026-05-08

## 1. 작업 개요

관리자 페이지의 문화센터 `액션 타입 관리`에서 액션 타입 목록 순서를 직접 지정할 수 있도록 추가/수정 모달에 `순서` 숫자 입력을 추가했다.
추가로 사용자 미션 상세 하단 탭에서 기본 액션 타입 이름을 하드코딩 라벨로 바꾸던 동작을 제거해, 관리자가 지정한 액션 타입 이름이 그대로 표시되도록 했다.

## 4. 핵심 변경 내용

- 액션 타입 추가/수정 모달에 `순서` 숫자 입력 필드를 추가했다.
- 수정 시 기존 `order`, 추가 시 현재 최대 `order + 1`을 기본값으로 사용하도록 했다.
- 액션 타입 저장 요청에 `order`를 포함하도록 프론트 저장 타입을 보강했다.
- 서버에서 `name`, `iconUrl`, `isActive`, `order`만 허용하고 `order`는 `0` 이상 정수로 검증하도록 했다.
- 액션 타입 목록과 활성 액션 타입 조회를 `order ASC, id ASC`로 정렬해 같은 순서 값이 있어도 표시 순서가 흔들리지 않게 했다.
- 신규 액션 타입 생성 시 서버가 입력받은 `order`를 무시하고 마지막 순서로 덮어쓰던 로직을 제거했다.
- 사용자 미션 상세 하단 탭의 라벨 매핑(`제출` → `제출하기`, `출석` → `출석인증`, `리뷰` → `리뷰작성`)을 제거하고 `actionType.name`을 그대로 표시하도록 했다.

## 5. 변경 이유

관리자 목록에는 순서 값이 노출되지만 수정 UI에는 입력 경로가 없어 운영자가 액션 타입의 사용자 노출 순서를 조정할 수 없었다. 기존 DB 컬럼과 정렬 구조를 활용해 최소 변경으로 운영 편의성을 확보했다.
또한 기본 액션 타입만 사용자 화면에서 다른 문구로 표시되면 관리자가 신규 액션 타입을 설정할 때 실제 노출명과 관리자 설정명이 달라 혼란이 생길 수 있어, 사용자 화면도 관리자 설정명을 기준으로 통일했다.

## 6. 동작 확인 상태

- `npm run build` 성공
- 브라우저 검증은 사용자 요청에 `!!테스트!!`가 포함되지 않아 자동 수행하지 않는다.

## 7. 주의사항 / 알려진 이슈

- `신청` 액션 타입은 기존 정책대로 수정/삭제 버튼이 잠겨 있어 순서도 관리자 UI에서 직접 바꿀 수 없다.
- 중복 순서 값은 허용한다. 같은 순서 값 안에서는 `id` 오름차순으로 표시된다.
- DB 스키마 변경은 없으므로 `drizzle-kit push`는 필요하지 않다.

## 8. 수동 테스트 참고

1. `localhost:5000/api/dev/auto-login` 접속
2. 관리자 페이지 → 메뉴관리 → 문화센터 → 미션관리 → 액션 타입 관리 이동
3. 수정 가능한 액션 타입 수정 버튼 클릭
4. `순서` 값을 변경하고 저장
5. 목록의 순서 값과 표시 위치가 변경됐는지 확인
