# 20260321-2 미션 500 에러 수정 인수인계

## 작업 개요
문화센터 미션 상세 페이지 접속 시 발생하던 500 Internal Server Error 수정

## 원인 분석

### 근본 원인
`shared/schema.ts`에 `rejectReason: text("reject_reason")` 컬럼이 추가됐으나, 실제 DB 테이블(`sub_mission_submissions`)에 해당 컬럼이 존재하지 않아서 Drizzle ORM의 `findFirst` 쿼리 실행 시 PostgreSQL 에러 발생.

**에러 코드**: `code: '42703' (errorMissingColumn)` - `column "reject_reason" does not exist`

### 에러 발생 경로
```
GET /api/missions/:missionId
→ user.mission.controller.ts getMissionDetail()
→ mission.user.service.ts getMissionDetail()
  → db.query.subMissionSubmissions.findFirst()  ← 여기서 42703 에러
→ catch (error) → res.status(500).json({ error: "미션 상세 조회 실패" })
```

### 관련 커밋
- `6a95e04`: "fix: unify AI models, add rejectReason schema, fix gallery showing user uploads"
- 이 커밋에서 `shared/schema.ts`에 `rejectReason`이 추가됐으나 DB 마이그레이션 누락

## 수정 내용

### 변경 파일
- `shared/schema.ts` - 기존 커밋에서 이미 `rejectReason` 필드 추가됨 (변경 없음)
- `server/routes/misc-routes.ts` - 임시 마이그레이션 엔드포인트 추가 후 실행 완료, 이후 제거

### 실제 수정 (DB)
```sql
ALTER TABLE sub_mission_submissions ADD COLUMN IF NOT EXISTS reject_reason TEXT;
```
- PowerShell curl로 임시 API 엔드포인트 호출하여 DB 컬럼 추가 완료

## 동작 확인 상태

| 항목 | 상태 |
|------|------|
| DB 컬럼 추가 | ✅ 성공 |
