## 2. 핵심 변경 내용 및 원인(왜 변경했는지)
**[Bug 1] 프로덕션 환경의 비밀번호 찾기 이메일 URL 오류**
- **기존:** `PRODUCTION_DOMAIN` 환경변수 부재 시 프로덕션 환경이어도 하드코딩된 `http://localhost:5000` 도메인으로 이메일에 링크가 생성되는 버그가 존재했습니다.
- **수정:** 링크 생성 시 `req.protocol` 및 `req.headers.host`를 읽어들여 런타임에 동적으로 실제 호스트 URL을 삽입하도록 변경했습니다. 

**[Bug 2] 발급된 재설정 토큰이 실시간으로 만료되는 버그 ("유효하지 않은 링크")**
- **기존:** JS의 `new Date()` 객체를 Drizzle ORM을 통해 `timestamp without time zone` 컬럼에 바로 삽입하고 조회하면서 `pg` 드라이버의 캐스팅 정책에 의해 시스템이 9시간의 타임존 오차를 발생시켰습니다. 생성된 토큰이 백엔드 내부적으로 8시간 전에 이미 만료된 것으로 처리되어 이메일 링크를 클릭하자마자 400 에러를 반환했습니다.
- **수정:** 자바스크립트의 시간 객체 생성을 폐기하고, 토큰 생성 및 검증 코드를 PostgreSQL 네이티브인 `sql\`NOW() + interval '1 hour'\``와 `sql\`NOW()\``로 대체하였습니다. 이를 통해 서버 환경과 상관없이 완벽하게 데이터베이스 시간과 동기화되어 유효 기간 로직이 정상 작동합니다.
