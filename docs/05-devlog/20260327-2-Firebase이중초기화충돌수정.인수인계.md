# 20260327-2-Firebase이중초기화충돌수정 인수인계

## 1. 개요
* **목적:** 관리자 페이지 푸시 발송 시 발생하는 Firebase Admin SDK 이중 초기화(`The default Firebase app already exists`) 에러 수정
* **작업일:** 2026년 3월 27일

## 2. 변경 파일 목록
* `server/services/firebase-admin.ts`

## 3. 핵심 변경 내용 & 변경 배경 (Why)
* **문제 배경:** FCM 푸시 직접 발송(스크립트)은 성공하지만 관리자 페이지를 통한 푸시 발송은 여전히 실패(`successCount: 0`)하는 현상 발생.
* **원인:** 다른 개발자분의 정확한 지적대로, 프로젝트 내에 Firebase Admin을 초기화하는 파일이 두 개 존재했습니다.
  1. `server/firebase.ts` (기존, Storage/Auth용) - 서버 실행 즉시 default app 초기화
  2. `server/services/firebase-admin.ts` (신규, FCM용) - API 호출 시 `getFirebaseAdmin()`에서 또다시 default app 초기화 시도
  이로 인해 실제 서버 환경에서는 두 번째 초기화가 충돌 에러를 뿜었고(`The default Firebase app already exists`), 이것이 `push.service.ts`의 `catch`에 걸려 실패 처리되었습니다. 
* **조치 내용:** `firebase-admin.ts`의 `getFirebaseAdmin()` 내부 로직을 수정하여, 이미 `admin.apps.length > 0`인 경우 기존에 초기화된 앱 인스턴스(`admin.apps[0]`)를 가져와 **재사용**하도록 예외 처리(이중 초기화 방지)를 추가했습니다.
