# 20260327-1-푸시발송오류수정 인수인계

## 1. 개요
* **목적:** 프로덕션 환경 및 로컬 환경에서 푸시 알림 전송이 동작하지 않고 성공/실패 카운트가 0으로 출력되는 문제 해결
* **작업일:** 2026년 3월 27일

## 2. 변경 파일 목록
* `server/services/firebase-admin.ts`

## 3. 핵심 변경 내용 & 변경 배경 (Why)
* **문제 배경:** 푸시 알림 발송 시, 앱의 내부 알림함(`my-page` 알림 목록)에는 정위치 저장되지만 실제 안드로이드 기기로 FCM 알림이 도착하지 않는 현상 발생. Database의 `push_delivery_logs` 확인 결과, 상태는 `completed`이나 `success_count` 및 `failure_count`가 모두 0인 상태 발견.
* **원인:** `server/services/firebase-admin.ts`에서 Firebase Admin SDK 초기화 시 `FIREBASE_SERVICE_ACCOUNT` 환경변수만 의존하고 있었음. 하지만 Railway 및 로컬 `.env`에는 `GOOGLE_APPLICATION_CREDENTIALS_JSON`이라는 다른 이름으로 Firebase 키가 등록되어 있어, `getFirebaseMessaging` 호출 단계에서 환경변수 누락 Exception 발생. 이 예외가 `push.service.ts`의 `Promise.allSettled` 바깥 (청크 반복문 이전)에서 발생하며 조기 캐치됨으로써 성공, 실패가 0으로 종료되었음.
* **조치 내용:** `firebase-admin.ts`에서 `process.env.FIREBASE_SERVICE_ACCOUNT` 뿐만 아니라 `process.env.GOOGLE_APPLICATION_CREDENTIALS_JSON`를 함께 체크하여 Admin SDK가 정상적으로 초기화되도록 수정 완료.

## 4. 동작 확인 상태
* ✅ 로컬 환경 터미널 스크립트 기반 `getFirebaseMessaging` 정상 할당 확인 완료 (`test-firebase-admin.ts` 스크립트 삭제 예정)
* 🟡 안드로이드 프로덕션 기기에서의 확인은 **반드시 이 코드를 Railway에 배포**한 후 다시 시도해야 합니다.

## 5. 주의사항 / 알려진 이슈
* 이 수정은 서버쪽 푸시 발송 로직(Firebase Admin SDK) 수정입니다. 수정된 코드를 Git commit 및 push 하여 Railway 등 프로덕션 서버에 반영한 직후, 다시 관리자 페이지 또는 미션 신청에서 푸시를 발생시켜야 정상 수신됩니다.
