# GPT 이미지 파이프라인 최적화 및 모델별 설정 체계 인수인계

## 1. 작업 개요

- 작업일자: 2026-04-29
- 작업명: GPT 이미지 파이프라인 속도 최적화 및 AI 모델별 capability registry 구축
- 작업 목적:
  - 관리자 컨셉 생성/수정 화면에서 AI 모델별 비율, 해상도, 품질, 출력 포맷을 정확히 설정한다.
  - 사용자는 관리자 설정값에 따라 이미지를 생성하고, 비율/품질/포맷을 직접 선택하지 않는다.
  - GPT-Image-2 생성 속도 저하 원인으로 추정되는 `quality=high` 고정, PNG 기본 출력, 원본 이미지 전송, text-only edit API 호출을 개선한다.
  - 흩어진 OpenAI 이미지 호출 경로를 provider adapter로 통합해 향후 모델 추가/삭제/수정이 쉬운 구조로 만든다.

## 2. 핵심 설계 원칙

- 모델 스펙은 회사별이 아니라 모델별 capability registry에서 관리한다.
- 프론트엔드는 `quality`, `outputFormat`, `inputMaxEdge`를 사용자 요청값으로 보내지 않는다.
- 백엔드는 컨셉 조회 후 `generationSettings[finalModel]`를 읽어 최종 생성 옵션을 결정한다.
- 기존 컨셉 호환을 위해 `availableAspectRatios`, `gemini3ImageSize`, `gemini3AspectRatio` fallback을 유지한다.
- OpenAI 이미지 호출은 모두 신규 image generation service 또는 adapter를 통해 수행한다.
- `git add`, `git commit`, `git push`는 사용자가 `!!푸시!!`를 명시한 경우에만 수행한다.

## 4. 모델별 Capability Registry

`shared/model-capabilities.ts`에 모델별 스펙을 정의한다.

- `openai_gpt2`
  - API 모델: `gpt-image-2`
  - 비율/size: `auto`, `1:1`, `2:3`, `3:2`
  - 참고: OpenAI Image API 공식 문서의 안정 공개 `size`는 `1024x1024`, `1024x1536`, `1536x1024`, `auto` 기준이다. 따라서 Gemini 10개 비율을 GPT-Image-2에 그대로 노출하지 않는다.
  - 품질: `low`, `medium`, `high`, `auto`
  - 출력 포맷: `webp`, `jpeg`, `png`
  - 기본값: `quality=medium`, `outputFormat=webp`, `resolutionPreset=standard`, `inputMaxEdge=1536`
- `openai_gpt1_5`
  - API 모델: `gpt-image-1.5`
  - 비율/size: `auto`, `1:1`, `2:3`, `3:2`
  - 자유 픽셀 지정 없음
- `gemini_3_1`
  - 기존 Gemini 3.1 Flash 이미지 생성 설정 유지
- `gemini_3`
  - 기존 Gemini 3.0 Pro 이미지 생성 설정 유지
  - `imageSize`: `1K`, `2K`, `4K`

## 5. DB 및 마이그레이션

- `concepts` 테이블에 `generationSettings` JSONB 필드를 추가한다.
- 기존 필드는 즉시 제거하지 않고 fallback으로 유지한다.
  - `availableModels`
  - `availableAspectRatios`
  - `gemini3AspectRatio`
  - `gemini3ImageSize`
- `server/utils/settings.ts`에서 기존 데이터를 새 구조로 자동 보정한다.
  - `availableAspectRatios` -> `generationSettings[model].aspectRatios`
  - `gemini3ImageSize` -> `generationSettings.gemini_3.imageSize`
  - `gemini3AspectRatio` -> Gemini fallback
  - `gemini` -> `gemini_3_1` 마이그레이션도 새 구조에 반영
- 스키마 변경 후 DB 반영을 확인한다.
  - `npm run db:push`는 기존 DB와 로컬 스키마 차이 때문에 unrelated constraint 변경을 대량 시도하며 실패할 수 있다.
  - 2026-04-29 작업에서는 `concepts.generation_settings` 컬럼만 단일 SQL로 추가했고, `available_aspect_ratios` 기본값도 OpenAI 3개 비율 기준으로 보정했다.

## 7. 사용자 화면 변경

- `ImageGenerationTemplate.tsx`는 사용자에게 품질/포맷/해상도 설정을 노출하지 않는다.
- 생성 요청에는 컨셉, 변수, 이미지 파일 중심 정보만 보낸다.
- 백엔드가 컨셉 설정을 권위값으로 사용한다.
- 생성 중 안내 문구를 개선한다.
  - 업로드 중
  - 이미지 생성 중
  - 저장 중
  - GPT 고품질 설정은 시간이 더 걸릴 수 있음

## 8. Backend Adapter 통합

### OpenAI adapter

- `modelKey` -> 실제 OpenAI 모델명 변환
- `aspectRatio + resolutionPreset` -> `size` 변환
- `quality`, `output_format` 적용
- text-only는 `/v1/images/generations`
- image-upload는 `/v1/images/edits`
- 입력 이미지는 `inputMaxEdge` 기준으로 sharp 리사이즈/압축
- `AbortController` 기반 timeout 적용
- data URL MIME 타입을 output format에 맞게 처리
  - `data:image/webp;base64,...`
  - `data:image/jpeg;base64,...`
  - `data:image/png;base64,...`

### Gemini adapter

- 기존 Gemini 서비스의 `imageConfig.aspectRatio`, `imageConfig.imageSize` 흐름을 유지한다.
- 새 `generationSettings`에서 비율과 해상도를 읽어 전달한다.

## 9. OpenAI 레거시 경로 정리

다음 경로를 신규 adapter 또는 image generation service로 통합한다.

- `/generate-image`
- `/generate-family`
- `/generate-stickers`
- `/public/image-transform`
- `/transform`

주의사항:

- `/public/image-transform`는 `uploadFields` 사용 중이므로 `req.file`이 아니라 `req.files?.image?.[0]`를 사용해야 한다.
- 업로드 설정이 memoryStorage이므로 `file.path`가 아니라 `file.buffer`를 기준으로 처리한다.
- `/transform`은 현재 실제 변환 없이 메시지만 반환하는 스텁이므로 호출처 확인 후 연결 또는 정리한다.
- `getOpenAIImageModelName`, `getOpenAIImageSize` 같은 하드코딩 유틸은 registry/adapter로 대체한다.
- `server/services/openai.ts`, `server/services/openai-simple.ts`, `server/storage.ts`의 레거시 OpenAI 이미지 경로는 제거, deprecated, 또는 adapter 위임 중 하나로 정리한다.

## 10. GPT 계열 해상도 매핑

`standard` 프리셋 기준 기본 매핑:

```text
auto -> auto
1:1  -> 1024x1024
2:3  -> 1024x1536
3:2  -> 1536x1024
```

Gemini의 `3:4`, `4:3`, `4:5`, `5:4`, `9:16`, `16:9`, `21:9`는 OpenAI GPT 계열에 노출하지 않는다. 향후 OpenAI가 추가 비율 또는 자유 픽셀 지정을 공식 문서/API에서 안정 지원하면 `shared/model-capabilities.ts`만 확장해 반영한다.

## 15. 2026-04-29 추가 변경: 모델별 단일 비율 선택 및 자동 모드

- 관리자 컨셉 수정 UI의 비율 선택은 모델별 단일 선택으로 변경했다.
- 저장 구조는 하위호환을 위해 `aspectRatios: string[]`를 유지하지만, 정규화 결과는 항상 1개 값만 남긴다.
- `auto` 선택 시 OpenAI GPT 계열은 `size: "auto"`를 전달한다.
- `auto` 선택 시 Gemini 계열은 `imageConfig.aspectRatio`를 전달하지 않는다.
- 사용자 생성 화면은 더 이상 `aspectRatio`를 생성 요청에 포함하지 않는다.
- Gemini 3.1 Flash Image Preview는 공식 문서 기준 `auto`, `1:1`, `1:4`, `1:8`, `2:3`, `3:2`, `3:4`, `4:1`, `4:3`, `4:5`, `5:4`, `8:1`, `9:16`, `16:9`, `21:9`를 노출한다.
- Gemini 3.1 Flash Image Preview의 해상도 옵션은 `512`, `1K`, `2K`, `4K`를 노출한다.
- Gemini 3 Pro Image Preview의 비율은 `auto`와 기존 10개 비율을 노출하고, 해상도는 `1K`, `2K`, `4K`를 노출한다.
- DB `concepts.available_aspect_ratios` 기본값은 신규 컨셉 호환을 위해 모델별 `["auto"]`로 보정했다.
