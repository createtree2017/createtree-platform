## 3. 핵심 변경 내용
모든 이미지 생성 페이지(만삭사진, 가족사진, 아기얼굴, 스티커 등)가 공유하는 `ImageGenerationTemplate` 컴포넌트 전체에 걸쳐 하드코딩된 색상 클래스를 테마 변수 클래스로 교체했습니다.

| 영역 | 변경 전 (하드코딩) | 변경 후 (시맨틱 컬러) |
|---|---|---|
| 제목 | `text-white` | `text-foreground` |
| 부제목 | `text-gray-300` | `text-muted-foreground` |
| 카드 배경 | `bg-gray-800 border-gray-700` | `bg-card border-border` |
| 업로드/선택 내부 배경 | `bg-gray-700 border-gray-600` | `bg-muted border-border` |
| 생성 버튼 컨테이너 | `bg-white` | `bg-card border-border` |
| 갤러리 섹션 | `bg-gray-800 text-white` | `bg-card text-foreground` |
| 변수입력 필드 | `border-gray-300 text-black` | `border-border bg-background text-foreground` |

또한, 제목(h1) 영역의 레이아웃도 `snapshot` 및 다른 메뉴들과 통일하여 `Sparkles` 아이콘과 `text-4xl md:text-5xl` 크기 구조를 적용했습니다.

## 4. 변경 배경 및 의사결정 이유
- `ImageGenerationTemplate`은 다크/라이트/파스텔 모드 지원이 안 되는 상태였습니다. 다크 배경에서 보기 좋도록 설계되어 밝은 테마에서는 어두운 카드 영역이 그대로 남아 미관상 이질감이 컸습니다.
- `bg-card`, `text-foreground` 등은 `use-theme.tsx`에서 테마별로 자동 계산되는 CSS 변수(`--card`, `--foreground`)에 매핑되므로, 이 변경으로 모든 테마에 자동 대응할 수 있습니다.
