# 이미지 부분 첨부 확인 인수인계

작성일: 2026-05-06

## 1. 작업 개요

문화센터 세부미션의 이미지 제출 슬롯이 여러 개인 경우, 사용자가 일부 이미지만 첨부하고 제출하려 할 때 확인 모달을 표시하도록 변경했다.

## 2. 변경 파일

- `client/src/pages/mission-detail.tsx`
- `client/src/components/modal/modalRegistrations.ts`
- `client/src/components/modal/user/ConfirmModal.tsx`

## 3. 핵심 변경 내용

- 이미지 제출 슬롯의 총 개수와 실제 첨부된 이미지 개수를 계산하는 로직을 추가했다.
- 이미지 슬롯이 2개 이상이고, 첨부된 이미지 수가 전체 이미지 슬롯 수보다 적으면 제출 직전에 확인 모달을 띄운다.
- 확인 문구는 `{전체}개의 첨부 이미지 중 {첨부}개만 첨부했습니다. 그래도 제출하시겠습니까?` 형식으로 표시한다.
- 사용자가 `그래도 제출하기`를 선택하면 제출을 계속 진행하고, `추가 첨부하기`를 선택하면 모달만 닫고 작성 화면에 머문다.
- 기존 제출물을 수정 제출하는 경우에는 이미지 부분 첨부 확인 문구에 기존 제출 내용 수정 안내를 함께 표시한다.
- `ConfirmModal`이 기존 `message` prop과 신규 `description` prop을 모두 안전하게 표시하도록 보강했다.

## 4. 변경 이유

관리자가 여러 개의 이미지 제출 슬롯을 설정했는데 사용자가 1개 또는 일부 이미지만 첨부한 채 제출하는 실수가 자주 발생했다. 제출 자체를 막지는 않되, 의도한 부분 제출인지 한 번 더 확인하도록 하여 사용자 실수를 줄이기 위한 변경이다.

## 5. 동작 확인 상태

- `npm run check` 실행 결과: 프로젝트 전역의 기존 타입 오류로 실패했다.
- 이번 변경 파일 기준 필터 확인: `client/src/pages/mission-detail.tsx`, `client/src/components/modal/user/ConfirmModal.tsx`, `client/src/components/modal/modalRegistrations.ts` 관련 신규 타입 오류는 출력되지 않았다.
- 이번 요청에 `!!테스트!!`가 포함되지 않아 브라우저 직접 검증은 수행하지 않았다.

## 6. 주의사항

- 이미지 슬롯을 모두 첨부한 경우에는 확인 모달 없이 기존 흐름대로 바로 제출된다.
- 이미지 슬롯이 1개뿐인 세부미션에는 부분 첨부 확인 모달이 표시되지 않는다.
- `git add`, `git commit`, `git push`는 수행하지 않았다.
