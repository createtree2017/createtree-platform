## 4. 경미한 미비 2건 수정 내역

### 수정 1: `shouldSendNotification()` mission case 추가

- **파일**: `server/services/notifications.ts` (389~398줄)
- **변경 배경**: switch-case에 `mission_approve`/`mission_reject` case가 없어 `default: true`로 처리되고 있었음. 향후 카테고리별 ON/OFF UI 구현 시 명시적 case가 필요하므로 사전 추가.

```diff
       case 'application_approved':
       case 'application_rejected':
+      case 'mission_approve':
+      case 'mission_approved':
+      case 'mission_reject':
+      case 'mission_rejected':
         return settings.applicationStatusChanges ?? true;
```

### 수정 2: Rate Limiting 적용

- **파일**: `server/routes/admin-routes.ts` (5줄, 3658~3666줄)
- **변경 배경**: 관리자 push-send API에 Rate Limiter 미적용으로 연타 시 대량 중복 발송 가능. `express-rate-limit` (이미 설치 완료)를 활용하여 분당 5회로 제한.

```diff
+ import rateLimit from "express-rate-limit";
  ...
+  const pushSendLimiter = rateLimit({
+    windowMs: 60 * 1000,
+    max: 5,
+    message: { error: "푸시 발송 요청이 너무 많습니다. 1분 후 다시 시도해주세요." },
+    standardHeaders: true,
+    legacyHeaders: false,
+  });
+
-  app.post("/api/admin/push-send", requireAdminOrSuperAdmin, async ...
+  app.post("/api/admin/push-send", requireAdminOrSuperAdmin, pushSendLimiter, async ...
```
