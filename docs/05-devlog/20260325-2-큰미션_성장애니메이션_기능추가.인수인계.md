# 20260325-2-큰미션_성장애니메이션_기능추가.인수인계.md

## 변경 개요

- **날짜**: 2026-03-25
- **작업자**: AI (Antigravity)
- **요청**: 관리자 > 큰미션 생성/수정 폼에 성장 애니메이션 기능 추가 (ON/OFF + 캐릭터이름 + 단계이미지 1~10개)

---

## 변경 파일

### 1. `shared/schema.ts` (1484~1492줄)
`bigMissions` 테이블에 컬럼 3개 추가:
- `growth_enabled` (BOOLEAN, default false)
- `growth_tree_name` (TEXT, default '사과몽')
- `growth_stage_images` (JSONB, default '[]')

### 2. `server/controllers/admin/admin.big.mission.controller.ts`
`createBigMission`과 `updateBigMission` 두 곳의 `allowedFields` 배열에 추가:
- `'growthEnabled'`, `'growthTreeName'`, `'growthStageImages'`

### 3. `client/src/components/modal/admin/BigMissionModals.tsx`
- `BigMission` 인터페이스에 `growthEnabled?`, `growthTreeName?`, `growthStageImages?` 추가
- `DEFAULT_FORM`에 기본값 추가
- `useEffect` 초기화 블록에 3개 필드 추가
- **신규 UI 섹션**: "🌱 성장 애니메이션" — Switch 토글, 캐릭터 이름 Input, Lv.1~10 이미지 단계 업로더
  - `+ 단계 추가` 버튼 (최대 10개 제한)
  - 각 단계: Lv.N 라벨 + 미리보기(40×40) + 파일 input(hidden) + 업로드 완료/없음 표시 + × 삭제 버튼
  - 업로드: `POST /api/upload` → GCS URL → `growthStageImages[idx]`에 저장
  - 업로드 로딩: 기존 `uploadingField` 상태 재사용 (`growth-{idx}` 키)

### 4. `client/src/components/ui/CreationTreeProgress.tsx`
- props에 `stageImages?: string[]` 추가
- 이미지 경로 로직: `stageImages.length > 0`이면 DB 배열 사용 (stageIndex 기반), 없으면 정적 파일 폴백(`apple_mong_stage_N.png`)

### 5. `client/src/pages/mymission-detail.tsx`
- `BigMissionDetail` 인터페이스에 3개 필드 추가
- `CreationTreeProgress`를 `mission.growthEnabled === true`일 때만 렌더링
- `treeName`, `stageImages` props 전달

---

## 왜 변경했는지

관리자가 큰미션마다 다른 성장 캐릭터 이미지를 설정할 수 있도록.
기존은 정적 파일(`apple_mong_stage_N.png`)에 의존 → 실제 이미지 없어서 AI 로고(fallback)만 보임.
이제 관리자 UI에서 직접 1~10단계 이미지를 GCS에 업로드하면 사용자 화면에 반영.

---

## DB 마이그레이션

`drizzle-kit push` 실패(FK 충돌) → `scripts/add-growth-columns.ts` 스크립트로 직접 ALTER TABLE 실행.
스크립트 실행 완료 후 파일 삭제됨. 컬럼 3개 추가 확인됨.

## 동작 확인 상태

- ✅ DB 컬럼 추가 확인
- ✅ 코드 수정 완료
- 👤 수동 테스트 필요:
  1. `localhost:5000/api/dev/auto-login` 접속
  2. 관리자 > 메뉴관리 > 나의미션 > 큰미션 추가
  3. 폼 하단 "🌱 성장 애니메이션" 섹션 스위치 ON
  4. 캐릭터 이름 입력, `+ 단계 추가` × 3, 각 이미지 선택 업로드
  5. 저장 후 `/mymissions/{id}` 에서 CreationTreeProgress 표시 확인

---

## 주의사항 / 알려진 이슈

- `growthEnabled = false`인 기존 큰미션은 상세 페이지에서 CreationTreeProgress가 **숨겨짐**
  - 기존에 AI로고(fallback)가 보이던 현상이 사라지는 것이 정상
- `growthStageImages` 배열은 최대 10개 이하로 제한 (UI에서 10개 도달 시 `+ 단계 추가` 버튼 숨김)
- 단계 이미지가 업로드된 개수보다 currentLevel이 높으면 마지막 이미지로 클램프 처리됨
