# 관리자 이미지 갤러리 다운로드 방식 수정 인수인계

작성일: 2026-04-30

## 1. 배경

관리자 페이지의 이미지 갤러리에서 이미지 상세 모달의 다운로드 버튼을 눌러도 다운로드가 정상 동작하지 않는 문제가 있었다.

사용자 갤러리에서는 다운로드가 정상 동작하고 있었으므로, 새 다운로드 API를 만들기보다 사용자 갤러리에서 검증된 다운로드 방식을 관리자 갤러리에 적용하기로 했다.

## 2. 원인

`client/src/components/admin/ImageGalleryAdmin.tsx`의 다운로드 버튼은 `handleDownloadClick()`을 호출했지만, 해당 함수는 `downloadDialogOpen` 상태만 `true`로 변경했다.

그러나 실제 다운로드 형식 선택 대화상자는 렌더링되지 않아 버튼 클릭 후 다운로드가 실행되지 않았다.

또한 기존 `handleDownload()` 내부는 외부 이미지 URL을 `a.href`에 직접 연결하는 방식이었고, 이후 `downloadMedia(image.id, 'image')`를 호출했다. 이 `downloadMedia()`는 `/api/image/:id/download` 경로를 사용하지만, 현재 서버의 실제 미디어 다운로드 라우트는 `/api/media/download/:type/:id` 구조라 경로가 맞지 않았다.

## 3. 변경 내용

### `client/src/components/admin/ImageGalleryAdmin.tsx`

- 관리자 이미지 갤러리 다운로드를 사용자 갤러리와 같은 방식으로 변경했다.
  - `image.transformedUrl || image.url`을 `fetch()`로 가져온다.
  - 응답을 `blob`으로 변환한다.
  - `window.URL.createObjectURL(blob)`으로 object URL을 만든다.
  - 임시 `a` 태그의 `download` 속성으로 파일을 저장한다.
  - 다운로드 실행 후 object URL을 해제한다.
- 실제로 렌더링되지 않던 다운로드 대화상자 관련 상태를 제거했다.
- 서버 경로가 맞지 않아 콘솔 오류를 만들던 `downloadMedia()` 호출과 import를 제거했다.
- 파일 확장자는 응답 MIME 타입을 우선 사용하고, 없으면 URL 확장자 또는 `webp`를 fallback으로 사용한다.
- 관리자 이미지 갤러리 상세 모달에서 `일반 공유` 버튼을 제거했다.
  - 관리자 모니터링 화면 목적과 맞지 않는 공유 기능을 숨겼다.
  - 현재 서버 라우트와 맞지 않는 `shareMedia()` import 및 `handleShare()` 함수도 함께 제거했다.

- 이번 변경은 관리자 이미지 갤러리의 다운로드 동작만 수정했다.
- `일반 공유` 버튼은 후속 승인 요청에 따라 제거했다.
- 서버 API, DB 스키마, 이미지 저장 구조는 변경하지 않았다.
