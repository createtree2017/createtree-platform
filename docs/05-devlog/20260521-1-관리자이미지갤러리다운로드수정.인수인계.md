# 관리자 이미지 갤러리 다운로드 수정

작성일: 2026-05-21

## 1. 작업 개요

관리자페이지 `메뉴관리 > AI 생성 > 이미지 갤러리`에서 썸네일은 보이지만 다운로드 클릭 시 GCS CORS 오류로 실패하던 문제를 수정했다.

## 2. 연결 문서

- Plan: 없음 (기존 기능 다운로드 경로만 바꾸는 Quick Fix)
- Design: 없음

## 3. 변경 파일

- `server/routes/admin-routes.ts`
- `client/src/components/admin/ImageGalleryAdmin.tsx`

## 4. 핵심 변경 내용

- 관리자 전용 다운로드 프록시 API `GET /api/admin/images/:imageId/download`를 추가했다.
- 새 API는 `requireAdminOrSuperAdmin` 권한 검사를 통과한 관리자만 사용할 수 있다.
- 다운로드 대상 URL은 `transformedUrl`을 우선 사용하고, 없으면 `originalUrl`을 사용한다.
- `resolveImageUrl`을 거쳐 만료된 GCS signed URL을 공개 URL로 보정한 뒤 서버에서 이미지를 받아 내려준다.
- base64 이미지와 `/uploads/`, `/static/` 로컬 이미지 케이스도 함께 처리한다.
- 관리자 갤러리 프론트는 더 이상 GCS URL을 브라우저에서 직접 `fetch()`하지 않고, 새 관리자 다운로드 API를 링크로 호출한다.
- 기존 썸네일 표시 로직과 `/api/admin/images` 목록 API 응답 구조는 변경하지 않았다.

## 5. 왜 변경했는지

관리자 갤러리 다운로드는 브라우저에서 `https://storage.googleapis.com/...`을 직접 `fetch()`하고 있었다. GCS 버킷 응답에 `Access-Control-Allow-Origin` 헤더가 없으면 브라우저 CORS 정책에 의해 blob 생성 전 단계에서 차단된다. 사용자 갤러리는 이미 서버 프록시(`/api/download-image/:imageId`)를 통해 다운로드하고 있어 정상 동작했으므로, 관리자 갤러리도 같은 원칙으로 서버를 경유하도록 수정했다.

## 6. 동작 확인 상태

- `npm run build`: 성공
- `npx tsc --pretty false --noEmit`: 실패. 기존 `PersonaCategoryManagement.tsx`, `PersonaManagement.tsx`, `server/services/milestones.ts` 타입 오류가 남아 있으며 이번 변경 파일 관련 신규 오류는 확인되지 않았다.
- `git diff --check`: 통과 (CRLF 변환 경고만 표시)
- 브라우저 검증: `http://localhost:5000/admin?menuItem=ai-create&panel=image-gallery`에서 관리자 이미지 갤러리 진입 및 썸네일 10개 로딩 확인
- 다운로드 API 검증: 관리자 인증 세션으로 `GET /api/admin/images/10321/download` 호출 시 `200`, `Content-Type: image/webp`, `Content-Disposition: attachment`, 파일 크기 `86328 bytes` 확인

## 7. 주의사항 / 알려진 이슈

- Codex 인앱 브라우저는 실제 파일 다운로드 이벤트를 지원하지 않아, 다운로드 버튼 클릭의 파일 저장 이벤트는 브라우저에서 끝까지 검증하지 못했다. 대신 브라우저로 화면/썸네일을 확인하고, 동일 인증 흐름의 HTTP 요청으로 다운로드 API 응답을 검증했다.
- 기존 전체 TypeScript 오류는 이번 작업 범위 밖이다.
- 새 다운로드 API는 전체 사용자 이미지를 대상으로 하므로 관리자 권한 미들웨어 유지가 중요하다.

## 8. 연관 파일

- 관리자 갤러리 다운로드 경로가 서버 프록시 방식으로 정리되었지만, 프로젝트 운영 스킬의 자동화 정책이나 이미지 provider/model 규칙 변경은 아니다.
- 스킬 문서 업데이트는 불필요하다고 판단했다.
