# 인수인계: Gemini 이미지 모델 마이그레이션

> **작업일**: 2026-03-19
> **작업 분류**: Major Feature (200줄 이상 다수 파일 변경)
> **목적**: `gemini-2.5-flash-image` (Nano Banana) 제거 → `gemini-3.1-flash-image-preview` (Nano Banana 2) 로 전환

---

## 1. 변경 개요

| 구분 | 변경 전 | 변경 후 |
|------|---------|---------|
| 표준 모델 키 | `gemini` | `gemini_3_1` |
| API 모델명 | `gemini-2.5-flash-image` | `gemini-3.1-flash-image-preview` |
| 프로 모델 | `gemini_3` (변경 없음) | `gemini_3` (변경 없음) |

- 경제형(gemini 2.5) **제거**, 표준형(3.1 Flash) + 프로형(3.0 Pro) **2개만 유지**
- 기존 gemini 2.5 사용 템플릿(서비스)을 모두 gemini 3.1로 변경

## 3. 핵심 변경 내용

### 3-1. 스키마 변경 (`shared/schema.ts`)
```diff
 export const AI_MODELS = {
   OPENAI: "openai",
-  GEMINI: "gemini",
+  GEMINI_3_1: "gemini_3_1",
   GEMINI_3: "gemini_3",
 } as const;
```

### 3-2. 자동 DB 마이그레이션 (`server/utils/settings.ts`)
- 서버 기동 시 `getSystemSettings()` 호출 시점에 자동 실행
- `system_settings` 테이블: `supportedAiModels`, `clientDefaultModel`, `defaultAiModel` 3개 필드 변환
- `concepts` 테이블: `availableModels`, `availableAspectRatios` 변환
- 한 번 실행되면 DB가 업데이트되어 이후 재실행 안됨

### 3-3. SystemSettings.tsx 크래시 방어
- `getModelDisplayInfo`에서 알 수 없는 모델 키에 대한 fallback 반환 추가
- DB에 이전 `gemini` 값이 남아있어도 크래시하지 않음
