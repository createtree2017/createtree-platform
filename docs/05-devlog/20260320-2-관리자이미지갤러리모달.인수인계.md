# 관리자 이미지 갤러리 상세 모달 추가 인수인계

**작성일**: 2026-03-20
**순번**: 2
**작업자**: Antigravity

## 1. 작업 개요
관리자 페이지의 전체 이미지 갤러리(`/admin?menuItem=ai-create&panel=image-gallery`)에서, 이미지 클릭 시 **고해상도(1024x1024)** 이미지를 확대해서 볼 수 있는 팝업 모달 화면을 추가했습니다.
기존에 구현되지 않았던 렌더링 부분을 보완하고, 팝업 내에서 바로 다운로드나 링크 공유를 할 수 있도록 구성했습니다.

## 2. 주요 변경 사항 및 파일 목록

### [MODIFY] `client/src/components/admin/ImageGalleryAdmin.tsx`
- **Dialog 컴포넌트 적용**: `@/components/ui/dialog`에서 Dialog 모달 모듈 import.
- **UI 렌더링 부문 추가**:
  - `selectedImage` 상태를 바인딩하여 선택된 이미지 데이터를 모달로 전달.
  - 최상단 해상도 적용을 위해 이미지 태그 로드 순서를 `transformedUrl` -> `originalUrl` -> `url(thumbnail)` 순으로 적용.
  - 클릭 한 번으로 상세 이미지, 이미지 ID, 생성일 및 상태를 확인 가능.
- **모달 버튼 연동**:
  - 기존 준비되어있던 `handleDownloadClick`, `handleShare` 함수를 모달 하단 버튼들과 각각 매핑.

## 6. 긴급 핸픽스: Firebase 업로드 경로 불일치로 인한 이미지 생성 실패(403/400) 수정

### 장애 현상
이미지 생성 시 Firebase Storage 업로드는 성공하나, `/api/save-url` API에서 **403 Forbidden** 발생 → 연쇄적으로 이미지 변환 API(`/api/generate-stickers`)에서 **400 Bad Request** 발생

### 근본 원인
Firebase Custom Token 생성이 **2곳에서 서로 다른 UID 형식**으로 이루어지고 있었음:
- `server/routes/auth.ts` → `createCustomToken(String(user.id))` (결과 UID: `"24"`)
- `server/services/firebase-auth.ts` → UID를 `"`user_24`"` 형식으로 설정

프론트엔드(`firebase-upload.ts`)는 `uploads/${user.uid}/...` → `uploads/24/...`로 업로드하지만,
백엔드 검증 코드(`image.ts`)는 `uploads/user_24/...`만 허용하여 **403 권한 오류** 발생.

### 수정 내역
| 파일 | 변경 내용 |
|------|----------|
| `server/routes/image.ts` (save-url API) | storagePath 소유권 검증을 **두 가지 형식 모두 허용**하도록 수정 (`uploads/user_{userId}/` 및 `uploads/{userId}/`) |
| `server/routes/image.ts` (generate-stickers API) | Firebase 미들웨어가 `req.downloadedBuffers`에 이미지를 저장했지만, 실제 `imageBuffer` 할당 로직에서 **이를 참조하지 않아 null로 남는 버그** 수정. 파일 업로드가 없을 때 downloadedBuffers를 imageBuffer/imageBuffers로 할당하는 fallback 로직 추가. |

> **참고**: `generate-family` 라우트는 이미 downloadedBuffers를 올바르게 처리하고 있어 수정 불필요.

## 7. openai_mini 레거시 모델 잔재 처리

### 발생 원인
- 어제 테스트 중 도입했다가 삭제한 `openai_mini` 모델이 `diz_family` 컨셉의 `availableModels`에 DB에 잔존
- Zod 스키마 검증(`insertConceptSchema`)이 `openai_mini`를 허용하지 않아 해당 컨셉 수정 시 **500 에러** 발생
- UI에서는 `openai_mini` 체크박스가 없으므로 아무것도 선택 안 된 것처럼 보임

### 수정 내역
| 파일 | 변경 내용 |
|------|----------|
| `server/routes/admin-routes.ts` | 컨셉 수정 PUT API에서 `availableModels`의 알 수 없는 모델 값(레거시)을 **자동 필터링**하는 로직 추가. 필터링 후 빈 배열이면 기본값(`openai`, `gemini_3_1`)으로 설정 |

### 다음 작업자 주의사항
- `diz_family` 컨셉은 관리자 페이지에서 직접 **삭제** 처리 필요 (또는 수정 저장 시 자동 정리됨)
- 비슷한 문제가 생기면 `server/routes/admin-routes.ts`의 레거시 필터링 로직이 자동 처리함
