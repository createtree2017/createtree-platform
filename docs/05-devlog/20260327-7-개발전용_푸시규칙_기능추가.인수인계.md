## 개발 전용(슈퍼관리자 대상) 자동 푸시 규칙 기능 추가

### 1. 개요 및 변경 배경
자동 푸시 발송 시스템에서 **실제 회원들에게 푸시를 보내지 않고, 개발자(슈퍼관리자)만 알림을 받아볼 수 있는 '개발전용' 타겟팅 기능**이 필요하다는 요청에 따라 기능을 추가 개발하였습니다.
이를 통해 프로덕션 환경에서 새로운 푸시 템플릿 추가/테스트 시 안전하게 작동 여부를 확인할 수 있습니다.

### 2. 변경 파일 목록
- `client/src/components/admin/PushAutomation.tsx` (규칙 추가/수정 UI에 개발전용 필터 옵션 추가)
- `server/services/push/push.automation.service.ts` (이벤트 트리거 시 개발전용 조건 검증 및 분류 로직 고도화)

### 3. 핵심 변경 내용 (왜 변경했는가?)
- **UI 필터 항목 추가**: `PushAutomation.tsx`의 "적용 대상 필터(누구에게?)" Select 박스에 `<SelectItem value="-2">개발전용 (슈퍼관리자)</SelectItem>`를 추가하여 직관적으로 룰을 분리 생성할 수 있도록 하였습니다.
- **Rule Engine 검증 로직 격리 (`push.automation.service.ts`)**:
  - 알림의 실발송 대상(이벤트를 유발시킨 당사자)의 등급(`memberType`)을 확인하여 대상이 `superadmin`일 경우에 한해 `hospitalId === -2` 인 개발 전용 규칙을 적용 대상으로 산정합니다.
  - 일반 회원이 이벤트를 발생시킬 경우에는 조건에서 -2(개발전용)가 원천 제외되어 안전하게 공통 또는 소속 병원의 규칙만 타게 됩니다.
- **다중 룰 충돌 시 JS 기반 강제 정렬 (Priority Override)**:
  - 슈퍼관리자가 이벤트를 발생시켜 (공통 null 룰, 개발 전용 -2 룰) 복수의 규칙에 매칭될 경우, DB의 `ORDER BY` 기능 대신 서버단 JS에서 Array `sort()`를 수행하여 **어떤 상황에서든 `개발 전용(-2) 룰`이 무조건 1순위로 채택되도록 우선순위 알고리즘을 개편**했습니다.

### 4. 주의사항 및 알려진 이슈
- **동작 원리 특수성 유의**: "개발 전용" 규칙이 켜져있다고 해서 아무 병원/회원의 이벤트를 관리자가 수신하는 **브로드캐스트(Broadcast) 기능이 아닙니다**. 
  자기가 제출한 테스트 미션을 자신이 승인하거나, 슈퍼관리자 본인의 계정에서 발생한 이벤트일 때만 해당 "개발 전용" 템플릿이 입혀져 발송됩니다. (개인 테스트 용도 호환)
- 만약 모든 회원의 미션 승인 내역을 관리자가 감청(Listen)하는 기능이 필요하다면, 현재의 '대상자(userId) 치환 룰 엔진'과 별도의 브로드캐스트 라우팅 시스템이 필요합니다.

### 5. 다음 작업 참고사항
- 현재 프론트엔드 React 개발 환경의 핫리오딩(`vite`)과 달리, Express 백엔드는 `package.json` 상 `dev` 커맨드가 `--watch` 가 없는 일회성 실행 환경으로 설정되어 있습니다. 백엔드 파일 수정 후 테스트 시 반드시 서버 재시작이 요구되니 참고 바랍니다.
