# 20260327-3-푸시중복발송수정 인수인계

## 1. 개요
* **목적:** 관리자 페이지에서 FCM 푸시 발송 시 사용자 휴대폰에 2번씩 (알림 2개) 중복 도착하는 문제 수정
* **작업일:** 2026년 3월 27일

## 2. 변경 파일 목록
* `server/services/notifications.ts`
* `server/routes/push-admin-routes.ts`

## 3. 핵심 변경 내용 & 변경 배경 (Why)
* **문제 배경:** 푸시 이중 초기화 버그 수정 이후, 관리자 사이트에서 푸시를 보내면 휴대폰에 একই 알림이 2번 오고 DB 로그에도 0.15초 간격으로 `specific_users`와 `specific_user` 2건이 기록되는 중복 문제가 확인되었습니다.
* **원인:** 다른 개발자분의 지적대로 라우터 흐름상 2번의 FCM 호출이 연속으로 발생했습니다.
  1. `push-admin-routes.ts`에서 대량 발송용 `sendToUsers` 호출로 1차 발송
  2. 직후 알림함에 기록을 남기기 위해 `createNotification`을 순회 호출했는데, 해당 함수 내부 스펙상 '알림함 저장 + FCM 개별 발송(`sendToUser`)'이 묶여있어 여기서 2차 발송
* **조치 내용:** 
  1. `notifications.ts`의 `createNotification` 인자에 `skipPush?: boolean` 옵션을 추가하고, `true`일 경우 내장된 FCM 발송을 무시하고 DB 저장만 하도록 수정했습니다.
  2. `push-admin-routes.ts`에서 `createNotification`을 호출할 때 `skipPush: true` 값을 넘겨주도록 처리했습니다. 이제 관리자 대량 발송 로직의 고효율성은 유지하면서 개별 알림함의 FCM 중복 발송을 완벽히 차단했습니다.
