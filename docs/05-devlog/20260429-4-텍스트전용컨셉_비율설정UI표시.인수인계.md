# 텍스트 전용 컨셉 비율 설정 UI 표시 인수인계

작성일: 2026-04-29

## 1. 배경

텍스트 전용 컨셉(`generationType = text_only`)에서도 백엔드는 `generationSettings.aspectRatios`를 읽어 이미지 생성 비율로 적용한다.

하지만 관리자 컨셉 수정 UI에서는 비율 설정 영역이 `image_upload` 컨셉에서만 보이도록 제한되어 있었다. 이 때문에 `cute_small_stiker`처럼 기존 DB에 `16:9`가 저장된 텍스트 전용 컨셉은 관리자가 화면에서 비율을 확인하거나 `자동`으로 바꿀 수 없었다.

## 2. 확인한 원인

- `cute_small_stiker` DB 값:
  - `available_aspect_ratios.gemini_3_1 = ["16:9"]`
  - `generation_settings.gemini_3_1.aspectRatios = ["16:9"]`
- 서버 로그의 `settings: { aspectRatios: ["16:9"], imageSize: "1K" }`는 위 DB 설정이 정상 반영된 결과였다.
- UI에서는 `newConcept.generationType === "image_upload"` 조건 때문에 텍스트 전용 컨셉에서 비율 설정이 숨겨져 있었다.

## 3. 변경 내용

### `client/src/components/modal/admin/ConceptFormModal.tsx`

- 생성 방식 라디오 버튼을 먼저 표시하도록 배치했다.
  - 이미지 첨부 생성
  - 프롬프트로 생성
- 모델 선택 영역과 생성 비율 설정 영역을 두 생성 방식 모두에서 보이도록 변경했다.
- 비율 설정 섹션명을 `이미지 비율 설정`에서 `생성 비율 설정`으로 변경했다.
- 기존처럼 모델별 capability registry 기반 비율 옵션을 사용한다.
- 기존 저장값이 있으면 그대로 선택 상태로 표시된다.
  - 예: `16:9`로 저장된 컨셉은 `16:9`가 선택되어 보인다.
  - 관리자가 `자동` 또는 다른 비율로 변경 후 저장하면 그 값이 `generationSettings`에 반영된다.

## 5. 주의사항

- 이번 작업은 UI 표시/저장 경로를 여는 변경이며, 기존 DB 값을 자동으로 바꾸지는 않았다.
- 현재 `cute_small_stiker`는 여전히 DB에 `16:9`가 저장되어 있으므로, 관리자 페이지에서 컨셉 수정 후 `자동` 또는 원하는 비율로 저장해야 실제 생성 비율이 바뀐다.
- 실제 브라우저 화면 검증은 `!!테스트!!` 지시가 없어 수행하지 않았다.
