# 배경제거 서비스 분리 및 의존성 정리 인수인계

작성일: 2026-05-06

## 1. 작업 개요

Railway 배포 중 `@imgly/background-removal-node` 내부 `sharp/libvips` 설치 무결성 오류가 반복되어, 배경제거 AI 런타임을 메인 Node.js 서버에서 분리했다. 메인 서버는 HTTP provider를 통해 별도 `rembg` 서비스에 요청하도록 바꾸고, 사용하지 않는 Hugging Face 의존성은 제거했다.

## 3. 핵심 변경 내용

- `package.json`에 Node 버전을 `20.x`로 고정했다.
- 새 Python 서비스 도입에 맞춰 `__pycache__/`, `*.pyc`를 git ignore 대상으로 추가했다.
- `@imgly/background-removal-node`와 `@huggingface/transformers`를 production dependency에서 제거했다.
- 기존 `server/services/backgroundRemoval.ts`는 외부 호출 호환성을 유지하는 facade로 축소했다.
- 배경제거 구현을 `server/services/background-removal/` 아래로 분리했다.
- 메인 서버는 `BACKGROUND_REMOVAL_SERVICE_URL`로 지정된 별도 rembg HTTP 서비스에 이미지를 전송하고 결과 PNG를 받는다.
- 기존 관리자 설정의 `bgRemovalQuality`, `bgRemovalModel`, `foreground/background` 출력 타입은 유지했다.
- 이미지 자동맞춤의 자식 프로세스 worker 방식은 제거하고 공통 배경제거 서비스 함수를 사용하도록 정리했다.
- `/api/background-removal/health` 엔드포인트를 추가해 배경제거 서비스 연결 상태를 확인할 수 있게 했다.
- 별도 배포 가능한 FastAPI 기반 `services/rembg-service`를 추가했다.

## 4. 변경 이유

`@imgly/background-removal-node`는 내부적으로 오래된 native dependency를 포함하고 있어 Railway 빌드 환경과 캐시 상태에 따라 `sharp/libvips` 설치 실패가 반복될 수 있었다. 메인 Node.js 서버에서 해당 런타임을 제거하면 메인 서비스 배포 안정성이 올라가고, 배경제거 모델은 별도 서비스에서 독립적으로 관리할 수 있다.

`@huggingface/transformers`는 현재 코드 import 사용처가 없어 배포 무게와 설치 실패 가능성만 높이는 상태였으므로 제거했다.

## 5. Railway 배포 필요 작업

메인 서비스와 별도로 Railway에 배경제거 전용 서비스를 하나 추가해야 한다.

1. 같은 GitHub repo로 새 Railway 서비스를 만든다.
2. 새 서비스 root directory를 `services/rembg-service`로 지정한다.
3. Dockerfile 기반으로 배포한다.
4. 배포 URL을 메인 `createtree-platform` 서비스 Variables에 등록한다.

```text
BACKGROUND_REMOVAL_SERVICE_URL=https://your-rembg-service.up.railway.app
```

필요 시 모델 매핑은 메인 서비스 Variables에서 조정할 수 있다.

```text
BACKGROUND_REMOVAL_MODEL_SMALL=isnet-general-use
BACKGROUND_REMOVAL_MODEL_MEDIUM=birefnet-general
BACKGROUND_REMOVAL_TIMEOUT_MS=60000
```

## 6. 동작 확인 상태

- `npm install --package-lock-only` 성공
- `npm prune` 성공
- `npm ls @huggingface/transformers @imgly/background-removal-node onnxruntime-node --depth=0` 결과 의존성 없음 확인
- `python -m py_compile .\services\rembg-service\main.py` 성공
- `npm run build` 성공
- `npx tsc --pretty false --noEmit` 전체 검사는 기존 보류 오류가 남아 실패
  - `client/src/components/admin/PersonaManagement.tsx`
  - `client/src/components/admin/PersonaCategoryManagement.tsx`
  - `server/services/milestones.ts`
- 이번 배경제거 변경 파일명 기준 필터 검사에서는 관련 TypeScript 오류가 출력되지 않았다.

## 7. 주의사항

- 로컬 Node 버전이 `24.15.0`이라 `package.json`의 `node: 20.x`와 달라 npm 명령에서 `EBADENGINE` warning이 출력된다. Railway 배포 안정성을 위한 의도된 고정이다.
- Windows 파일 잠금 때문에 로컬 `node_modules/@imgly` 아래 임시 폴더가 남을 수 있다. `package.json`, `package-lock.json`, npm dependency tree에서는 제거된 상태이며 배포에는 포함되지 않는다.
- 기존 `NO_CACHE=1` 변수는 캐시 우회용 임시 설정이므로 배포 안정화 후 제거하는 것이 좋다.
- `git add`, `git commit`, `git push`는 수행하지 않았다.
