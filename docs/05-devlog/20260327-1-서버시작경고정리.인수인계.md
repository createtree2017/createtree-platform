## 핵심 변경 내용

### 1. `.env` — GEMINI_API_KEY 주석처리

- **왜**: `GOOGLE_API_KEY`와 `GEMINI_API_KEY`가 동시에 설정되어 `Both GOOGLE_API_KEY and GEMINI_API_KEY are set` 경고 발생
- **조치**: `GEMINI_API_KEY` 주석처리 (값은 보존, 필요시 복원 가능)
- **결과**: 경고 제거됨

### 2. `.env` — GOOGLE_CLOUD_STORAGE_BUCKET 추가

- **왜**: 환경변수 누락으로 `⚠️ GOOGLE_CLOUD_STORAGE_BUCKET 환경변수 누락, 기본값 사용` 경고 발생
- **조치**: `GOOGLE_CLOUD_STORAGE_BUCKET=createtree-upload` 명시적 추가
- **결과**: 경고 제거됨

### 3. `server/firebase.ts` — Private Key 디버그 로그 제거

- **왜**: 서비스 계정 Private Key 일부가 콘솔에 출력되어 **보안 위험** (로그가 외부 노출 시 키 유출 가능)
- **조치**: 키 내용이 출력되는 `console.log` 6줄 삭제, 에러 로그는 유지
- **결과**: 이제 `✅ Private key 처리 완료`만 출력

### 4. Sentry ESM `--import` 구조 수정

- **왜**: Sentry v8+에서 Express 자동 계측(auto-instrumentation)이 되려면 `--import` 플래그로 앱 코드보다 먼저 로드해야 함
- **기존**: `index.ts`에서 `import './instrument'` → Sentry가 Express를 감지하지 못함
- **변경**:
  - `package.json` dev 스크립트: `tsx --import ./server/instrument.ts server/index.ts`
  - `index.ts`: 정적 import 제거, 프로덕션 빌드용 조건부 dynamic import 유지
- **결과**: `[Sentry] express is not instrumented` 경고 제거됨

## 주의사항

- 프로덕션 빌드(`npm run build`)에서는 esbuild가 `server/index.ts`를 번들링하므로, `index.ts` 내 `if (process.env.NODE_ENV === 'production') await import('./instrument.js')` 로직이 프로덕션 Sentry 초기화를 담당
- 프로덕션 start 스크립트(`node dist/start.js`)에도 `--import` 적용을 고려할 수 있으나, 현재 조건부 import로 동작 중
