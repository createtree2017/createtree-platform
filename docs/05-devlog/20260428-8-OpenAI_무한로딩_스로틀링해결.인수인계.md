## 3. 핵심 변경 내용
- `node-fetch`와 `form-data`를 사용하여 OpenAI API(`v1/images/edits`)로 전송할 때 발생하는 극심한 성능 저하(4~6분 소요) 문제를 해결했습니다.
- **변경 배경/의사결정 이유:** 
  - `form-data` 모듈 사용 시 `Content-Length` 헤더를 명시적으로 보내지 않으면, `node-fetch`가 'Chunked Transfer-Encoding(청크 방식)'으로 데이터를 전송합니다.
  - OpenAI 측의 보안 방화벽(Cloudflare WAF 등)은 이러한 청크 방식의 파일 업로드를 악의적 공격(Slowloris)으로 간주하거나 극도로 제한하여 처리(스로틀링)합니다.
  - 이로 인해 1~2MB에 불과한 이미지가 업로드되는 데 5분 이상이 소요되는 치명적인 병목이 발생했습니다.
- **수정 사항:**
  - `formData.getLength()`를 사용해 전송할 폼 데이터의 정확한 바이트 길이를 계산.
  - `fetch` 옵션 헤더에 `...formData.getHeaders()` 및 `'Content-Length': contentLength.toString()`를 명시적으로 추가하여 단일 요청으로 고속 처리되도록 수정.

## 4. 동작 확인 상태
- ✅ 파일 크기 계산 로직 정상 적용
- ✅ API 요청 헤더에 Boundary 및 Content-Length 정상 주입

## 5. 주의사항 / 알려진 이슈
- **Node.js의 `fetch` 규격 차이:** 프론트엔드의 `fetch`와 달리 Node.js에서는 `form-data` 라이브러리를 쓸 때 헤더 처리에 각별한 주의가 필요합니다. 추후 다른 외부 API에 파일을 전송할 때도 무조건 `Content-Length`를 수동으로 계산해서 넣어주어야 스로틀링을 피할 수 있습니다.
