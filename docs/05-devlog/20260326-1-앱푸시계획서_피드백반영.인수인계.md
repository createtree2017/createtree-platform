# 앱 푸시 시스템 계획서 피드백 반영

## 변경 파일

## 핵심 변경 내용

다른 개발자 피드백 11개 항목을 코드베이스와 대조 검증한 후, 유효한 8개 항목을 계획서에 반영:

### 반영된 항목 (8개)

1. **`push_delivery_logs`에 `trigger_type` 컬럼 추가** — 시스템 자동 발송(admin_id=null)의 원인 구분용
2. **`sendToUser()` 토큰 0개 처리 명시** — 앱 미설치/푸시 거부 유저에게는 no-op (에러 아님)
3. **`firebase-admin` 초기화 `admin.app()` 방어 패턴** — 동시 요청 시 이중 초기화 방지
4. **`notifications.userId` text↔integer 타입 변환** — `String(approved.userId)` 필수
5. **`shouldSendNotification()` switch에 mission_approved/rejected case 추가**
6. **`bulkUpdateStatus()` FCM 트리거 추가** — 일괄 승인/반려 시에도 푸시 발송 보장
7. **회원 탈퇴 시 `deactivateAllUserTokens()` 즉시 호출** — GDPR/개인정보 보호
8. **iOS 포그라운드 이중 알림의 전제 조건 명시 + 관리자 UI에 발송가능 기기수 표시**

### 기각된 항목 (2개 — 사실 오류)

1. "`userDevices` 테이블이 schema에 없다" → schema.ts **2244줄에 존재** (파일 끝부분이라 누락)
2. "`users.hospitalId` 없다" → **29줄에 존재**
