# 2026-05-24 문화센터 리스트 선착순 대기자 수 표시 인수인계

## 작업 개요

문화센터 미션 상세 페이지에서 이미 표시되던 선착순 대기자 수를 `/missions` 리스트 카드에도 동일한 정책으로 노출했다. 정원이 찬 선착순 미션에 대기자가 있는 경우 리스트의 모집인원 영역에 `1 / 1명 (대기 1명)` 형태로 표시한다.

## 핵심 변경 내용

- 리스트 API 응답에 `waitlistCount`를 추가했다.
- 선착순 미션(`isFirstCome`)에서만 신청 세부미션의 `WAITLIST` 제출 수를 집계한다.
- 대기자 집계는 중복 제출 데이터에 안전하도록 `count(DISTINCT userId)` 기준으로 계산한다.
- 선착순 미션의 `currentApplicants`는 상세 페이지와 맞춰 `APPROVED` 신청자만 계산한다.
- 선착순이 아닌 미션의 `currentApplicants`는 기존처럼 `SUBMITTED`, `APPROVED` 기준을 유지한다.
- 리스트 카드에서는 `mission.isFirstCome && mission.waitlistCount > 0`일 때만 `(대기 N명)`을 표시한다.
- 리스트 카드의 참여 방식은 `누구나 참여`, `선정`, `선착순` 뱃지로 구분한다. 정원 있는 미션은 모집인원 행 안에 `모집인원 | [선정/선착순]` 형태로 표시하고, 정원 없는 미션은 `누구나 참여` 뱃지를 단독 표시한다.
- 진행바는 기존 정책대로 `currentApplicants / capacity`만 사용하며 대기자 수는 진행바 비율에 포함하지 않는다.
- 좁은 화면에서 모집인원과 대기자 텍스트가 겹치지 않도록 모집인원 행에 wrapping 처리를 추가했다.
