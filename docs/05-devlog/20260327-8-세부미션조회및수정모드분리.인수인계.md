## 작업 개요
세부미션 제출 UI에서 조회(View) 모드와 수정(Edit) 모드를 명확히 분리하여 사용자 경험을 대폭 개선했습니다.

## 핵심 변경 내용
- **하단 네비게이션 아이콘**: 세부미션 탭에서 사용자가 제출을 완료했으나 검수 중(`submitted` 또는 `pending`)인 상태일 경우, 아이콘 우측 상단에 파란색 배지(Send 아이콘)를 추가하고 텍스트를 파란색(`text-blue-500`)으로 표기하여 검수 진행 중임을 직관적으로 표시했습니다.
- **제출 폼 모달(SubmissionForm)**
  - `isEditMode` 상태(초기값: 제출 이력이 없으면 `true`, 있으면 `false`)를 도입하여 제출 이력이 있는 미션을 클릭하면 기본적으로 View 모드로 진입하게 했습니다.
  - View 모드(`!isEditMode`) 동작 시 모든 파일 업로드 버튼 및 링크/텍스트/평점 입력 필드에 `disabled={!isEditMode}`를 적용해 입력을 차단하고 업로드 UI의 일부 버튼(디바이스/갤러리 선택 등)도 비활성화했습니다.
  - View 모드 하단 버튼 레이아웃은 **[제출완료 - 검수중...]** (너비를 넓게 차지하는 파란색, 클릭 불가) 버튼과 **[수정]** (우측 24px/회색, 모드 전환 버튼)으로 분할 제공되도록 구현했습니다.
  - [수정] 버튼 클릭 시 `isEditMode = true` 상태로 전환되어 기존의 보라색 수정 완료 제출 폼이 활성화됩니다.
  - 승인 이후(`isLocked===true`)의 상태에서는 최상단에서 이미 고정된 승인 컴포넌트로 대체 렌더링되므로, 이번에 추가한 `isEditMode` 뷰어 분기점 이전에 차단되어 안전합니다.

## 동작 확인 상태
- ✅ 제출 완료 후 검수 대기 중 상태에서 하단 탭 파란색 배지가 정상 작동함
- ✅ 제출된 미션 클릭 시 첫 화면이 읽기 전용 뷰로 나타나며 입력이 방지됨
- ✅ View 모드 하단의 [수정] 버튼 클릭 시 Edit 모드로 매끄럽게 전환됨

## 주의사항 / 알려진 이슈
- 컴포넌트 내부 State(`isEditMode`)를 통해 가볍게 모드를 제어하므로, 사용자가 수정 모드에서 링크나 텍스트를 고치다가 뒤로가기 혹은 닫기로 모달을 종료한 뒤 다시 들어오면 서버 데이터를 기준으로 다시 View 모드로 초기화됩니다. (오히려 안전한 방향입니다.)
- [다시 제작하기] 등의 동작을 통해 외부 주소(파티 템플릿 등)를 다녀오면 컴포넌트가 다시 마운트되므로 View 모드로 초기 시작할 수 있으나, 드래프트 복원 로직이 작동하여 작업물은 유지됩니다.

## 다음 작업 참고사항
