# 전화번호 하이픈 표시 개선

작성일: 2026-05-14

## 1. 작업 개요

회원가입과 고객/관리자 화면에서 전화번호가 `01012345678`처럼 붙어 보여 불편한 문제를 개선했다. 저장 데이터는 기존처럼 숫자만 유지하고, 입력/표시 UI에서만 하이픈이 보이도록 정리했다.

## 4. 핵심 변경 내용

- `formatPhoneNumber()`와 `normalizePhoneNumberInput()` 공통 유틸을 추가했다.
- 회원가입, 프로필 완성, 계정 설정, 아이디 찾기 입력값은 내부적으로 숫자만 유지하되 화면에는 자동 하이픈을 표시한다.
- 관리자 회원 목록/수정, 사용자 상세, 푸시 대상 선택, 미션 검수 목록/상세의 전화번호 표시를 하이픈 포함 형태로 변경했다.
- 11자리는 `010-1234-5678`, 10자리는 `010-123-4567`로 표시한다.

## 5. 왜 변경했는지

DB 값을 하이픈 포함 문자열로 바꾸면 기존 검색, 중복 확인, 아이디 찾기, 서버 검증 흐름에 부작용이 생길 수 있다. 그래서 저장 형식은 유지하고 UI 표시와 입력 경험만 개선하는 방식으로 안정성을 확보했다.

## 6. 동작 확인 상태

- `npm run build`: 성공
- `npx tsc --pretty false --noEmit` 변경 파일명 필터: 관련 신규 오류 없음
- 전체 `tsc`: 기존 Persona/마일스톤 타입 오류가 남아 있어 전체 통과는 아님
- `!!테스트!!` 키워드가 없어 브라우저 에이전트 검증은 수행하지 않았다.

## 7. 주의사항

- 서버 DB에는 계속 숫자만 저장된다.
- 브라우저에서 실제 커서 위치 UX까지 확인하려면 `!!테스트!!`로 회원가입/관리자 화면 검증을 수행해야 한다.

## 8. Skill Impact Check

- 전화번호 포맷 유틸과 입력/표시 패턴이 추가됐지만, 프로젝트 운영 정책이나 스킬 자동화 기준 변경은 아니다.
- 스킬 문서 업데이트는 불필요하다고 판단했다.
