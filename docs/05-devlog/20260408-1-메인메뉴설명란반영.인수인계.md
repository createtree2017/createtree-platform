# 20260408-1 메인 메뉴 설명란 반영 변경 인수인계

## 1. 개요
* 관리자 페이지의 `메뉴관리` 에서 메인 메뉴의 설명 텍스트를 수정하면 프론트엔드 유저 화면(ex: 문화센터, 나의미션)의 타이틀 하단 안내 문구가 동적으로 변경되는 기능

## 2. 핵심 변경 내용
* **DB**: `mainMenus` 스키마에 `description` 텍스트 필드를 추가하고 마이그레이션 진행
* **API**: `/api/admin/main-menus` 라우트(생성, 수정)에서 `description` 필드 핸들링
* **Admin**: `MenuManagement.tsx`의 관리 패널 및 수정 다이얼로그에 관련 폼 필드 추가
* **Client**: `useMainMenus.ts`에 필드 확장 후, `missions.tsx` 및 `mymissions.tsx`의 하드코딩 텍스트를 API 응답 기반 텍스트로 치환

## 3. 변경된 파일 목록
* `shared/schema.ts`
* `server/routes/admin-routes.ts`
* `client/src/hooks/useMainMenus.ts`
* `client/src/components/admin/MenuManagement.tsx`
* `client/src/pages/mymissions.tsx`
* `client/src/pages/missions.tsx`

## 4. 동작 확인 상태
* ✅ 컬럼 마이그레이션 (`npx drizzle-kit push`) 완료
* ✅ 관리자 페이지 메뉴 설정창 및 수정 폼에 텍스트 렌더링 확인 완료
* ✅ 유저 화면단 연동 확인 완료

## 5. 주의사항 및 알려진 이슈
* 기본적으로 하드코딩된 fallback 기능들이 `mymissions.tsx`, `missions.tsx` 안쪽 혹은 `useMainMenus.ts` 내에 남아있기 때문에, 관리자가 빈 텍스트를 입력하면 최초 설정된 기본문구가 표출됩니다.
