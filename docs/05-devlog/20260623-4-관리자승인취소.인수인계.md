# 2026-06-23 관리자 승인 취소 인수인계

## 작업 개요

관리자 검수 화면에서 이미 승인된 제출을 다시 검수 대기 상태로 되돌릴 수 있는 `승인 취소` 기능을 추가했다. 사용자 신청 취소와 구분하기 위해 `cancelled` 상태는 사용하지 않고 `approved -> submitted` 전환으로 처리한다.

## 핵심 변경 내용

### 승인 취소 전용 API

`POST /api/admin/review/submissions/:submissionId/cancel-approval` 라우트를 추가했다.

서비스는 `cancelApprovalSubmission` 전용 메서드에서 다음을 처리한다.

- `approved` 상태만 승인 취소 허용
- 승인 취소 시 `status=submitted`
- `isLocked=false`
- `reviewedBy=null`
- `reviewedAt=null`
- `reviewNotes`에 `승인 취소: 관리자가 승인 상태를 검수 대기로 되돌렸습니다.` 흔적 추가
- 병원 관리자는 공개 미션 또는 본인 병원 미션만 처리 가능
- 승인 취소 대상 사용자 큰미션 진행률 재계산

기존 승인/보류 API는 수정하지 않았다.

### 선착순 대기자 자동승격

승인 취소 대상이 `신청` 타입 세부미션이고 주제미션이 `isFirstCome=true`이면, 같은 세부미션의 가장 빠른 `waitlist` 제출을 `approved`로 자동승격한다.

- 자동승격 제출은 `isLocked=true`
- 기존 `대기자 자동승격` reviewNotes 문구를 재사용
- 자동승격된 사용자도 큰미션 진행률 재계산
- 같은 세부미션 기준 advisory transaction lock을 사용해 승인 취소와 승격을 한 트랜잭션에서 처리

### 관리자 모달 UI

`SubmissionDetailModal`에서 상태별 버튼 노출을 분리했다.

- `submitted`: 기존처럼 `보류`, `승인` 버튼 표시
- `approved`: `승인 취소` 버튼만 표시
- 승인 취소 클릭 시 `window.confirm` 확인 후 API 호출

`MissionManagement.tsx`에는 승인 취소 mutation을 추가하고, 성공 시 검수 목록/통계/주제미션 쿼리를 invalidate한다.

## 변경 이유

운영 중 승인 실수나 일정 변경이 발생했을 때 기존 구조는 승인된 제출을 되돌릴 방법이 없었다. 기존 `cancelled`는 사용자 신청 취소 의미이므로 관리자 운영 취소에 쓰면 상태 의미가 섞인다. 따라서 검수 대기로 되돌리는 전용 기능이 필요했다.

## 주의사항 / 알려진 이슈

- 승인 취소는 기존 승인 이력을 별도 테이블로 보존하지 않고 `reviewNotes`에 흔적을 남긴다.
- 승인 취소된 제출은 다시 `검수 대기`로 돌아가므로, 관리자가 재승인하면 기존 수동 승인 정책처럼 모집인원 초과 승인이 가능하다.
- 선착순 미션에서 승인 취소 시 대기자가 자동승격되므로, 운영자가 단순히 자리를 비워두고 싶다면 이후 자동승격된 건을 별도로 검토해야 한다.
- 기존 사용자 참여취소 로직은 변경하지 않았다.

## 수동 테스트 순서

1. `localhost:5000/api/dev/auto-login` 접속 후 관리자 계정으로 로그인한다.
2. 관리자 > 검수 대기 화면에서 상태 필터를 `승인`으로 바꾼다.
3. 승인된 제출의 `검수` 버튼을 눌러 상세 모달을 연다.
4. `승인 취소` 버튼만 보이고 `승인`/`보류` 버튼은 보이지 않는지 확인한다.
5. `승인 취소`를 클릭하고 확인창에서 확인한다.
6. 제출이 `검수 대기(submitted)` 상태로 돌아가는지 확인한다.
7. 선착순 미션에서 대기자가 있는 승인 제출을 승인 취소했을 때 가장 빠른 대기자가 승인되고 `대기자 자동승격` badge가 보이는지 확인한다.
