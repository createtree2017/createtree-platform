# Android 강제 인앱 업데이트 인수인계

작성일: 2026-05-02

## 1. 목표

웹 연동형 Android 앱의 장점은 유지하되, Android 네이티브 기능 변경이 필요한 릴리스에서는 사용자가 오래된 앱으로 진입하지 못하도록 한다.

목표 UX는 다음과 같다.

1. 사용자가 Android 앱을 실행한다.
2. Google Play에 새 앱 버전이 있으면 앱 진입 전에 "업데이트가 필요합니다" 안내를 표시한다.
3. 사용자가 확인을 누르면 Google Play 인앱 업데이트 화면으로 이동한다.
4. 업데이트를 완료한 뒤 앱에 진입한다.
5. 앱 진입 후 웹 Service Worker 업데이트/새로고침이 발생해도 정상 흐름으로 허용한다.

## 2. 구현 계획

- Android 네이티브에 Google Play In-App Updates `IMMEDIATE` 플로우를 추가한다.
- 앱 시작 시 `AppUpdateManager.getAppUpdateInfo()`로 업데이트 여부를 확인한다.
- 업데이트 가능하고 `IMMEDIATE` 타입이 허용되면, 앱 자체 안내 다이얼로그를 먼저 띄운 뒤 Google Play 업데이트 플로우를 시작한다.
- 사용자가 업데이트를 취소하거나 실패하면 앱 사용을 허용하지 않고 안내 다이얼로그를 다시 표시한다.
- 업데이트가 이미 진행 중인 상태(`DEVELOPER_TRIGGERED_UPDATE_IN_PROGRESS`)에서는 `onResume()`에서 업데이트 플로우를 재개한다.
- Play Store 외부 설치/개발 빌드처럼 인앱 업데이트 API가 동작하지 않는 환경에서는 앱 진입을 막지 않는다.
- 새 릴리스 기준 버전은 `versionCode 10`, `versionName "2.0"`으로 갱신한다.

## 3. 참고 기준

- Google Play In-App Updates 공식 문서의 Java/Kotlin 흐름을 기준으로 구현한다.
- 현재 앱은 Capacitor WebView 앱이므로 웹 자동 업데이트는 유지하고, Android 시스템 업데이트만 네이티브에서 강제한다.

## 4. 작업 결과

- `android/app/build.gradle`
  - `versionCode 10`, `versionName "2.0"`으로 갱신했다.
  - Google Play In-App Updates 의존성 `com.google.android.play:app-update:2.1.0`을 추가했다.
- `android/app/src/main/java/com/createtree/app/MainActivity.java`
  - `AppUpdateManagerFactory.create(this)`로 Google Play 인앱 업데이트 매니저를 초기화한다.
  - 앱 시작 시 `checkForMandatoryUpdate()`를 실행해 새 앱 버전 여부를 확인한다.
  - `UPDATE_AVAILABLE`이고 `AppUpdateType.IMMEDIATE`가 허용되면 앱 자체 안내 다이얼로그를 먼저 표시한다.
  - 사용자가 "업데이트"를 누르면 `startUpdateFlowForResult(..., AppUpdateOptions.newBuilder(AppUpdateType.IMMEDIATE).build())`로 Google Play 즉시 업데이트 플로우를 시작한다.
  - 사용자가 업데이트를 취소하거나 실패하면 "다시 업데이트" / "앱 종료" 다이얼로그를 표시해 앱 사용을 계속 허용하지 않는다.
  - `onResume()`에서 `DEVELOPER_TRIGGERED_UPDATE_IN_PROGRESS` 상태를 확인해 중단된 즉시 업데이트 플로우를 재개한다.
  - 기존 Android 갤러리 저장 브리지(`CreateTreeAndroid`) 등록은 유지했다.

## 5. 검증

- `npm run build`
  - 통과.
  - Vite 빌드 및 서버 esbuild 번들 완료.
- `android/gradlew.bat :app:compileDebugJavaWithJavac`
  - Android Studio 내장 JDK 21(`C:\Program Files\Android\Android Studio\jbr`)을 명시해 실행.
  - 통과.
  - 새 `app-update` 의존성 및 `MainActivity` Java 컴파일 확인 완료.

## 6. 주의사항

- Google Play In-App Updates는 Play Store를 통해 설치/배포된 앱에서 의미 있게 동작한다.
- 로컬 APK 직접 설치나 Android Studio 실행에서는 업데이트 없음/불가 상태로 처리될 수 있다.
- 현재 1.9 앱에는 인앱 업데이트 코드가 없으므로, 1.9 사용자가 2.0 업데이트를 앱 내부에서 강제 안내받지는 못한다.
- 2.0 설치 이후부터 다음 네이티브 앱 업데이트가 배포될 때 강제 업데이트 흐름이 동작한다.
- 웹 업데이트 자동 새로고침은 유지했다. 앱 시스템 업데이트가 없을 때는 기존 웹 연동 앱 장점이 그대로 유지된다.
- `!!푸시!!` 지시가 없었으므로 git add/commit/push는 수행하지 않는다.
