# 검수 대기 목록 복귀 UI 개선

작성일: 2026-05-11

## 1. 작업 개요

관리자 페이지의 `미션 시스템 관리 > 검수 대기`에서 특정 주제 미션을 선택한 뒤 검수 대기 전체 목록으로 돌아가기 어려운 문제를 개선했다.

## 2. 연결된 PDCA 문서

- 없음: 기존 검수 대시보드의 단일 컴포넌트 내비게이션 UI 개선으로 판단해 별도 Plan/Design 없이 처리했다.

## 3. 변경 파일

- `client/src/components/admin/MissionManagement.tsx`

## 4. 핵심 변경 내용

- 검수 대시보드 하위 화면에서 `목록으로` 버튼을 표시하도록 추가했다.
- `목록으로` 버튼은 기존 `navigateToThemeMissions()` 흐름을 사용해 선택된 미션과 세부미션 상태를 해제하고 URL의 `mission`, `submission` 파라미터도 정리한다.
- 브레드크럼의 `검수 대시보드` 항목도 하위 화면에서는 클릭 가능한 버튼으로 유지해 같은 복귀 동작을 수행하도록 정리했다.
- 루트 목록 화면에서는 `검수 대시보드`를 현재 위치 텍스트로 표시해 불필요한 클릭 affordance를 줄였다.

## 5. 변경 이유

기존에는 URL에 `mission=...` 파라미터가 있는 상태에서 브레드크럼의 `검수 대시보드`가 클릭 불가능한 텍스트로 렌더링되었다. 이 때문에 운영자가 특정 미션의 세부미션 검수 화면에 들어간 뒤 전체 검수 대기 리스트로 돌아가려면 브라우저 뒤로가기 등에 의존해야 했다.

운영 흐름상 검수 대기 목록과 특정 미션 검수 화면을 반복해서 오갈 가능성이 높으므로, 명시적인 복귀 버튼과 클릭 가능한 브레드크럼을 함께 제공하는 방식으로 접근성을 높였다.

## 6. 동작 확인 상태

- `npm run build` 성공
- `npm run check` 실패: 기존 `PersonaCategoryManagement.tsx`, `PersonaManagement.tsx`, `server/services/milestones.ts` 타입 오류로 실패했다. 이번 변경 파일 관련 신규 타입 오류는 빌드에서 확인되지 않았다.
- 브라우저 검증은 사용자 요청에 `!!테스트!!`가 포함되지 않아 자동 수행하지 않았다.

## 7. 주의사항 / 알려진 이슈

- `목록으로`는 항상 검수 대기 전체 주제 미션 목록으로 이동한다. 제출 목록에서 직전 세부미션 목록으로만 돌아가고 싶은 경우에는 브레드크럼의 미션명을 사용할 수 있다.
- 기존 타입 체크 실패 항목은 이번 작업 범위 밖의 기존 오류다.

## 8. 수동 테스트 참고

1. `localhost:5000/api/dev/auto-login` 접속
2. 관리자 페이지 → 메뉴관리 → 문화센터 → 미션 시스템 관리 → 검수 대기 이동
3. `[5월 16일] 캔들만들기 행사` 같은 주제 미션 클릭
4. 우측 `목록으로` 버튼 클릭
5. 검수 대기 전체 주제 미션 목록으로 돌아오고 URL에서 `mission` 파라미터가 제거되는지 확인
