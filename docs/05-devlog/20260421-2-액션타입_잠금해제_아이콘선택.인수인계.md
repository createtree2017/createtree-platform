# 액션 타입 잠금 해제 + 아이콘 선택 기능 추가

## 변경 배경
문화센터 미션의 액션 타입(신청, 제출, 출석, 리뷰) 4개가 `isSystem=true`로 잠겨 있어 관리자가 수정/삭제할 수 없었음.
또한 새로운 액션 타입을 생성해도 아이콘이 하드코딩되어 있어 기본 원(○) 아이콘만 표시되는 문제가 있었음.

**사용자 요청**: 
- "신청"만 시스템 보호 유지, 나머지(제출, 출석, 리뷰)는 잠금 해제
- 액션 타입 생성/수정 시 아이콘 선택 기능 추가

## 변경 파일 목록

### 1. `server/services/mission/action.type.service.ts`
- **44행**: `isSystem` 기반 이름 변경 차단 → `name === "신청"` 기반으로 변경
- **67행**: `isSystem` 기반 삭제 차단 → `name === "신청"` 기반으로 변경
- **결과**: 제출, 출석, 리뷰 등 기존 시스템 타입도 이름 변경/삭제 가능

### 2. `client/src/components/modal/admin/ActionTypeModal.tsx` (전면 재작성)
- Lucide 아이콘 50개를 한글 태그와 함께 정의 (예: "카메라" → Camera)
- **아이콘 검색기**: 한글/영문 모두 검색 가능 (예: "사진", "촬영", "Camera")
- **아이콘 그리드**: 6열 그리드, 스크롤 가능, 선택 시 보라색 하이라이트
- **미리보기**: 선택된 아이콘을 상단에 크게 표시 + "해제" 버튼
- `iconUrl` 필드 추가 (Lucide 아이콘 이름을 DB `icon_url` 컬럼에 저장)

### 3. `client/src/components/admin/MissionManagement.tsx`
- `LUCIDE_ICON_OPTIONS` 배열 + `getLucideIcon()` 헬퍼 함수 추가
- 테이블 컬럼에 **아이콘 미리보기** 추가
- "시스템 여부" 컬럼 → "신청"만 `시스템` 배지 표시
- `disabled` 조건: `isSystem` → `name === "신청"` 으로 변경

### 4. `client/src/pages/mission-detail.tsx`
- `SubMission` 인터페이스: `actionType.iconUrl` 필드 추가
- `getActionTypeTabIcon()`: DB `iconUrl` 우선 사용 → 없으면 기존 하드코딩 fallback
- `dynamicTabs` 생성 시 `iconUrl` 전달
- `LucideAllIcons` wildcard import 추가

## 동작 확인 상태
- ✅ TypeScript 컴파일 에러 없음 (수정 파일 기준)
- ⬜ 수동 테스트 필요 (관리자 페이지 액션타입 관리)

## 주의사항 / 알려진 이슈
- **"신청" 보호 방식**: 기존 `isSystem` boolean 대신 `name === "신청"` 문자열 비교로 변경됨. 만약 "신청" 이름을 가진 다른 액션 타입이 생성되면 해당 타입도 보호됨 (현실적으로 발생 가능성 낮음)
- **DB 스키마 변경 없음**: `icon_url` 컬럼은 이미 존재했으므로 `drizzle-kit push` 불필요
- **하위호환**: 기존 아이콘이 설정되지 않은 액션 타입은 기존 하드코딩 매핑(제출→Search, 출석→CheckSquare 등)으로 자동 fallback

## 연관 파일 (직접 수정 안함, 영향 받음)
- `server/controllers/admin/admin.action.type.controller.ts` — 에러 메시지 분기 (SYSTEM_IMMUTABLE, SYSTEM_NAME_IMMUTABLE) 동일하게 유지
- `server/services/mission/mission.user.service.ts` — `actionType: true` relation 조회 시 `iconUrl` 자동 포함
- `shared/schema.ts` — `actionTypes` 테이블에 `iconUrl` 컬럼 이미 정의됨

## 수동 테스트 순서
1. `localhost:5000/api/dev/auto-login` 접속 (자동 로그인)
2. 관리자 → 미션 시스템 관리 → 액션 타입 관리 탭 이동
3. "제출" 행의 수정 버튼 활성화 확인 (기존에는 비활성)
4. 수정 클릭 → 아이콘 선택기 표시 확인
5. 한글로 "사진" 검색 → 관련 아이콘 필터링 확인
6. 아이콘 선택 후 저장
7. 미션 상세 페이지에서 하단 탭바 아이콘 변경 확인
