## 핵심 변경 내용

### Phase 1: 라우트 분리 + 리치 푸시

- **왜**: `admin-routes.ts`가 3713줄로 비대하여 유지보수 불가
- **변경**: 푸시 관련 코드를 `push-admin-routes.ts`로 복사 → 확인 → 원본 삭제
- **리치 푸시**: `push.service.ts`에서 `payload.data.image_url`이 있으면 FCM `notification.image`에 자동 매핑

### Phase 2: 병원별 발송 + 회원 검색

- **왜**: 관리자가 특정 병원 소속 회원에게만 푸시를 보내거나, 회원을 검색/선택할 수 없었음
- **변경**:
  - `push-send` API에 `targetType: "hospital"` 분기 추가 (`sendToTopic("hospital_${id}")`)
  - `GET /api/admin/push-users?search=xxx` API 신규 (이름/이메일/전화번호 검색, 병원ID 필터)
  - `GET /api/admin/push-hospitals` API 신규 (병원 선택 드롭다운용)
  - `PushUserSelector.tsx` — 디바운스 검색, 체크박스 다중선택, 선택된 유저 뱃지 표시

### Phase 3: 동적 알림 템플릿

- **왜**: 기존에 하드코딩된 템플릿만 존재하여 관리자가 새 알림 유형 추가 불가
- **변경**:
  - `push_templates` DB 테이블 생성 (id, name, title, body, actionUrl, imageUrl, category, isActive 등)
  - CRUD API 4개 (GET/POST/PATCH/DELETE `/api/admin/push-templates`)
  - `PushTemplates.tsx` 관리 UI (목록 카드, 생성/수정 다이얼로그)
  - `PushSend.tsx`에 템플릿 선택 드롭다운 — 선택 시 제목/본문/URL 자동 채움

## 수동 테스트 요청

```text
👤 수동 테스트 요청:
1. localhost:5000/api/dev/auto-login 접속 (자동 로그인)
2. /admin?tab=push-management 이동
3. "발송 내역" 탭 → 기존 로그 정상 표시
4. "수동 알림 발송" 탭 → 발송 대상 3개 옵션 확인
   - "전체 앱 사용자" / "특정 사용자(ID)" / "지정 병원(거래처)"
5. "특정 사용자(ID)" 선택 → 검색어 입력 → 회원 체크박스 선택
6. "지정 병원(거래처)" 선택 → 병원 드롭다운 표시 확인
7. "알림 템플릿" 탭 → "템플릿 추가" → 생성/수정/삭제 확인
8. "수동 알림 발송" 탭 → 템플릿 선택 드롭다운 → 자동 채움 확인
```

## 주의사항

1. **git 커밋하지 않았음** — 사용자가 `!!푸시!!`로 직접 커밋해야 함
2. **프로덕션 DB 반영 필요** — Railway에서도 `npx drizzle-kit push`로 `push_templates` 테이블 생성 필요
3. **FCM 토픽 구독** — 병원별 발송(`hospital_{id}`)은 클라이언트에서 FCM 토픽 구독이 선행되어야 실제 도달됨
