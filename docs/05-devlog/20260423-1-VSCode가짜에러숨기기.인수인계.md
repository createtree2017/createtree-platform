# 2026-04-23 VSCode 가짜 에러(Java 확장) 숨기기 인수인계

## 1. 변경 배경 및 의사결정 이유
- `node_modules/@capacitor` 내부에 있는 안드로이드 `.java` 파일들을 VS Code의 Java 확장 프로그램이 임의로 스캔하여 에러가 있는 것으로 오탐지하는 현상 발생.
- 프로젝트 빌드/실행에는 전혀 영향을 주지 않는 가짜 에러임에도, 빨간 줄로 표시되어 작업 시 혼동을 주고 IDE 리소스를 낭비함.
- 사용자(관리자)의 승인 하에 `settings.json`을 수정하여 Java 분석기에서 `node_modules`를 무시하도록 처리함.

## 2. 변경 파일 목록
- `.vscode/settings.json`

## 3. 핵심 변경 내용
- `java.import.exclusions` 설정 추가
- `**/node_modules/**` 경로를 분석 제외 대상으로 등록

## 4. 동작 확인 상태
- ✅ 파일 변경 완료 (VS Code에서 해당 설정을 인식하면 즉시 에러가 사라집니다.)

## 5. 주의사항 / 알려진 이슈
- 향후 안드로이드 네이티브 코드를 수정하더라도 실제 안드로이드 소스는 `/android` 하위에 위치하므로, 이 제외 설정 때문에 문법 검사가 안 되는 일은 없습니다. 안심하셔도 됩니다.
