# 미션 단일 이미지 제출 버튼 복구 인수인계

작성일: 2026-04-29

## 1. 배경

문화센터 미션의 세부미션에서 제출 타입을 `image` 1개만 설정한 경우, 사용자 화면의 세부미션 모달에 이미지 선택/첨부 버튼이 표시되지 않고 `제출하기` 버튼만 보이는 문제가 있었다.

반면 제출 타입을 `["image", "image"]`처럼 2개 이상으로 설정하면 `제출 항목 선택` 목록이 렌더링되어 이미지 선택 진입 버튼이 표시됐다.

## 2. 원인

`client/src/pages/mission-detail.tsx`의 사용자 제출 폼은 제출 타입이 2개 이상일 때만 `제출 항목 선택` 목록을 표시한다.

이미지 선택 모달(`missionCollage`)을 여는 로직이 이 목록 버튼의 `onClick` 안에만 들어 있어, `submissionTypes = ["image"]`인 단일 이미지 제출 세부미션에서는 이미지 선택 진입점이 사라졌다.

이미지 타입 본문 영역에는 숨겨진 `input type="file"`만 있고 실제 버튼이 없어 사용자가 이미지를 첨부할 수 없었다.

## 3. 변경 내용

### `client/src/pages/mission-detail.tsx`

- `openImageSubmissionPicker(slotIndex)` 공통 함수를 추가했다.
  - 선택 슬롯 인덱스를 갱신한다.
  - `missionCollage` 모달을 연다.
  - 완료된 이미지 URL을 해당 슬롯의 `imageUrl`에 저장한다.
- 다중 제출 타입 목록에서 이미지 항목 클릭 시 기존 인라인 로직 대신 공통 함수를 사용하도록 변경했다.
- `selectedSubmissionType === "image"` 영역에 이미지 선택 버튼을 추가했다.
  - 단일 이미지 제출 타입에서도 버튼이 항상 표시된다.
  - 이미 이미지가 선택되어 있으면 `이미지 변경`으로 표시된다.
  - 이미지가 없으면 관리자 설정 라벨(`추가이미지` 등)을 버튼 텍스트로 사용한다.

## 4.1 추가 UI 보완

- 단일 이미지 제출 버튼의 시각적 구분이 약해, 사용자가 이미지를 선택하지 않고 바로 제출할 수 있다는 우려가 있었다.
- 이미지 미첨부 상태의 버튼을 다중 이미지 제출 항목과 같은 흰색 배경/검정 텍스트 스타일로 변경했다.
- 이미지 첨부 완료 상태에서는 초록색 계열 배경/테두리/텍스트와 `완료` 표시를 보여주도록 변경했다.
- 긴 관리자 라벨이 들어와도 버튼 내부에서 말줄임 처리되도록 `truncate`를 적용했다.

## 5. 주의사항

- 이번 수정은 프론트엔드 사용자 제출 모달의 이미지 선택 진입점 복구만 수행했다.
- 서버 업로드 API나 DB 스키마는 변경하지 않았다.
- 브라우저 직접 화면 검증은 이번 요청에 `!!테스트!!`가 없어 수행하지 않았다.
- 수동 확인 시 `submissionTypes = ["image"]`인 세부미션을 열었을 때 `제출하기` 위에 이미지 선택 버튼이 표시되고, 버튼 클릭 시 `missionCollage` 모달이 열리는지 확인하면 된다.
