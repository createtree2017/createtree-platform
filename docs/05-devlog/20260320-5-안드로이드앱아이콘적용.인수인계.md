# [작업명] 안드로이드 앱 런처 아이콘 적용 및 v1.4 번들(AAB) 배포
작업 일시: 2026.03.20

## 변경 파일 목록
- `package.json`, `package-lock.json`
- `assets/icon.png`, `assets/splash.png` (추가)
- `android/app/src/main/res/...` 내 런처/스플래시 아이콘 파일 74개 생성 및 갱신
- `android/app/build.gradle` (versionCode 4, versionName 1.4 로 수동 업데이트 완료)

## 핵심 변경 내용 및 히스토리
1. **런처 아이콘 및 스플래시 스크린 일괄 적용**
   - 첨부된 원본 크기 이미지를 `assets/` 폴더에 복사.
   - `@capacitor/assets` 플러그인 설치 후 `npx capacitor-assets generate --android` 실행하여 모든 해상도(74개) 대응 에셋 자동 생성 완료.
2. **웹 코드 빌드 및 안드로이드 동기화**
   - `npm run build` 및 `npx capacitor sync android` 실행으로 최신 웹 코드를 안드로이드 네이티브 단으로 이관.
3. **버전 코드 퍼블리싱 (안드로이드 스튜디오)**
   - `build.gradle (Module: app)`에서 앱 버전을 갱신(`versionCode 3 -> 4`, `versionName "1.3" -> "1.4"`)하고 Gradle 싱크 진행.
4. **문제 해결: 서명된 번들(AAB) 추출 에러**
   - 기존 Keystore 파일 경로 인식 오류(폴더명 앞의 `Z.` 누락)를 발견하고 경로 재지정 팝업을 통해 해결, 릴리즈 앱 번들(`.aab`) 성공적으로 추출 완료.
5. **구글 플레이 콘솔 배포 (비공개 테스트 트랙으로 우회 연동)**
   - 1.4 버전을 콘솔에 정상적으로 업로드 성공.
   - 현재 **프로덕션** 트랙 접근 권한이 구글 측에서 심사 중(비공개 테스트 14일 조건 달성 후 심사 단계)인 상태라, 프로덕션에 즉시 번들을 출시하는 것이 불가능함.
   - 대안으로 **비공개 테스트 트랙**에 업로드한 버전 4(1.4) 번들을 배포하여, 심사 대기 없이 즉각적으로 개발자 기기에서 앱 아이콘 갱신 여부를 확인할 수 있도록 로직을 정리함.

## 동작 확인 상태 (✅/❌)
- ✅ `@capacitor/assets`를 통한 해상도별 아이콘 자동 생성 성공
- ✅ 경로명 오류 수정 후 Signed App Bundle (`.aab`) 정상 추출 및 구글 콘솔 업로드 성공

## 다음 작업 참고사항
- 바탕화면/앱 서랍에 설치되는 앱 아이콘은 본 배포로 갱신되었습니다. 하지만 **구글 플레이 콘솔 검색 및 스토어 탭에 노출되는 아이콘**은 콘솔 웹사이트의 **[앱 정보] -> [기본 스토어 등록정보]**에서 512x512 사이즈로 직접 재업로드 해주셔야만 완벽하게 갱신됩니다.
- 약 며칠 뒤 구글로부터 "프로덕션 액세스 권한 허용" 안내가 오면, 이번에 비공개 테스트에 올려서 확인을 마치신 v4 버전(1.4)을 **'프로덕션으로 승급(Promote)'** 버튼 클릭만으로 즉시 정식 출시하실 수 있습니다.
