# 20260320-6 새컨셉추가500에러수정 인수인계

## 작업 요약
- **작업일시**: 2026-03-20 17:49
- **작업 유형**: Bug Fix (Quick Fix)
- **문제**: 관리자 페이지에서 '새 컨셉 추가' 시 500 Internal Server Error 발생
- **원인**: DB `system_settings.supported_ai_models`에 과거 폐기된 `openai_mini` 값이 잔존 → 프론트엔드에서 그대로 상속 → POST API에서 Zod 검증 실패 → 500 에러

## 변경 파일 목록

### 1. `server/routes/admin-routes.ts`
- **변경 위치**: POST `/api/admin/concepts` 핸들러 (L1460~1494)
- **변경 내용**:
  - PUT 쪽에만 있던 `validModels` 필터링 로직을 POST에도 동일하게 추가
  - `openai_mini` 등 폐기 모델이 `availableModels`에 포함되어 있으면 자동 제거
  - 필터링 후 빈 배열이면 기본값 `["openai", "gemini_3_1"]` 설정
  - catch 블록에 `ZodError` 분기 추가 → 검증 실패 시 400 응답 (기존: 일괄 500)

### 2. `server/utils/settings.ts`
- **변경 위치**: `getSystemSettings()` 함수 내 자동 마이그레이션 섹션 뒤 (L175~215)
- **변경 내용**:
  - 기존 `gemini → gemini_3_1` 마이그레이션 뒤에 폐기 모델 자동 정화 로직 추가
  - `AI_MODELS` 상수에 없는 모든 모델을 `supportedAiModels`에서 자동 제거
  - `defaultAiModel`, `clientDefaultModel`이 폐기 모델이면 `openai`로 교체
  - 서버 시작 시 첫 설정 조회 때 자동 실행되어 DB 데이터 정합성 보장

## 동작 확인 상태
- ✅ 코드 수정 완료
- ✅ `z` import 확인 (admin-routes.ts L2)
- ⬜ 서버 재시작 후 '새 컨셉 추가' 실행 테스트 필요

## 다음 작업 참고사항
- 서버 재시작 시 `settings.ts`의 정화 로직이 자동 실행되어 DB에서 `openai_mini`가 제거됨
- 이후에는 프론트엔드에서 더 이상 `openai_mini`를 상속받지 않으므로 경고도 사라짐
- 만약 추후 새 AI 모델을 추가할 때는 `shared/schema.ts`의 `AI_MODELS` 상수에 추가하면 `settings.ts` 정화 로직이 자동으로 인식함
