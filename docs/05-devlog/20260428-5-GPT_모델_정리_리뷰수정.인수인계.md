# GPT 모델 정리 리뷰 수정 인수인계

## 작업 일시
- 2026-04-28

## 수정 배경
- 이전 GPT Image 공식 스펙 정리 작업 보고와 실제 코드 상태가 일부 맞지 않았다.
- `openai_gpt2_mini` 제거 누락, OpenAI 실패 시 placeholder URL 반환, `transformedImageUrl` 미초기화 타입 오류 가능성, `AI_MODELS` 검증 타입 오류가 리뷰에서 지적되었다.

## 수정 내용
- `shared/schema.ts`
  - concepts 기본 `availableModels`와 `availableAspectRatios`에서 `openai_gpt2_mini`를 제거하고 `openai_gpt1_mini`로 정리했다.
  - `insertConceptSchema.availableModels` enum에 `openai_gpt1_mini`를 반영했다.
- `server/services/openai.ts`
  - `generateImageWithGPT2`, `generateImageWithGPT1Mini`, `generateImageWithDALLE`, `transformImageWithOpenAI`에서 OpenAI 실패를 `placehold.co` URL로 반환하지 않고 예외로 전파하도록 정리했다.
- `server/services/openai-dalle3.ts`
  - `SERVICE_UNAVAILABLE` placeholder 상수를 제거했다.
  - API 키 없음, 프롬프트 오류, 응답 파싱 실패, OpenAI API 오류가 실제 실패로 전파되도록 수정했다.
- `server/routes/image.ts`
  - `transformedImageUrl` 초기값을 빈 문자열로 두어 TypeScript의 used before assigned 가능성을 제거했다.
  - 공식 모델 라우팅을 `openai`, `openai_gpt2`, `openai_gpt1_mini` 기준으로 정리했다.
- `server/routes/admin-routes.ts`
  - `Object.values(AI_MODELS).includes(m)` 타입 불일치를 `Set<string>.has(m)` 방식으로 수정했다.

## 검증 결과
- `npm.cmd run check` 실행 결과 전체 저장소의 기존 TypeScript 오류들 때문에 실패했다.
- 필터링 확인 결과, 이번 수정 범위인 `shared/schema.ts`, `ConceptFormModal`, `server/routes/admin-routes.ts`, `server/routes/image.ts`, `server/services/openai.ts`, `server/services/openai-dalle3.ts` 관련 오류는 출력되지 않았다.
- `server/services/openai.ts`, `server/services/openai-dalle3.ts`에서 `placehold.co`와 `SERVICE_UNAVAILABLE` 검색 결과는 더 이상 나오지 않았다.

## 남은 주의사항
- 전체 타입 체크 실패는 기존 관리 화면, 보호 라우트, 배경제거, 마일스톤 등 다른 영역의 타입 오류가 원인이다.
- `gpt-image-2`는 코드상 공식 모델명으로 호출되지만, 현재 OpenAI 프로젝트 권한이 없으면 API에서 403이 발생한다.

## 추가 수정 (Firebase 이미지 업로드 로직 오류 해결)
- **원인**: Firebase 기반 이미지 업로드 시, 미들웨어에서 처리한 `downloadedBuffers`를 이미지 존재 여부(`hasAnyImage`) 및 개수 판단 로직에서 누락하여 텍스트 전용 모드로 오판하는 버그 발생.
- **해결**: `hasAnyImage` 판단 조건에 `downloadedBuffers.length > 0`을 추가하고 로그 상 `actualImageCount`도 Firebase 모드를 우선 반영하도록 수정 완료.
