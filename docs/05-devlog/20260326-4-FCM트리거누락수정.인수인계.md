# 인수인계: FCM 미션 검수 트리거 누락 수정 (2026-03-26)

## 1. 변경 파일 목록
- `server/services/mission/admin.mission.review.service.ts`: 미션 검수 로직에 FCM 알림 트리거 추가

## 2. 핵심 변경 내용
- **알림 트리거 통합**: `approveSubmission`, `rejectSubmission`, `bulkUpdateStatus` 메서드 완료 시점에 `createNotification`을 호출하도록 수정.
- **동적 제목 조회**: 알림 메시지에 미션 제목을 포함하기 위해 `subMission.themeMission` 관계 조회 추가.
- **안전한 호출**: 알림 발송은 비동기(`.catch`)로 처리하여 메인 비즈니스 로직(DB 업데이트)에 영향을 주지 않도록 설계.

## 3. 변경 배경 및 의사결정 이유
- **누락 발견**: 정밀 검침 과정에서 핵심 인프라는 완벽하나 실제 사용자에게 알림을 보내는 비즈니스 로직 연결부가 누락된 것을 확인.
- **UX 개선**: 사용자가 미션 결과(승인/반려)를 확인하기 위해 앱을 계속 켜두는 대신, 푸시를 통해 실시간으로 인지할 수 있도록 보완.

## 4. 동작 확인 상태
- ✅ **미션 승인 시 알림 생성**: `AdminMissionReviewService.approveSubmission` 호출 시 `notifications` 테이블에 정상적으로 레코드 생성됨을 확인.
- ✅ **FCM 엔진 연동**: 활성 기기 토큰이 있는 경우 FCM 서버로 전송 시도 로집 정싱 확인. (테스트 환경에서는 토큰 부재로 no-op 로그 확인)

## 5. 주의사항 및 알려진 이슈
- **토큰 누락**: 사용자가 앱 설정에서 알림 권한을 거부하거나 앱을 삭제한 경우 `[Push] userId=... 활성 토큰 없음` 로그가 남으며 발송되지 않습니다.
- **데이터 타입**: `notifications.userId`는 `text` 필드이므로, `Number(userId)` 변환 시 주의가 필요합니다. (이번 수정에 반영 완료)

## 6. 다음 작업 참고사항
- 안드로이드/iOS 실기기 빌드 시 딥링크(`actionUrl: /mymissions`)가 올바른 화면으로 랜딩되는지 최종 확인 권장.
