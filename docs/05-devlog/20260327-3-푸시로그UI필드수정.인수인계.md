# 20260327-3 푸시 발송 내역 UI 필드 수정

## 변경 배경
외부 코드 검증 + 자체 팩트체크 결과, 관리자 페이지 "발송 내역" 탭(`PushLogs.tsx`)에서 **DB 스키마와 프론트엔드 필드명 불일치 3건**이 발견되었음. FCM 발송 기능 자체에는 영향 없으나, 관리자가 발송 내역을 조회할 때 제목/전체건수/상태 뱃지가 정상 표시되지 않는 UI 버그.

## 변경 파일 목록

### 1. `client/src/components/admin/PushLogs.tsx` (3곳 수정)

| # | 줄 | 변경 전 | 변경 후 | 이유 |
|---|:---:|---------|--------|------|
| 1 | 26 | `case "success":` | `case "completed":` | `push.service.ts`의 `insertDeliveryLog()`가 `status: 'completed'`로 저장하므로 매칭 |
| 2 | 85~86 | `log.payloadTitle` | `log.title` | DB 컬럼명은 `title` (Drizzle camelCase 변환 결과도 `title`) |
| 3 | 88 | `log.totalAttempted` | `(log.successCount \|\| 0) + (log.failureCount \|\| 0)` | `totalAttempted` 컬럼 자체가 없음. 두 값의 합으로 계산 |

### 2. `server/routes/push-admin-routes.ts` (1곳 수정)

| # | 줄 | 변경 전 | 변경 후 | 이유 |
|---|:---:|---------|--------|------|
| 4 | 30 | `orderBy(desc(completedAt))` | `orderBy(desc(createdAt))` | `completedAt`은 nullable → 미래 pending 상태 로그 도입 시 정렬 불안정. `createdAt`은 NOT NULL이라 항상 안전 |

## 동작 확인

- ⬜ 브라우저 테스트 미실행 (!!테스트!! 미사용)

- `server/services/push/push.service.ts` — 265줄에서 `status: 'completed'`로 INSERT (수정 근거)
- `shared/schema.ts` — 2271~2284줄 `pushDeliveryLogs` 테이블 정의 (필드명 근거)

## 주의사항
- 이 수정은 **관리자 UI 표시만** 영향받음. FCM 실발송/토큰관리/트리거 등 핵심 로직은 무관
- 수동 테스트 시: 관리자 페이지 → 앱 푸시 알림 → 발송 내역 탭에서 제목/전체건수/상태 뱃지 정상 표시 확인
