- **1분할 레이아웃 추가**: 사용자가 1장의 사진만 그대로 제출하고 싶을 때를 대비해, 콜라주 생성 기능 내에 '1분할' 옵션을 추가했습니다.
- **MissionCollageModal 신규 생성**: 
  - `gallery-collage.tsx`의 뼈대를 활용해, 모달 내에서 레이아웃 선택 → 이미지 추가 → 병합 및 자동 제출이 가능한 컴포넌트를 만들었습니다.
  - 생성 API 호출 후 바로 상태를 폴링하여 `outputUrl`을 얻고 팝업을 닫습니다.

## [추가 작업] 미션 보류(Hold) 상태 UI 및 재제출 기능 추가

### 1. 추가 작업 내용
- 미션 제출 후 관리자가 '보류(rejected)' 처리했을 때, 사용자 화면에 명확한 보류 상태와 사유를 표시하도록 UI를 수정했습니다.

### 2. 변경 파일
- \client/src/pages/mission-detail.tsx\: 보류 상태 UI 렌더링 조건 추가

### 3. 핵심 변경 내용
- **보류 사유 표시**: \subMission.submission.status === 'rejected'\일 경우 상단에 빨간색 배너로 '보류 사유(reviewNotes 또는 rejectReason)'를 렌더링합니다.
- **상태 버튼 변경**: 기존 파란색 '제출완료 - 검수중' 버튼을 빨간색 '보류됨 - 다시 제출 필요' 버튼으로 대체했습니다.
- **수정 후 제출 유도**: 기존 '수정' 버튼 텍스트를 보류 상태일 때는 '수정 후 제출'로 변경하여 사용자가 반려 사유를 확인하고 즉각적으로 다시 제출할 수 있도록 조치했습니다.

- **하단 탭바 보류 상태 연동**: 하단 세부미션 내비게이션 아이콘에도 보류(\ejected\) 상태를 반영하여, 기존의 미제출 보라색 아이콘으로 보이지 않도록 빨간색 스타일링 및 경고 배지(\AlertCircle\)를 추가했습니다.

- **[버그수정] 검수 의견(보류 사유) 저장 오류**: 프론트엔드 관리자 화면(\MissionManagement.tsx\)에서 보류 API 호출 시 \eviewerNote\라는 키로 전송하던 데이터를 백엔드가 기대하는 \ejectReason\ 키로 일치시켜, 정상적으로 DB에 저장되고 사용자 화면에 노출되도록 수정했습니다.

- **[개선] 관리자 모달 검수 의견(보류 사유) 연동**: 관리자가 검수 모달(\SubmissionDetailModal.tsx\)을 열었을 때, 이전에 본인이 작성해서 사용자에게 보냈던 보류 사유(\ejectReason\)를 자동으로 \	extarea\에 불러오도록 \useEffect\ 훅을 추가했습니다.

- **[UX 개선] 이미지 제출 클릭 뎁스 축소 및 완료 UI 직관성 강화**: 사용자가 '이미지제출' 슬롯을 클릭하면 하단의 버튼을 다시 누를 필요 없이 즉시 콜라주/갤러리 모달이 열리도록 연동했습니다. 또한 제출이 완료된 슬롯의 초록색 체크표시를 버튼 우측 내부로 이동시키고 '완료' 텍스트를 추가하여 UI를 안정감 있게 개선했습니다.

- **[스타일 개선] 완료 배지 시인성 향상**: 완료 표시가 배경색에 묻히지 않도록 반투명한 흰색 둥근 배경(pill shape)을 추가하여 대비(Contrast)를 대폭 높였습니다.

- **[UI 정리] 불필요한 중복 버튼 제거**: 슬롯 클릭 시 갤러리/콜라주 창이 바로 열리게 됨에 따라, 기능이 중복되는 하단의 '사진 선택 및 콜라주 만들기' 버튼을 삭제하여 화면을 더욱 깔끔하게 정리했습니다.
