# 2026-06-23 세부미션 신청 검수 문구 수정 인수인계

## 작업 개요

관리자 세부미션 수정 모달에서 `신청` 액션 타입일 때 표시되는 `검수 필요` 안내 문구를 수정했다.

## 연결된 PDCA 문서

- 없음: 단일 UI 문구 수정 Quick Fix

## 변경 파일

- `client/src/components/modal/admin/SubMissionFormModal.tsx`

## 핵심 변경 내용

- 기존 문구는 `신청` 타입이면 실제 주제미션의 `isFirstCome` 여부와 관계없이 “선착순 신청”이라고 표시되어 운영자가 선착순 설정으로 오해할 수 있었다.
- 문구를 아래처럼 변경했다.
  - 기존: `선착순 신청은 정원 이내 자동 승인되므로 검수 필요 옵션이 적용되지 않습니다.`
  - 변경: `신청 타입은 주제미션의 모집 인원/선착순 설정에 따라 처리되므로 검수 필요 옵션을 직접 설정할 수 없습니다.`

## 변경 이유

세부미션 모달의 안내 문구가 실제 선착순 설정을 읽은 결과처럼 보였지만, 코드상으로는 액션 타입이 `신청`이면 고정으로 노출되는 문구였다. 주제미션의 모집 인원과 선착순 설정에 따라 신청 처리 방식이 달라진다는 실제 구조를 더 정확히 안내하기 위해 문구만 보정했다.

## 동작 확인 상태

- `rg`로 변경 문구 반영 확인 완료
- 기능 로직 변경 없음
- `!!테스트!!` 요청이 아니므로 브라우저 검증은 수행하지 않음

## 주의사항 / 알려진 이슈

- 이번 작업은 UI 안내 문구만 수정했다.
- 기존 신청 승인 데이터나 주제미션의 `capacity`, `isFirstCome` 값은 변경하지 않았다.
- 신청 자동승인/관리자승인/대기자 처리 로직은 `server/services/mission/mission.submission.service.ts` 기존 동작을 유지한다.
