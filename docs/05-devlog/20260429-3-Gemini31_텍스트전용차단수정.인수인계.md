# Gemini 3.1 Flash 텍스트 전용 차단 수정 인수인계

작성일: 2026-04-29

## 1. 배경

스티커 텍스트 전용 컨셉에서 `Gemini 3.1 Flash`를 선택하면 "텍스트 전용 이미지 생성을 지원하지 않습니다" 오류가 발생했다.

공식 문서 기준 `gemini-3.1-flash-image-preview`는 text-to-image와 image-to-image를 모두 지원하므로, 해당 오류는 모델 스펙 문제가 아니라 코드의 오래된 제한 로직 문제로 확인했다.

## 2. 원인

- `shared/model-capabilities.ts`에서 `gemini_3_1.supportsTextOnly`가 `false`로 설정되어 있었다.
- `/api/generate-stickers` 라우트에 `finalModel === "gemini_3_1"`일 때 text-only 생성을 직접 차단하는 하드코딩이 남아 있었다.
- DB 기준 `gemini_3_1`을 사용하는 text-only 컨셉이 12개 있어, 특정 컨셉 하나가 아니라 텍스트 스티커 계열 전반에서 같은 문제가 발생할 수 있었다.

## 3. 변경 내용

### `shared/model-capabilities.ts`

- `gemini_3_1.supportsTextOnly`를 `true`로 변경했다.
- capability registry 기준으로 Gemini 3.1이 text-to-image와 image-to-image를 모두 지원하는 모델로 판정되도록 맞췄다.

### `server/routes/image.ts`

- `/generate-stickers`의 Gemini 3.1 전용 text-only 차단 블록을 제거했다.
- 스티커 생성에서도 `generationType`을 `generateImageWithConfiguredModel()`에 전달하도록 보강했다.

## 4. 검증

- `npx.cmd tsx` 스모크 테스트 통과
  - `gemini_3_1.supportsTextOnly === true`
  - `text_only + gemini_3_1 + referenceImageCount = 0` 조합이 오류 없이 생성 모드 판정을 통과함.
- `npm.cmd run build` 통과
  - 기존 Vite chunk 경고만 표시됨.
- `git diff --check` 통과
  - CRLF 변환 경고만 표시됨.
- 소스 검색 확인
  - `Gemini 3.1 제한`
  - `Gemini 3.1 Flash 모델은 텍스트 전용 이미지 생성을 지원하지 않습니다`
  - `supportsTextOnly: false`
  - 위 패턴이 실제 소스에서 더 이상 발견되지 않음.

## 5. 주의사항

- 실제 브라우저/AI 생성 검증은 `!!테스트!!` 지시가 없어 수행하지 않았다.
- `/generate-family`, `/generate-stickers`의 이미지 입력 정규화는 아직 `/generate-image` 수준까지 완전 공통화되지 않았다. 후속 Phase에서 `request-images.ts` / `generation-mode.ts` 적용을 계속 진행하면 된다.
- 이번 수정은 Gemini 3.1의 잘못된 text-only 차단을 제거하는 범위이며, DB 스키마 변경은 없다.
