# 20260321-1-AI모델오류해결및타입통일

## 📝 작업 내역
- **작업 목적**: 과거 폐기되었던 `gptmini`(`openai_mini`) 모델의 찌꺼기로 인한 이중 생성 및 에러가 발생하던 문제와, `gemini_3` 모델 타입이 일부 환경설정 로직과 불일치하여 발생(ts-2345/2353 에러)하던 현상 동시 해결
- **작업 시간**: 서버 시간 2026년 3월 21일

## 📁 변경된 파일 목록
- `shared/schema.ts`
- `server/utils/settings.ts`

## 🛠 핵심 변경 내용
1. **AI 모델 타입 완벽 통일**
   - 현재 사용하는 3가지 모델(`openai`, `gemini_3_1`, `gemini_3`)이 시스템상에 완벽히 호환되도록 Drizzle ORM 및 Zod 배열 타입 검증 부분을 수정했습니다.
   - `server/utils/settings.ts` 내부의 에러 폴백(fallback) 시 반환되는 배열에도 `gemini_3`를 포함하도록 하드코딩된 누락 구문을 수정했습니다.
2. **rejectReason 컬럼 누락 수정**
   - 개발 중이던 `admin.mission.review.service.ts`의 345, 368라인에서 발생했던 반려 사유 대입과 관련된 `ts(2353)` 에러를 해결하기 위해, `schema.ts` 안의 `subMissionSubmissions` 테이블 정의에 `rejectReason` 텍스트 필드를 공식적으로 추가했습니다.

## ✅ 동작 확인 상태
- [✅] Typescript 컴파일(`npx tsc --noEmit`) 에러 해결 확인
- [✅] `gptmini`/`openai_mini` 관련 하드코딩 잔재 전체 검색 및 더 이상 코드상에서 관여하지 않음 확인 (`settings.ts`의 과거 잔존 데이터 자동 제거 구문만 정상 활성화 됨 유지 중)

## 📌 다음 작업 참고사항
- DB 스키마에 `rejectReason` 컬럼이 추가되었으므로 `npx drizzle-kit push` 등의 변경 적용이 필요할 수 있습니다.
- 이미지 생성 오류(두 번 요청)는 존재하지 않는 옛 모델(`gptmini`)을 불러오려던 fallback 로직 문제가 주된 원인이었으므로, 이 타입 안정화를 통해 해결될 가능성이 매우 큽니다. 계속 에러가 발생한다면 프론트엔드의 루프(이중 렌더링 또는 API 중복 호출) 쪽을 점검해야 합니다.

### 핵심 변경 내용
- **원인**: 이미지 생성 시 AI API에 전달하기 위해 Firebase에 업로드된 원본 첨부 사진이 `images` DB에 `firebase_upload` 카테고리 / `firebase-direct` 스타일로 저장되는데, 전체 갤러리 탭이 필터 없이 모든 이미지를 조회하여 원본도 함께 노출되던 문제
- **해결**: 전체(all) 탭 쿼리에 `ne(images.categoryId, 'firebase_upload')` 및 `ne(images.style, 'firebase-direct')` 조건 추가
- 카테고리별 탭(가족사진, 스티커 등)은 기존대로 정상 동작
