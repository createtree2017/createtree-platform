## 변경 파일

### `client/src/components/modal/admin/BigMissionModals.tsx`

**변경 내용:**
- `import`에 `useRef`, `Upload`, `X` 아이콘 추가
- `BigMissionTopicFormModal` 컴포넌트 내 추가 state/ref:
  - `iconPreview: string | null` — 로컬 미리보기 (FileReader base64)
  - `isIconUploading: boolean` — 업로드 중 로딩 표시
  - `iconFileInputRef: useRef<HTMLInputElement>` — 숨겨진 파일 input 제어
- **핸들러 추가**:
  - `handleIconFileSelect`: 파일 선택 시 미리보기 즉시 표시 → `POST /api/upload` (GCS) 호출 → 성공 시 `topicFormData.iconUrl` 업데이트
  - `handleRemoveIcon`: 미리보기 및 iconUrl 초기화
- **UI 변경**: 아이콘 URL 텍스트 Input → 원형 미리보기 박스 + 이미지선택 버튼 + 제거 버튼

---

## 왜 변경했는지

URL을 직접 입력하는 방식은 관리자가 GCS URL을 알아야 하고 복붙 실수 가능성이 높아 UX가 나쁨.
파일 업로드 방식으로 변경하면 이미지를 직접 선택 → 자동으로 GCS에 저장 → URL을 자동 주입함.

---

## 사용된 API

- `POST /api/upload`: 기존 관리자 전용 범용 업로드 API (GCS, `field=file`, 응답 `{ url }`)
- **백엔드 변경 없음**

---

## 동작 확인 상태

- ✅ 코드 수정 완료
- 👤 수동 테스트 필요:
  1. `localhost:5000/api/dev/auto-login` 접속
  2. 관리자 > 메뉴관리 > 나의미션 > 큰미션 탭
  3. 큰미션의 토픽 관리(격자 아이콘) 클릭
  4. 토픽 추가 → 아이콘 이미지 선택 → 업로드 위치에 원형 미리보기 표시 확인
  5. 추가 저장 후 토픽 목록에 아이콘 표시 확인

## 주의사항

- `POST /api/upload` 응답 필드는 `url` (url 키로 GCS 공개 URL 반환). `fileUrl`이 아님에 주의.
- 업로드 전에는 `iconUrl`이 빈 문자열로 저장되므로 취소 시 기존 값이 유지되지 않을 수 있음 → 수정 모달 열 때 기존 iconUrl이 `iconPreview`에 표시됨 (기존 이미지 유지 가능)
