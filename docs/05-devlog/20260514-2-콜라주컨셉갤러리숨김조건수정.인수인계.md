# 콜라주 컨셉 갤러리 숨김 조건 수정

작성일: 2026-05-14

## 1. 작업 개요

가족사진 컨셉 ID가 `collage`인 이미지 생성 결과가 성공적으로 저장되지만 사용자 갤러리에서 보이지 않는 문제를 수정했다.

## 2. 변경 파일

- `server/utils/imageFilter.ts`
- `server/routes/gallery.ts`
- `server/routes/image.ts`

## 3. 핵심 변경 내용

- `isHiddenFromUserGallery()` 공통 판정 함수를 추가했다.
- 사용자 갤러리에서 `style = collage` 전체를 제외하던 조건을 제거했다.
- 내부 미션 콜라주는 아래 조건으로만 숨기도록 좁혔다.
  - `categoryId = collage`
  - `metadata.hiddenFromGallery = true`
  - `metadata.purpose = mission_submission`
  - 구형 데이터 보호용 `style = collage`이면서 `categoryId`가 없는 경우
- `/api/gallery?filter=family_img`에서 `style = collage`, `categoryId = family_img`인 가족사진 컨셉 결과가 조회되도록 했다.
- 갤러리 응답 `type`은 `style`보다 `categoryId`를 우선 사용하도록 조정했다.

## 4. 왜 변경했는지

컨셉 ID `collage`를 바꾸면 기존 관리자 설정, 생성 이력, URL 파라미터, 컨셉 변수 연결에 영향이 생길 수 있다. 실제 문제는 컨셉 ID가 아니라 사용자 갤러리 조회 조건이 `style = collage`를 내부 콜라주 기능과 가족사진 컨셉 결과에 동일하게 적용한 것이므로, 조회 조건을 내부 콜라주 판정으로 좁히는 방식이 더 안전하다.

## 5. 동작 확인 상태

- `npm run build`: 성공
- `npx tsc --pretty false --noEmit`: 기존 Persona/마일스톤 타입 오류로 실패
- 변경 파일 대상 오류 필터: `server/routes/gallery.ts`, `server/routes/image.ts`, `server/utils/imageFilter.ts` 관련 신규 오류 없음

## 6. 주의사항

- `!!테스트!!` 키워드가 없어 브라우저 에이전트 검증은 수행하지 않았다.
- 실제 생성 API 호출은 비용이 발생할 수 있어 smoke 호출은 수행하지 않았다.
- 기존 작업 트리에 스킬 설치 관련 변경과 미추적 파일이 있어 건드리지 않았다.

## 7. Skill Impact Check

- 갤러리 숨김 기준이 공통 유틸로 이동했지만, 새 운영 정책이나 반복 개발 절차 변경은 아니다.
- 프로젝트 전용 스킬 문서 업데이트는 불필요하다고 판단했다.
