# 성장 애니메이션 반응형 이미지 크기 인수인계

작성일: 2026-04-29

## 1. 배경

`/mymissions/:id` 나의 미션 상세 화면의 성장 애니메이션 이미지가 고정된 정사각 영역 안에 표시되어, 가로형 성장 이미지가 실제 화면 폭을 충분히 활용하지 못하고 작게 보이는 문제가 있었다.

사용자 요청은 화면 비율에 맞춰 이미지의 가로 사이즈를 기준으로 표시하되, 데스크탑에서는 최대 1024px 정도까지만 커지도록 하는 것이다.

## 2. 원인

`client/src/components/ui/CreationTreeProgress.tsx`에서 이미지 표시 영역이 아래처럼 고정 크기로 구성되어 있었다.

- 컨테이너: `min-h-[340px]`
- 이미지 래퍼: `w-80 h-80 sm:w-96 sm:h-96`
- 이미지: `w-full h-full object-contain`

이 구조 때문에 가로형 이미지도 320~384px 정사각 박스 안에 축소되어 표시됐다.

## 3. 변경 내용

### `client/src/components/ui/CreationTreeProgress.tsx`

- 상태 배지의 음수 마진(`mb-[-1rem]`)을 제거하고 `mb-3`으로 변경했다.
- 이미지 영역의 고정 최소 높이(`min-h-[340px]`)를 제거했다.
- 이미지 래퍼를 `w-full max-w-[1024px]`로 변경했다.
- 이미지는 `block w-full h-auto object-contain`으로 변경하여 원본 비율을 유지하면서 가로폭 기준으로 표시되게 했다.

## 4. 기대 동작

- 모바일: 부모 컨테이너 폭에 맞춰 이미지가 줄어든다.
- 데스크탑: 이미지가 화면 폭을 활용하되 최대 1024px까지만 커진다.
- 가로형 성장 이미지가 더 이상 작은 정사각 카드처럼 보이지 않는다.

## 6. 주의사항

- 이번 변경은 성장 애니메이션 이미지 표시 크기만 조정했다.
- `/mymissions/:id` 화면의 브라우저 직접 검증은 요청에 `!!테스트!!`가 없어 수행하지 않았다.
- 기본 fallback 트리 이미지도 같은 반응형 규칙을 사용하므로, 기본 이미지가 과하게 크게 느껴질 경우 이후 `stageImages.length > 0` 여부로 커스텀 이미지와 기본 트리 이미지를 분기할 수 있다.
