# 미션 콜라주 사용자 노출 숨김

작성일: 2026-05-06

## 1. 작업 개요

문화센터 세부미션 이미지 제출에서 사용하는 콜라주 기능은 유지하되, 일반 사용자 화면의 AI 생성 메인, 사용자 갤러리, 갤러리 필터, 직접 콜라주 제작 경로에서 노출되지 않도록 정리했다.

## 2. 변경 파일

- `client/src/App.tsx`
- `client/src/pages/home.tsx`
- `shared/constants.ts`
- `server/routes/gallery.ts`
- `server/routes/image.ts`
- `server/services/collageServiceV2.ts`

## 3. 핵심 변경 내용

- AI 생성 메인 화면의 정적 `콜라주` 버튼을 제거했다.
- `/gallery-collage`, `/collage-builder` 직접 접근 시 홈(`/`)으로 이동하도록 변경했다.
- 사용자 갤러리 공통 필터 목록에서 `콜라주` 필터를 제거했다.
- `/api/gallery` 전체 조회에서 `style = collage`, `categoryId = collage` 이미지를 제외했다.
- `/api/gallery?filter=collage` 요청은 빈 배열을 반환하도록 처리했다.
- 사용자 이미지 보조 조회 라우트(`/api`, `/api/recent`, `/api/list`)에서도 콜라주를 제외하도록 보강했다.
- 새로 생성되는 콜라주 이미지의 `metadata`에 `purpose: "mission_submission"`, `hiddenFromGallery: true`를 저장하도록 했다.

## 4. 변경 이유

콜라주는 문화센터 이미지 제출을 돕는 내부 보조 기능으로 필요하지만, 생성 즉시 일반 갤러리에 저장되면서 제출 여부와 무관하게 사용자 갤러리를 지저분하게 만들고 있었다. 기존 콜라주 결과물은 삭제하지 않고 조회 단계에서 숨기는 방식으로 처리해 과거 제출 URL이나 관리자 확인 가능성에 대한 부작용을 줄였다.

## 5. 동작 확인 상태

- `npm run build` 성공
- `npx tsc --pretty false --noEmit` 전체 검사는 기존 Persona/마일스톤 보류 오류로 실패
- 변경 파일 기준 TypeScript 오류 필터 결과: 관련 오류 없음
- API 검증 성공
  - `/api/gallery`: 콜라주 누수 0건
  - `/api/gallery?filter=collage`: 0건
  - `/api/list`: 콜라주 누수 0건
  - `/api/recent?categoryId=collage`: 0건
- 브라우저 검증 성공
  - `/` AI 생성 메인에서 `콜라주` 버튼 미노출
  - `/gallery` 사용자 갤러리에서 `콜라주` 필터/텍스트 미노출
  - `/gallery-collage`, `/collage-builder` 직접 접근 시 홈(`/`)으로 이동
  - `/party` 기본 편집기 화면에서 `콜라주` 텍스트 미노출

## 6. 주의사항

- 콜라주 생성 API와 `MissionCollageModal`은 유지했다. 문화센터 이미지 제출 흐름에서 내부 기능으로 계속 사용할 수 있다.
- 기존 DB에 저장된 콜라주 이미지는 삭제하지 않았다. 사용자 조회에서만 숨긴다.
- `server/routes/image.ts`는 현재 `app.use('/api', imageRouter)`로 마운트되어 있어 보조 조회 경로가 `/api/list`, `/api/recent` 형태다. `/api/image/list`는 SPA HTML을 반환하므로 검증 대상이 아니다.
- `client/src/pages/gallery-collage.tsx` 파일은 라우트에서 분리됐지만 삭제하지 않았다. 향후 관리자/내부 전용 제작 화면으로 되살릴 가능성이 있으면 재사용할 수 있다.

## 7. 다음 작업 참고사항

- 관리자 화면에서는 콜라주를 계속 볼지, 관리자 이미지 갤러리에서도 숨길지 정책 결정이 필요하다.
- 장기적으로는 `images` 테이블에 `visibility` 또는 `hiddenFromGallery` 정규 컬럼을 추가하면 metadata 파싱 없이 더 명확하게 제어할 수 있다.
