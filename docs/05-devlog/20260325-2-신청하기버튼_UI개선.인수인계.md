**작업일**: 2026-03-25  
**작업자**: Antigravity

---

## 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `client/src/pages/mission-detail.tsx` | applicationSubMission 선언 위치 조정, dynamicTabs 필터 조건부 변경 |
| `client/src/pages/mission-detail.tsx` | `ApplicationButton` 컴포넌트 신규 추가 (탭바 내 렌더링) |

---

## 핵심 변경 내용

### Before
- 신청 세부미션이 탭바에 소형 탭으로 표시
- 페이지에 `ApplicationButton` floating 버튼이 탭바 위에 별도 노출 (중간에 만든 버전)

### After (최종)
- **미승인 상태**: 하단 탭바 좌측에 [신청하기] 버튼 크게 표시 → 클릭 시 기존 신청 모달 오픈
- **승인 후**: dynamicTabs에 신청 탭 포함 → "신청완료" 표시 + 취소 버튼 사용 가능
- floating 버튼 없음 → 탭바에 통합된 깔끔한 구조

---

## 상태별 버튼 표시 (탭바 좌측)

| 상태 | 버튼 텍스트 | 색상 |
|------|------------|------|
| 미신청 (기간 중) | 신청하기 | 퍼플 |
| 신청 제출됨 (검토 중) | 신청 중 + 서브텍스트 | 블루 |
| 대기 등록 | 대기 중 | 오렌지 |
| 취소 후 재신청 가능 | 다시 신청하기 | 퍼플 |
| 기간 전 | 신청 준비 중 | 그레이 |
| 기간 후 (마감) | 신청 마감 | 그레이 |
| **승인됨** | 탭바에 신청완료 탭으로 표시 | 그린 (기존 탭) |

---

## 동작 확인 상태

- ✅ [신청하기] 버튼이 탭바 내에서 크게 표시됨
- ✅ 클릭 시 기존 신청 모달(행사 신청) 정상 오픈
- ✅ [완료선물] 버튼은 오른쪽에 유지

---

## 주의사항

- `applicationSubMission` 선언이 반드시 `isApplicationApproved` 선언보다 위에 있어야 함 (JS 참조 순서)
- 신청 탭은 `dynamicTabs` 안에서 `isApplicationApproved` 조건으로 포함 여부 결정
- 다른 세부미션 탭(제출, 리뷰 등)은 신청 승인 여부와 관계없이 항상 표시됨

## 핵심 변경 내용

### Before
- 세부미션의 액션타입 "신청"도 하단 탭바에 소형 아이콘+텍스트로 표시

### After
- 하단 탭바에서 "신청하기" 탭 완전 제거
- 페이지 콘텐츠 하단, 탭바 바로 위에 **전폭 신청하기 버튼** 표시

---

## 변경 배경 및 의사결정 이유

- 신청은 다른 액션(제출, 리뷰 등)과 성격이 다름 — 미션 시작 전 1회성 게이트
- 작은 탭 아이콘으로 표현하면 사용자가 신청 진입점을 놓치기 쉬움
- 백엔드/DB 변경 없이 UI 렌더링 방식만 수정하여 리스크 최소화

## 주의사항 / 알려진 이슈

- `applicationSubMission`은 세부미션 중 isActive=true & actionType.name='신청'인 **첫 번째** 항목을 사용
- 여러 개의 신청 세부미션이 등록된 경우 첫 번째만 버튼으로 표시됨 (현재 실운영 데이터에는 미션당 1개)
- 신청 취소 버튼은 기존 모달 내에 존재하며 그대로 유지됨
