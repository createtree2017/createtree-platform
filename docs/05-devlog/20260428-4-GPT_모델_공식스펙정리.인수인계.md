# 일일 개발 인수인계

## 1. 작업 개요
- **작업일**: 2026.04.28
- **작업명**: OpenAI GPT-Image 계열 공식 스펙에 맞춘 코드 전면 개편
- **작업 목적**: API 인증 및 권한이 정상화될 미래(5월 초 롤아웃)를 대비하여, 기존에 잘못된 정보로 짜여진 `dall-e-3` 우회 로직을 삭제하고 OpenAI가 발표한 공식 스펙(`gpt-image-2`, `gpt-image-1`, `gpt-image-1-mini`)으로 정확히 연동되도록 코드를 정돈합니다.

## 2. 변경 파일 목록
- `shared/schema.ts`: `openai_gpt2_mini` 제거, `openai_gpt1_mini` 추가
- `server/services/openai.ts`: 우회 매핑 제거 (`model: "gpt-image-2"` 등 실제 이름 사용), 403 에러 발생 시 명확한 에러 투척
- `server/routes/image.ts`: 텍스트 전용 모드는 `v1/images/generations`를 호출하고, 이미지 변환(Edit) 모드는 `v1/images/edits`를 호출하도록 분기 구조 단순화 및 합침
- `client/src/components/ImageGenerationTemplate.tsx`: UI 상에서 `GPT-Image-2 Mini` 라벨을 `GPT-Image-1 Mini`로 변경
- `client/src/components/modal/admin/ConceptFormModal.tsx`: 관리자 설정창에서도 미니 모델 이름을 `GPT-Image-1 Mini`로 변경
- `client/src/hooks/useModelCapabilities.ts`: aspect ratio 폴백 목록에 `openai_gpt1_mini` 반영

## 3. 핵심 변경 내용
- **임시 폴백/우회 코드 완전 제거**: 이전에는 `gpt-image-2` 선택 시 억지로 `dall-e-3`를 부르는 코드가 얽혀 있었으나, 이를 제거하고 공식 `model: "gpt-image-2"` 파라미터를 던지도록 깔끔하게 정리했습니다.
- **저비용 모델 라인업 교체**: `gpt-image-2-mini`라는 비공식 모델명을 걷어내고, 존재하는 공식 저비용 모델인 `gpt-image-1-mini`로 스키마 및 프론트엔드 라벨을 교체했습니다.
- **403 에러 핸들링**: OpenAI 프로젝트에 아직 `gpt-image-2` 권한이 풀리지 않은 상태에서 API 호출이 일어나면 403 에러를 반환합니다. 이를 잡아내어 "현재 OpenAI 프로젝트에 해당 모델 권한 없음" 이라는 명확한 원인을 던지도록 개선했습니다.

## 4. 알려진 이슈 및 주의사항
- **DB 스키마 동기화**: `shared/schema.ts`의 Enum 기본값이 변경되어 `npx drizzle-kit push`를 실행했습니다. 기존 DB에 `openai_gpt2_mini` 값이 들어간 행이 있다면, 프론트에서 해당 모델 아이콘은 무시되거나 빈 값으로 처리될 수 있습니다.
- **OpenAI 롤아웃 대기**: 5월 초에 개발자 API 권한(Tier)에 `gpt-image-2`가 노출되면, 관리자 콘솔 `Limits`에서 토글을 켜는 것만으로 코딩 없이 즉시 2.0 모델이 동작하게 됩니다.
