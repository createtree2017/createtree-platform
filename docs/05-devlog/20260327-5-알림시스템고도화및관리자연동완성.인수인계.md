## 1. 핵심 변경 내용

### 🔧 [백엔드] push.service.ts — 발송 내역 로그 누락 수정
- **변경 배경**: 관리자가 수동 푸시를 발송해도 `push_delivery_logs`에 기록이 남지 않았음
- **근본 원인**: userId=24에 Android FCM 토큰 3개가 등록되어 있어 `tokens.length > 0` → `sendBatch()` 실행 → FCM 발송 에러 throw → catch 블록에서 `insertDeliveryLog()` 호출 없이 return
- **수정**: 
  - `sendToUsers()` catch 블록에 `insertDeliveryLog()` 추가 (에러 시에도 무조건 로그 남김)
  - `sendToTopic()` catch 블록에도 동일 처리 추가
  - `tokens.length === 0` 분기에서도 `insertDeliveryLog()` 추가 (토큰 없는 사용자 대상 발송 추적)

### 🎨 [프론트엔드] notifications.tsx — 바텀 모달 상세보기
- **변경 배경**: 알림 리스트에서 클릭 시 아무 상세 화면 없이 딥링크로만 이동 → 사용자가 내용을 못 봄
- **수정**:
  - 알림 클릭 → `vaul` Drawer(바텀 모달)로 상세 내용 표시
  - 표시 항목: 유형 아이콘 + 제목 + 전체 본문 + 발송 시각
  - `actionUrl` 있으면 "바로가기" 버튼 표시 (외부 URL → 새탭, 앱 내부 → wouter 라우팅)
  - `admin_push` 알림 유형에 파란색 메가폰 아이콘 매핑 추가
  - `useState` import 추가 (누락 수정)

### 🔧 [백엔드] push-admin-routes.ts — 디버그 로그 추가
- push-send 엔드포인트 진입 시 `[push-send]` 디버그 로그 추가 (targetType, adminId, targetIds 출력)

## 3. 최종 검증 결과

### ✅ 관리자 발송 → 사용자 알림함 연동
- 관리자 수동 푸시 발송 시 `notifications` 테이블에 정상 INSERT → 사용자 "나의 알림"에 표시
- `createNotification()` 비동기 호출로 API 응답 속도에 영향 없음

### ✅ 발송 내역 로그 (push_delivery_logs)
- FCM 토큰 있는 사용자 대상 발송: sendBatch → 성공/실패 관계없이 로그 기록
- FCM 토큰 없는 사용자 대상 발송: tokens.length=0 분기에서도 로그 기록
- 관리자 "발송 내역" 탭에 정상 표시 확인

### ✅ 바텀 모달 상세보기
- 알림 클릭 → Drawer 모달 → 제목/본문/시각/아이콘 정상 표시
- actionUrl이 있는 알림: "바로가기" 버튼 표시 + 클릭 이동 작동
- 외부 URL (https://...) → 새 탭 열기
- 앱 내부 경로 (/mymissions 등) → wouter 라우팅

### ✅ 알림 CRUD API
- `GET /api/notifications` — 페이지네이션 + 읽지않은 개수 동시 반환
- `GET /api/notifications/unread-count` — 하단 네비 뱃지용 경량 API
- `POST /api/notifications/read/:id` — 단건 읽음 처리
- `POST /api/notifications/read-all` — 전체 읽음 처리
- 모든 API에 `requireAuth` 미들웨어 + `req.user!.userId` 기반 필터링

### ✅ 코드 품질
- `notifications.tsx` — 364줄 (1000줄 제한 이내)
- `push.service.ts` — 321줄 (1000줄 제한 이내)
- `push-admin-routes.ts` — 371줄 (1000줄 제한 이내)
- catch 블록에서도 insertDeliveryLog를 try-catch로 감싸 이중 안전장치

## 5. 주의사항 / 알려진 이슈

- **서버 재시작 필수**: `tsx`가 `--watch` 없이 실행되므로, 백엔드 코드 수정 시 반드시 서버 재시작 필요
- **FCM 토큰 상태**: userId=24에 Android 토큰 3개 활성 상태. PC 브라우저에서 FCM 발송 시도 → 에러는 정상 (디바이스 토큰이 Android 전용이므로). 발송 자체는 실패하지만 로그+알림함 저장은 정상
- **연관 파일**: `server/routes/misc-routes.ts` (알림 API 엔드포인트) — 이번에 수정하지 않았지만 알림 시스템의 핵심 API 호스트

## 6. 다음 작업 참고사항 (지금은 불필요, 향후 스케일 확장 시 검토)

- **FCM 에러 유형 분석**: `sendBatch()` 내부에서 이미 `registration-token-not-registered` / `invalid-registration-token` 감지 → 좀비 토큰 자동 비활성화 처리 구현 완료. 현재 추가 분석 불필요. 다만 프로덕션에서 대량 실패 발생 시 catch 로그를 통해 에러 유형 모니터링 가능
- **Bulk Insert 최적화**: 현재 `Promise.allSettled` 개별 `createNotification()` 호출 — 수십 명 발송에는 성능 이슈 없음. **유저 1,000명 이상 전체 발송** 시에만 Drizzle `db.insert().values([...])` Bulk Insert로 전환 검토
