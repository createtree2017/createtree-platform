## 1. 변경 개요

| 변경 대상 | Before | After |
|-----------|--------|-------|
| 표준 OpenAI 모델 | `gpt-image-1` | **`gpt-image-1.5`** |

### 최종 모델 구성

```
[OpenAI]
  표준형: openai       → gpt-image-1.5      ($0.034/이미지)
  경제형: openai_mini   → gpt-image-1-mini   ($0.011/이미지)

[Gemini]
  표준형: gemini_3_1   → Gemini 3.1 Flash    ($0.067/이미지)
  프로형: gemini_3     → Gemini 3.0 Pro      ($0.134/이미지)
```

## 3. 핵심 설계 결정

### 3-1. `openai` 키 유지 (DB 마이그레이션 불필요)
- 기존 `"openai"` 식별자는 그대로 유지하고 뒤에서 호출하는 모델명만 변경
- DB의 기존 데이터와 호환성 유지

### 3-2. `openai-dalle3.ts`에 `modelName` 파라미터 추가
- `callGptImage1Api`, `transformWithOpenAI`, `transformWithOpenAIMulti` 3개 함수에 `modelName` 매개변수 추가
- 기본값 `'gpt-image-1.5'` → `openai_mini` 선택 시 `'gpt-image-1-mini'` 전달
- 같은 서비스 로직 재사용으로 코드 중복 방지

### 3-3. `openai_mini` 비율 제한
- `openai_mini` (`gpt-image-1-mini`)는 최대 1024px만 지원
- 비율은 `1:1`만 기본 제공

## 4. ⚠️ 롤백 조치 및 openai_mini 제거 내역 (수정 업데이트)

API 접근 권한 문제(500 에러)로 인해 다음과 같이 조치를 진행하고 추가했던 변경을 원복/제거했습니다.

1. **`gpt-image-1.5` → `gpt-image-1` 롤백**
   - OpenAI API 호출 시 `gpt-image-1.5` 모델 지원/권한 이슈 발생.
   - 백엔드 (`openai.ts`, `openai-dalle3.ts`, `image.ts` 등 4개 파일) 및 프론트엔드 라벨을 가장 안정적인 `gpt-image-1`으로 롤백 처리 완료.
2. **`openai_mini` 모델 완전 제거**
   - 경제형 옵션을 사용자 UI에서 완전히 숨김.
   - `shared/schema.ts` enum 제거 및 시스템 설정 기본값(fallback/placeholder) 롤백.
   - `ImageGenerationTemplate`, `ConceptFormModal`, `SystemSettings` 컴포넌트에서 관련 체크박스 및 버튼 제거.
   - `misc-routes`, `admin-routes`, `image.ts` 라우트 등에서 `openai_mini` 관련 하드코딩된 분기 전면 삭제.
