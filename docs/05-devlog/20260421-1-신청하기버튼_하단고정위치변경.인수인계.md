# 신청하기 버튼 하단 고정 위치 변경

## 1. 개요
- **작업 일자**: 2026-04-21
- **작업자**: Antigravity
- **작업 목적**: 사용자가 미션 상세 페이지에서 '신청하기' 버튼을 스크롤 위치와 무관하게 항상 접근할 수 있도록 하단 탭바 바로 위쪽 고정 영역으로 이동시킵니다.

## 2. 변경된 파일 목록
- `client/src/pages/mission-detail.tsx`

## 3. 핵심 변경 내용
1. **ApplicationStatusCard 컴포넌트 이동**: 
   - 스크롤되는 본문 영역(`<!-- 신청 상태 카드 -->` 주석 처리 부분)에 있던 `<ApplicationStatusCard />`를 제거했습니다.
   - `fixed bottom-0` 클래스를 가진 하단 고정형 바(Action Tab Bar) 내부로 컴포넌트를 이동시켰습니다. 탭들 바로 위쪽에 가로 전체 너비(`w-full`)를 가지도록 배치했습니다.
2. **하단 패딩(pb) 증가**:
   - 하단 고정 바의 높이가 기존 탭만 있을 때보다 높아짐에 따라, 페이지 맨 아래까지 스크롤 시 본문 내용이 가려지지 않도록 메인 컨테이너의 하단 패딩을 `pb-24`에서 `pb-44`로 상향 조정했습니다.
3. **불필요한 마진 제거**:
   - `ApplicationStatusCard` 컴포넌트 내부 최상단 div의 `mb-6`을 제거하여 하단 탭바와 너무 멀어지지 않도록 깔끔하게 정돈했습니다.

## 4. 동작 확인 상태
- [x] 페이지 중앙에서 '신청하기' 카드 제거 확인
- [x] 스크롤 시 '신청하기' 버튼과 '제출하기/리뷰작성' 탭이 화면 하단에 함께 고정되어 표시되는지 확인
- [x] 페이지 끝까지 스크롤 시 본문(미션 안내 이미지 등)이 하단 바에 가려지지 않고 온전히 보이는지 확인

## 5. 주의사항 / 연관 파일
- 본 변경사항은 `mission-detail.tsx` 내 UI 레이아웃 변경에 한정됩니다. 기능적인 로직은 변경되지 않았습니다. 
- 이후 하단 탭바에 다른 액션 컴포넌트를 추가할 경우 `pb-44`를 추가적으로 조정해야 할 수 있습니다.
