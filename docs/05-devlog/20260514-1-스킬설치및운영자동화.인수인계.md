# 스킬 설치 및 운영 자동화 인수인계

## 변경 파일

- `.agents/skills/`
- `.agents/scripts/generate_skills_index.py`
- `.agents/scripts/validate_skills.py`
- `.agents/skills/SKILLS_INDEX.md`
- `AGENTS.md`
- `GEMINI.md`
- `package.json`

## 핵심 변경 내용

- Codex repo-local 스킬 위치를 `.agents/skills/`로 구성했다.
- AI문화센터 전용 스킬 4개를 추가했다.
  - `createtree-aicc-platform`
  - `aicc-ai-image-pipeline`
  - `aicc-capacitor-fcm`
  - `aicc-railway-drizzle`
- 공통 품질 스킬 7개를 설치했다.
  - `security-best-practices`
  - `react-best-practices`
  - `web-design-guidelines`
  - `web-quality-audit`
  - `performance`
  - `accessibility`
  - `use-railway`
- `.agents/skills/SKILLS_INDEX.md`를 자동 생성 목록 문서로 추가했다.
- `npm run skills:sync`, `npm run skills:check` 명령을 추가했다.
- `npm run verify`가 스킬 검증을 먼저 수행하도록 연결했다.
- `AGENTS.md`와 `GEMINI.md`에 Skill Impact Check와 운영 정책 변경 기준을 추가했다.

## 왜 변경했는지

AI문화센터는 이미지 생성, FCM 푸시, Railway DB, 앱/관리자 운영 흐름이 복잡하게 연결되어 있어 일반 지침만으로는 반복 작업 기준이 누락될 수 있다. 프로젝트별 스킬과 자동 목록 검증을 두어 신규 기능 개발 후 스킬이 낡지 않게 관리하기 위해 변경했다.

## 검증 결과

- `npm run skills:sync`: 통과
- `npm run skills:check`: 통과
- `npm run verify`: 스킬 검증은 통과, 기존 환경 검증에서 `DATABASE_URL`, `SESSION_SECRET`, `JWT_SECRET` 누락으로 실패

## 주의사항

- `SKILLS_INDEX.md`는 직접 수정하지 말고 `npm run skills:sync`로 재생성한다.
- 기능 개발, 기존 기능 변경, 업데이트 완료 시 Skill Impact Check를 수행한다.
- 배포/DB/보안/개인정보/의료광고/AI provider/금전/자동화 권한 정책 변경은 사용자 확인 후 스킬에 반영한다.
- git add, commit, push는 수행하지 않았다.

## 추가 보강

- 컨텍스트와 개발 속도 보호를 위해 자동 호출 범위를 프로젝트 전용 핵심 스킬 4개로 제한했다.
- 공통 품질 스킬 7개는 설치 상태를 유지하되 `$스킬명`으로 명시 호출할 때만 사용하도록 `allow_implicit_invocation: false`로 변경했다.
- `npm run skills:sync`와 `npm run skills:check` 재검증을 통과했다.
