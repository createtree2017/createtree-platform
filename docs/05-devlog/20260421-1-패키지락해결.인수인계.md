# 2026-04-21 인수인계 (패키지 버전 충돌 해결)

## 변경 파일
- package-lock.json

## 핵심 변경 내용
- @capacitor/assets 내 sharp 라이브러리의 무결성 검증 오류 해결을 위해 package-lock.json 파일을 삭제 후 npm install로 재생성함.
- Railway 배포 시 npm ci 과정에서 발생하던 무결성 검증(Integrity check failed) 오류 수정됨.

## 왜 변경했는지 (의사결정 이유)
- npm ci는 package-lock.json에 기록된 패키지 해시와 실제 다운로드된 바이너리의 해시를 엄격하게 비교함.
- sharp의 최신 linux-x64 바이너리(v8.14.5)가 GitHub 릴리즈에서 재업로드되면서 해시가 변경되었고, 기존 lock 파일의 해시와 불일치하여 배포 실패가 발생함.
- 이 문제는 코드 오류가 아니라 캐시/바이너리 업데이터 문제이므로, 가장 깔끔하고 권장되는 해결책인 lock 파일 재생성(npm install 다시 실행)을 수행함.

## 동작 확인 상태
- ✅ (로컬 npm install 시도 후 정상 작동 확인, Railway 배포 진행 중)

## 주의사항 / 알려진 이슈
- 향후 비슷한 패키지 무결성 오류가 발생하면, 다시 한번 package-lock.json을 재생성하여 해결 가능한지 확인 필요.
