# 큰미션 공개범위 병원명 표시

작성일: 2026-05-24

## 1. 작업 개요

관리자페이지 `메뉴관리 > 나의미션 > 큰미션 관리` 목록의 공개범위 컬럼에서 병원 전용 큰미션이 단순히 `병원`으로 표시되던 부분을 실제 지정 병원명으로 표시하도록 수정했다.

## 2. 연결 문서

- Plan: 없음 (기존 목록 표시 문구만 바꾸는 Quick Fix)
- Design: 없음

## 3. 변경 파일

- `client/src/components/admin/BigMissionManagement.tsx`

## 4. 핵심 변경 내용

- 큰미션 목록의 공개범위 배지에서 `visibilityType === "hospital"`인 경우 기존 고정 문구 `병원` 대신 API 응답의 `mission.hospital?.name`을 표시하도록 변경했다.
- 병원 관계 데이터가 없는 예외 상황에서는 `병원명 없음`으로 표시해 데이터 누락을 확인할 수 있게 했다.
- 서버 API는 이미 `hospital: true` 관계를 포함해 큰미션 목록을 내려주고 있어 변경하지 않았다.

## 5. 왜 변경했는지

관리자가 큰미션 목록을 볼 때 `병원`이라는 범위명만으로는 어느 거래처 병원에 지정된 미션인지 바로 구분할 수 없었다. 생성/수정 폼에서는 병원을 선택하고 있으므로 목록에서도 같은 병원명을 보여주는 것이 관리 효율에 맞다.

## 6. 동작 확인 상태

- `npm run build`: 성공
- `npm run check -- --pretty false`: 실패. 기존 `PersonaCategoryManagement.tsx`, `PersonaManagement.tsx`, `server/services/milestones.ts` 타입 오류가 남아 있으며 이번 변경 파일 관련 신규 오류는 확인되지 않았다.
- `git diff --check`: 통과 (CRLF 변환 경고만 표시)
- 브라우저 검증: `!!테스트!!` 키워드가 없어 프로젝트 정책에 따라 직접 실행하지 않음

## 7. 주의사항 / 알려진 이슈

- `GET /api/admin/big-missions` 응답에서 `hospital` 관계가 누락되면 목록에는 `병원명 없음`이 표시된다.
- 이번 변경은 관리자 목록 표시만 바꾸며, 공개범위 저장 방식이나 사용자 노출 필터링 로직은 변경하지 않았다.

## 8. 연관 파일

- `server/controllers/admin/admin.big.mission.controller.ts`: 큰미션 목록 API에서 `hospital: true` 관계를 포함한다.
- `client/src/components/modal/admin/BigMissionModals.tsx`: 큰미션 생성/수정 폼에서 병원을 선택한다.

## 9. Skill Impact Check

- 반복 작업 규칙, API 계약, DB 구조, 운영 정책 변경은 없다.
- 기존 프로젝트 스킬 설명과 운영 규칙에 반영할 변경 사항은 없어 스킬 업데이트는 불필요하다.

# 최고관리자 나의미션 전체 큰미션 노출

추가 작업일: 2026-05-24

## 1. 작업 개요

최고관리자로 로그인해도 사용자 페이지 `/mymissions`에서 병원 전용 큰미션이 보이지 않던 문제를 수정했다. 문화센터 `/missions` 페이지처럼 최고관리자는 공개범위와 관계없이 활성화된 큰미션을 모두 볼 수 있도록 맞췄다.

## 2. 연결 문서

- Plan: `docs/01-plan/features/1-20260407-리워드미션_선물신청_시스템.plan.md`
- Design: 없음

## 3. 변경 파일

- `server/controllers/user/user.big.mission.controller.ts`
- `docs/01-plan/features/1-20260407-리워드미션_선물신청_시스템.plan.md`

## 4. 핵심 변경 내용

- `GET /api/big-missions` 목록 필터에서 `memberType === "superadmin"`이면 모든 활성 큰미션을 통과하도록 변경했다.
- 일반 사용자 조건은 기존처럼 `public` 또는 자기 `hospitalId`와 일치하는 `hospital` 공개범위만 유지했다.
- Plan 작업 관리표에 최고관리자 나의미션 전체 노출 항목을 추가하고 `검증중`으로 표시했다.

## 5. 왜 변경했는지

최고관리자는 관리자 관점에서 병원별 큰미션 상태를 확인해야 하지만, 기존 로직은 `dev` 미션만 예외로 추가하고 병원 전용 미션은 사용자의 `hospitalId`가 일치해야만 보여줬다. 최고관리자는 보통 소속 병원이 없기 때문에 병원 전용 큰미션이 목록에서 빠졌다.

## 6. 동작 확인 상태

- `npm run build`: 성공
- `npm run check -- --pretty false`: 실패. 기존 `PersonaCategoryManagement.tsx`, `PersonaManagement.tsx`, `server/services/milestones.ts` 타입 오류가 남아 있으며 이번 변경 파일 관련 신규 오류는 확인되지 않았다.
- 브라우저 검증: `!!테스트!!` 키워드가 없어 프로젝트 정책에 따라 직접 실행하지 않음

## 7. 주의사항 / 알려진 이슈

- 이번 변경은 최고관리자의 사용자 페이지 조회 범위만 넓힌다. 일반 사용자와 병원회원의 노출 범위는 기존과 동일하다.
- `/api/big-missions/:id` 상세 API는 기존에도 활성 큰미션 상세를 반환하고 있어 이번 목록 필터 수정 범위에는 포함하지 않았다.

## 8. 연관 파일

- `client/src/pages/mymissions.tsx`: `/api/big-missions` 응답을 렌더링하는 사용자 페이지
- `server/services/mission/mission.user.service.ts`: 문화센터 `/missions`의 최고관리자 전체 조회 기준

## 9. Skill Impact Check

- 큰미션 사용자 목록의 최고관리자 권한 조건을 문화센터와 일치시킨 변경이다.
- API 계약, DB 구조, 배포/보안 운영 원칙 변경은 없어 스킬 업데이트는 불필요하다.

# 사용자 나의미션 메뉴 노출 복구

추가 작업일: 2026-05-24

## 1. 작업 개요

일반 사용자 화면의 하단 네비게이션에서 `나의미션` 메뉴가 보이지 않던 문제를 수정했다. 기존에는 프론트 메뉴 훅에서 `my-missions` 메뉴를 최고관리자에게만 노출하도록 필터링하고 있었다.

## 2. 연결 문서

- Plan: 없음 (기존 메뉴 노출 조건을 되돌리는 Quick Fix)
- Design: 없음

## 3. 변경 파일

- `client/src/hooks/useMainMenus.ts`
- `client/src/pages/profile.tsx`

## 4. 핵심 변경 내용

- `useMainMenus()`에서 `my-missions`를 `superadmin`에게만 남기던 클라이언트 필터를 제거했다.
- `/api/main-menus` 실패 시 사용하는 fallback 메뉴에도 `나의미션`과 `/mymissions` 경로를 포함했다.
- 마이페이지 내부 `나의미션` 링크를 `/my-missions`에서 현재 메인 메뉴 기준 경로인 `/mymissions`로 정리했다.

## 5. 왜 변경했는지

관리자 메뉴관리에서 `나의미션` 메뉴가 활성화되어도 프론트 훅이 일반 사용자에게 해당 메뉴를 다시 제거하고 있어, 사용자 화면에서 메뉴가 보이지 않았다. 이제 메뉴 활성/비활성의 기준은 기존처럼 `/api/main-menus`가 내려주는 활성 메뉴 목록을 따르도록 복구했다.

## 6. 동작 확인 상태

- `npm run build`: 성공
- `npm run check -- --pretty false`: 실패. 기존 `PersonaCategoryManagement.tsx`, `PersonaManagement.tsx`, `server/services/milestones.ts` 타입 오류가 남아 있으며 이번 변경 파일 관련 신규 오류는 확인되지 않았다.
- 브라우저 검증: `!!테스트!!` 키워드가 없어 프로젝트 정책에 따라 직접 실행하지 않음

## 7. 주의사항 / 알려진 이슈

- `/api/main-menus`는 여전히 `isActive = true`인 메뉴만 반환한다. 따라서 관리자 메뉴관리에서 `나의미션`이 비활성화되어 있으면 사용자에게도 보이지 않는 것이 정상이다.
- 이번 변경은 메뉴 노출 조건만 복구하며, `/mymissions`의 큰미션 목록 조회 권한 로직은 이전 작업의 정책을 그대로 유지한다.

## 8. 연관 파일

- `server/routes/public-routes.ts`: `/api/main-menus`가 활성 메인 메뉴만 반환한다.
- `client/src/components/BottomNavigation.tsx`: `useMainMenus()`의 `navItems`를 하단 네비게이션으로 렌더링한다.

## 9. Skill Impact Check

- 반복 작업 규칙, API 계약, DB 구조, 운영 정책 변경은 없다.
- 기존 프로젝트 스킬 설명과 운영 규칙에 반영할 변경 사항은 없어 스킬 업데이트는 불필요하다.
