# Railway 재배포 트리거 인수인계

작성일: 2026-04-29

## 1. 배경

Railway에서 PR #45 배포가 Metal builder에 예약된 뒤 실제 빌드가 실행되지 않아 실패했다.
Railway 진단 결과도 일시적인 인프라 오류로 판단되어 동일 코드의 재배포가 필요했다.

## 2. 변경 내용

- 코드 변경 없이 GitHub 자동 배포를 다시 트리거하기 위한 문서 전용 변경을 추가했다.
- 운영 중인 기존 배포는 PR #44 상태로 유지되어 있으며, 새 커밋 푸시 후 Railway가 최신 GitHub 커밋 기준으로 다시 빌드하도록 한다.

## 3. 검증

- 문서 변경만 있으므로 별도 빌드 검증은 수행하지 않았다.
- 푸시 후 Railway `createtree-platform` 서비스의 Deployments 탭에서 새 배포 생성 여부를 확인해야 한다.

## 4. 주의사항

- 새 배포 로그에서 `npm ci`, `npm run build`, `vite build` 단계가 출력되는지 확인한다.
- 다시 `scheduling build on Metal builder` 이후 로그가 멈추면 Railway Metal builder 인프라 문제로 보고 재시도하거나 Railway 지원에 문의한다.
