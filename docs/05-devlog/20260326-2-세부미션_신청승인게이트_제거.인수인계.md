# 세부미션 신청승인 게이트 제거

## 1. 개요
* **날짜:** 2026-03-26
* **목적:** 신청(application) 세부미션의 승인 여부로 나머지 세부미션 탭 전체를 잠그던 기능 제거. 세부미션 접근 제어는 기존의 **순차 등급(sequentialLevel)** 시스템으로만 처리.

## 2. 변경 파일
* `client/src/pages/mission-detail.tsx`

## 3. 핵심 변경 내용

### 제거한 것 (2가지)
1. **하단 탭바 잠금 분기 (L836~845)**: `applicationSubMission && !isApplicationApproved` 조건으로 세부미션 탭 전체를 "선정 후 참여" 잠금 버튼으로 교체하던 로직 완전 제거
2. **안내 문구 (L799~803)**: "현재 미션의 세부 미션을 모두 완료하고 승인을 받으면 다음 미션에 접근할 수 있습니다." 문구 제거

### 유지되는 것
* **순차 등급 잠금**: `getTabUnlockStatus()`의 `sequentialLevel` 기반 탭별 개별 잠금은 그대로 유지
* **신청 상태 카드**: ApplicationStatusCard (신청하기 버튼, 검토 중, 선정완료 등 상태 표시) 유지
* **기간 잠금**: 세부미션 날짜 설정에 따른 시작 전/마감 처리 유지

## 4. 동작 확인 상태
* ✅ 신청이 있는 미션에서 신청 승인 여부와 무관하게 세부미션 탭 노출
* ✅ 순차 등급 설정된 세부미션은 이전 등급 완료 전까지 여전히 잠금 유지

## 5. 주의사항
* **서버 레벨 제거 미포함**: `server/services/mission/mission.user.service.ts`의 `getChildMissions()`에서 부모 미션 `APPROVED` 체크(`UNAUTHORIZED_FOR_CHILD`)는 이번에 제거하지 않음. 이는 **계층 미션(자식 미션 페이지 이동)**에 해당하는 별개 로직임.
* 추후 계층 미션의 상위 미션 승인 게이트도 제거 원하면 별도 요청 필요.
