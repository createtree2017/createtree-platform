# 20260320-3 DB 타임아웃 원인 분석 및 개발 지침

## 작성일
2026-03-20

## 1. 발생 상황
AI(Antigravity)가 데이터 직접 수정을 위해 `npx tsx -e "..."` 스크립트 방식으로 DB 접근을 시도했으나, 매번 타임아웃이 발생하여 작업이 지연됨.

## 2. 원인 분석

### DB 연결 방식
- **DB 서비스**: Neon Serverless PostgreSQL
- **연결 방식**: WebSocket (`drizzle-orm/neon-serverless`)
- **연결 설정**: `db/index.ts`

### 왜 타임아웃이 발생하나?
| 상황 | 이유 |
|------|------|
| `npm run dev` 실행 중 | 이미 연결 풀(Pool) 유지 중 → DB 요청 즉시 처리 |
| `npx tsx -e "..."` 별도 실행 | 새 프로세스가 DB에 **새 WebSocket 연결**을 맺어야 함 |
| Neon Serverless의 슬립 정책 | 비활성 시 DB 인스턴스가 슬립 → 깨어나는 데 10~30초 |
| 결과 | 새 스크립트 프로세스에서 연결 타임아웃 발생 |

### 해결 불가능한 구조적 문제인가?
- **아님.** Neon 유료 플랜($19/월)으로 업그레이드하면 슬립 없이 항상 즉시 연결 가능
- 단, 개발 중 `npx tsx -e` 방식은 빈도가 낮아 **업그레이드 대비 효과가 미미**

## 3. 올바른 개발 방식 (결론)

### ✅ DB 데이터 수정이 필요할 때
1. `npm run dev` 서버가 켜진 상태에서 **API 호출 방식** 사용
   - AI가 브라우저 에이전트로 `localhost:5000` API를 직접 호출
   - 관리자 페이지 UI에서 직접 클릭으로 처리
2. `npx tsx -e` 스크립트 방식은 **사용하지 않음**

### ❌ 피해야 할 방식
- 서버가 켜진 상태에서 별도 `npx tsx -e` 스크립트로 DB 직접 접근 시도
- 타임아웃 발생 후에도 같은 방식 반복 시도

## 4. AI에게 요구되는 행동 규칙
- `npm run dev`가 실행 중인 경우 → DB 작업 = **API 방식 또는 관리자 UI 방식**
- 스크립트 방식이 1회 타임아웃 → **즉시 API 방식으로 전환**
- 외부 서비스 연결이 안 되는 경우 → AI가 직접 시도하지 않고 **사용자에게 요청**

## 6. 추가 발생: PowerShell git 명령어 오류

### 발생 상황
오전 개발 종료 시 git commit/push 실행 중 PowerShell 파서 오류(`InvalidEndOfLine`) 발생으로 명령어가 멈춤.

### 원인
| 원인 | 설명 |
|------|------|
| `&&` 연산자 | PowerShell에서 미지원 (`git add -A && git commit` 형식 불가) |
| 한글 + 줄바꿈 커밋 메시지 | PowerShell 파서가 멀티라인/한글 특수문자 처리 실패 |

### 올바른 git 명령어 패턴
```
1. git add -A                                 (완료 확인 후)
2. git commit -m "feat: short english summary"  (영문 단문, 한 줄)
3. git push origin develop                    (별도 실행)
```

### 결론
- GEMINI.md에 "⚠️ PowerShell 터미널 규칙" 섹션 추가 완료
- 앞으로 모든 git 명령어는 위 패턴 준수
