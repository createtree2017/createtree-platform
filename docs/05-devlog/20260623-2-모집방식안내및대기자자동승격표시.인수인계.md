# 2026-06-23 모집 방식 안내 및 대기자 자동승격 표시 인수인계

## 작업 개요

주제미션 수정/추가 모달의 행사 정보 영역에 모집 방식 안내문구를 추가하고, 선착순 미션에서 대기자가 자동승격된 경우 관리자 검수 화면에서 `대기자 자동승격` badge로 식별되도록 구현했다.

## 연결된 PDCA 문서

- 없음: 기존 미션 신청/검수 흐름의 UI 안내 및 표시 보강

## 변경 파일

- `client/src/components/admin/MissionManagement.tsx`
- `client/src/components/modal/admin/SubmissionDetailModal.tsx`
- `server/services/mission/mission.submission.service.ts`

## 핵심 변경 내용

### 주제미션 모집 방식 안내문구

`MissionManagement.tsx`의 주제미션 수정/추가 모달에서 `행사 정보 (선택)` 영역의 `모집 인원` / `선착순` 입력 아래에 운영방식 안내문구를 추가했다.

- 모집 인원을 0 또는 비워두면 무제한으로 운영됩니다.
- 선착순을 선택하면 정원 내 신청자는 자동 승인되고, 정원이 차면 대기자로 등록됩니다.
- 선착순을 선택하지 않고 모집 인원을 입력하면 신청자는 관리자 승인 대기 상태로 등록됩니다.

### 선착순 대기자 자동승격 표시

`mission.submission.service.ts`의 `cancelApplication`에서 선착순 승인자가 참여취소하고 가장 빠른 대기자가 자동승격될 때, 승격된 제출의 `reviewNotes`에 아래 문구를 저장한다.

```text
대기자 자동승격: 기존 승인자 취소로 가장 빠른 대기자가 자동 승인되었습니다.
```

관리자 검수 화면에서는 `reviewNotes`에 `대기자 자동승격`이 포함된 제출을 감지해:

- 제출 목록 상태 영역에 `대기자 자동승격` badge 표시
- 제출 상세 모달 상태 영역에 `대기자 자동승격` badge 표시

상태값은 기존대로 `approved`를 유지하므로 `submitted`, `approved`, `waitlist`, `cancelled` 필터 동작은 변경하지 않았다.

### 스킬 반영

자동승격 흔적 저장 및 관리자 badge 표시 규칙을 프로젝트 스킬에 추가하고, `npm run skills:sync`로 인덱스를 재생성했다.

## 변경 이유

운영자는 선착순 미션에서 승인자가 취소되어 대기자가 자동으로 승인된 경우를 일반 승인자와 구분해 알아야 한다. 별도 DB 컬럼을 추가하지 않고 기존 `reviewNotes`를 활용하면 스키마 변경 없이 관리자 화면에서 자동승격 이력을 식별할 수 있다.

## 주의사항 / 알려진 이슈

- 기존 신청/대기/승인 데이터는 보정하지 않았다.
- 자동승격 badge는 이번 변경 이후 발생하는 자동승격 건부터 표시된다.
- 기존에 이미 자동승격되었지만 `reviewNotes` 흔적이 없는 건은 badge로 식별되지 않는다.
- 별도 DB 스키마 변경은 없다.

## 수동 테스트 순서

1. `localhost:5000/api/dev/auto-login` 접속 후 관리자 계정으로 로그인한다.
2. 관리자 > 미션 시스템 관리 > 주제 미션에서 미션 추가/수정 모달을 연다.
3. `행사 정보 (선택)`의 모집 인원/선착순 아래에 모집 방식 안내문구 3줄이 보이는지 확인한다.
4. 선착순 ON, 모집 인원 1명인 테스트 미션을 준비한다.
5. 사용자 A가 신청해 승인 상태가 되고, 사용자 B가 신청해 대기 상태가 되게 한다.
6. 사용자 A가 참여취소한다.
7. 관리자 > 검수 대기 > 해당 세부미션 제출 목록에서 사용자 B가 `승인` 상태와 함께 `대기자 자동승격` badge로 표시되는지 확인한다.
8. 사용자 B의 검수 상세 모달에서도 `대기자 자동승격` badge가 보이는지 확인한다.
