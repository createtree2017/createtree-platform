# GPT 이미지 모델 4종 정리 인수인계

## 작업 일시
- 2026-04-28

## 작업 배경
- OpenAI 이미지 생성 모델을 `GPT-Image-2`, `GPT-Image-1.5` 중심으로 정리하고, 기존 `GPT-Image-1`, `GPT-Image-1 Mini`는 관리자 화면과 컨셉 설정에서 제거해야 했다.
- Gemini 모델은 기존처럼 `Gemini 3.1 Flash`, `Gemini 3.0 Pro`를 유지한다.
- GPT 모델 선택 시에도 Gemini처럼 비율 선택 UI를 사용할 수 있도록 기본 비율 정보를 제공해야 했다.

## 핵심 변경 내용
- 관리자페이지에 표시되는 이미지 생성 모델을 4개로 정리했다.
  - `openai_gpt2` → `gpt-image-2`
  - `openai_gpt1_5` → `gpt-image-1.5`
  - `gemini_3_1`
  - `gemini_3`
- 기존 레거시 모델값은 런타임 설정 조회 시 자동 정리되도록 했다.
  - `openai` → `openai_gpt1_5`
  - `openai_gpt1_mini` → `openai_gpt1_5`
  - `gemini` → `gemini_3_1`
- 시스템 설정 기본값과 클라이언트 폴백값에 4개 모델을 모두 반영했다.
- GPT 모델에도 비율 선택 UI가 표시되도록 `model-capabilities` 기본값에 GPT 비율을 추가했다.
- 사용자 이미지 생성 화면에서 비율 선택 영역을 실제로 표시하도록 `hidden` 클래스를 제거했다.
- 이미지 생성 라우트에서 선택된 내부 모델값을 실제 OpenAI 모델명으로 변환하도록 연결했다.
  - `openai_gpt2` → `gpt-image-2`
  - `openai_gpt1_5` → `gpt-image-1.5`
- 가족사진/스티커/일반 이미지 변환 라우트에서 선택한 비율을 OpenAI 변환 서비스까지 전달하도록 했다.
- OpenAI 호출 시 선택 비율을 API `size` 값으로 변환하도록 했다.
  - 정사각형 → `1024x1024`
  - 가로형 → `1536x1024`
  - 세로형 → `1024x1536`

## 검증 결과
- `npm.cmd run check 2>&1 | Select-String -Pattern ...` 실행 결과, 이번 변경 파일명 기준으로 걸리는 타입 오류는 확인되지 않았다.
- 전체 타입체크는 기존 관리자/페르소나/마일스톤/보호 라우트 등 다른 영역의 기존 타입 오류로 실패한다.
- 이번 요청에는 `!!테스트!!`가 없어서 브라우저 에이전트 검증은 수행하지 않았다.

## 주의사항
- 실행 중인 개발 서버는 재시작해야 `server/utils/settings.ts`의 모델 마이그레이션이 반영된다.
- 서버 재시작 후 `/api/system-settings` 조회가 실행되면 기존 DB의 레거시 모델값이 새 4종 체계로 정리된다.
- `gpt-image-1.5`는 공식 문서에는 언급되지만 프로젝트 권한/계정 상태에 따라 실제 호출 가능 여부가 달라질 수 있으므로, API 오류가 나면 OpenAI 프로젝트 Allowed models에서 `gpt-image-1.5` 노출 여부를 확인해야 한다.
