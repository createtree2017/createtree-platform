## 작업 개요

문화센터 주제미션의 `신청` 세부미션에서 모집인원이 `0` 또는 비어 있을 때 선착순 여부와 관계없이 자동승인되던 문제를 정정했다. 자동승인은 오직 주제미션의 `isFirstCome=true`일 때만 발생하도록 서버 로직과 관리자 안내문구를 맞췄다.

## 핵심 변경 내용

### 신청 자동승인 조건 정정

`submitSubMission`에서 `신청` 타입 세부미션은 일반 세부미션의 `requireReview=false` 자동승인 분기를 타지 않도록 분리했다.

- `isFirstCome=false`: 모집인원이 `0`, `null`, `N` 중 무엇이든 `submitted`로 등록
- `isFirstCome=true` + 모집인원 `0`/비어 있음: `approved` 및 `isLocked=true`로 무제한 자동승인
- `isFirstCome=true` + 모집인원 `N`: 기존 advisory lock 기반 정원 체크 후 `approved` 또는 `waitlist`

관리자 수동 승인 로직(`admin.mission.review.service.ts`)은 수정하지 않았다. 운영자가 예외적으로 모집인원보다 많이 승인할 수 있어야 한다는 정책을 유지한다.

### 관리자 안내문구 정정

주제미션 수정/추가 모달의 행사 정보 영역 안내를 실제 정책과 일치하도록 변경했다.

- 선착순을 선택하지 않으면 모집인원과 관계없이 신청자는 관리자 승인 대기 상태로 등록됩니다.
- 모집인원을 0 또는 비워두고 선착순을 선택하면 무제한 자동승인으로 운영됩니다.
- 모집인원을 입력하고 선착순을 선택하면 정원 내 신청자는 자동 승인되고, 정원이 차면 대기자로 등록됩니다.

### 스킬 반영

프로젝트 스킬에 `신청` 타입 세부미션 자동승인 정책을 추가했다. 이후 같은 영역을 수정할 때 `requireReview=false`를 신청 자동승인 근거로 다시 사용하지 않도록 기준을 남겼다.

## 변경 이유

신청 타입 세부미션은 생성 시 `requireReview=false`로 고정되는 흐름이 있어, 모집인원이 없으면 선착순 여부와 무관하게 자동승인되는 문제가 있었다. 운영 정책은 자동승인을 선착순 선택의 의미로만 사용하므로, 신청 타입은 `isFirstCome`을 기준으로 별도 처리해야 한다.

## 주의사항 / 알려진 이슈

- 기존에 이미 자동승인된 신청 데이터는 보정하지 않았다.
- `선착순 OFF + 모집인원 N`에서 관리자가 N명 이상 수동 승인할 수 있는 현행 동작은 그대로 유지했다.
- `isFirstCome=true + 모집인원 0/null`은 정원 제한과 대기자 없이 무제한 자동승인된다.
- 사용자 목록/상세 화면의 모집인원 표시 정책은 이번 범위에서 변경하지 않았다.

## 수동 테스트 순서

1. `localhost:5000/api/dev/auto-login` 접속 후 관리자 계정으로 로그인한다.
2. 관리자 > 미션 시스템 관리 > 주제 미션에서 테스트 미션을 만든다.
3. 모집인원 `0`, 선착순 OFF로 저장 후 사용자가 신청하면 관리자 검수 대기(`submitted`)로 들어가는지 확인한다.
4. 모집인원 `0`, 선착순 ON으로 저장 후 사용자가 신청하면 즉시 승인(`approved`)되는지 확인한다.
5. 모집인원 `1`, 선착순 ON으로 저장 후 두 명이 신청하면 첫 번째는 승인, 두 번째는 대기자(`waitlist`)가 되는지 확인한다.
6. 모집인원 `1`, 선착순 OFF로 저장 후 여러 명이 신청하면 모두 관리자 검수 대기 상태로 쌓이고, 관리자 수동 승인은 기존처럼 가능한지 확인한다.
