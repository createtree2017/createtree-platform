# 20260423-1 메뉴관리 UI 서브탭 전환

## 변경 배경
관리자 페이지의 "메뉴관리" 탭이 아코디언(접었다 펴기) 형태로 되어 있었는데, 다른 관리 탭들(회원관리, 앱 푸시 알림, 마일스톤)은 모두 가로 서브탭 형태를 사용하고 있어 UI 일관성이 떨어졌음. 또한 좌측 드래그 핸들(GripVertical)이 실제로는 작동하지 않는 상태였음.

## 변경 파일

### `client/src/components/admin/MenuManagement.tsx`
- **아코디언 → 가로 서브탭 전환**: `Accordion` 컴포넌트 제거, `Tabs` 기반 가로 서브탭으로 변경 (다른 페이지와 동일하게 **좌측 정렬** 적용)
- **드래그 핸들 제거**: 작동하지 않는 `GripVertical` 아이콘 및 관련 import 삭제
- **활성/비활성 토글 위치 이동**: 아코디언 우측에 플로팅이던 토글을 "메뉴 설정" 카드 헤더 우측으로 이동
- **순서(order) 수정 기능 추가**: 메뉴 수정 팝업에 순서 입력란(숫자) 추가, 저장 시 order 값도 API로 전송
- **state 단순화**: `openMenus: string[]` (다중 아코디언) → `activeMenuTab: string` (단일 탭 선택)
- **URL 동기화 유지**: `menuItem`, `panel` 쿼리 파라미터 기반 동기화 로직 그대로 유지

## 변경하지 않은 부분
- 서버 API (`admin-routes.ts`): PATCH `/api/admin/main-menus/:id`에서 이미 `order` 필드 지원 중이므로 변경 불필요
- 관리 기능 서브탭 (하위 lazy 컴포넌트들): 모두 기존과 동일하게 동작
- 홈 설정 다이얼로그: 변경 없음

## 동작 확인
- ✅ 코드 수정 완료
- ❌ 브라우저 테스트 미수행 (!!테스트!! 미사용)

## 수동 테스트 요청
1. `localhost:5000/api/dev/auto-login` 접속
2. `/admin?tab=menu-management` 이동
3. 5개 메뉴가 가로 서브탭으로 표시되는지 확인
4. 각 탭 클릭 시 메뉴 설정 + 관리 기능이 정상 표시되는지 확인
5. "수정" 버튼 클릭 → 순서 입력란이 있는지 확인, 순서 변경 후 저장 정상 동작 확인
6. 활성/비활성 토글이 카드 내에서 정상 작동하는지 확인

## 주의사항
- `Accordion` import를 완전히 제거했으므로, 이 컴포넌트에서 아코디언 관련 코드가 남아있으면 빌드 에러 발생
- 순서 변경 시 같은 순서값을 가진 메뉴가 있을 수 있음 (DB에서 unique 제약 없음) — 의도된 동작

- `server/routes/admin-routes.ts` — PATCH API에서 order 필드 처리 (변경 없음, 이미 지원)
